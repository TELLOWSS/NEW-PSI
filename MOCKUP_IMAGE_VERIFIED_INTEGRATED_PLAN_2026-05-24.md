## 3) 목업 대비 현재 놓친 사항 (핵심 누락)

## P0 (즉시 체감 누락)
1. 모바일 정보구조(IA) 불일치
- 목업은 8코어 카드, 현재 소개 허브는 12카드 중심
- 조치: 8코어 표시 모드 추가(12기능은 "확장"으로 접기)

2. 모바일 4번 카드 의미 불일치
- 목업: "현장 지도(위험)"
- 현재: "위험인지 진단(worker-training)"
- 조치: 위험지도 전용 진입점(또는 대시보드 위험지도 딥링크) 신설

3. 모바일 공통 네비게이션 일관성 부족
- 목업은 카드/화면 하단 내비 일관 패턴이 강함
- 조치: 모바일 핵심 페이지 8개에 공통 하단 탭 바 정책 통일

4. PC 목업의 "운영형 한눈 보드" 고정 요약 부족
- 현재는 기능 풍부하나 첫 인지 3초 요약이 약함
- 조치: PC 대시보드에 "목업형 요약 보드 모드"(기본) 추가, 기존은 고급모드로 유지

## P1 (완성도 누락)
5. 카드별 즉시 액션 텍스트/상태 배지 통일 부족
- 조치: 8코어 카드 각각에 1개 주 CTA + 1개 상태배지 고정

6. 목업 톤 기준 타이포/간격 시스템 고정 부족
- 조치: 소개 허브/핵심 페이지에 spacing/token 고정값 적용

7. 운영 증빙 흐름의 "한 곳 확인" 부족
- 조치: 소개 보드에 Reports 검증 결과(성공/주의/대기) 배지 + 진입 버튼 고정

## P2 (안정화 누락)
8. 이미지-실화면 정합 QA 양식 부재
- 조치: 목업 정합 전용 QA 시트 생성(화면별 pass/fail + 캡처 링크)

9. 재시작 즉시 실행 패킷(중단 복구) 문서화 부족
- 조치: 본 문서 7장 재시작 런북을 운영 표준으로 고정

  - 공통 하단 내비게이션 정책 적용(핵심 8페이지)

### Phase D. 위험지도 전용 동선 보강 (익일)
- 목표: 목업 4번 카드 의미 일치
- 작업:
  - 대시보드 위험지도 섹션으로 딥링크 이동 또는 별도 page 신설
  - 모바일에서 위험지도 단독 확인 흐름 제공
- 대상 파일:
  - pages/Dashboard.tsx

  - 4번 카드 클릭 시 반드시 위험지도 화면/섹션으로 이동

- 대상 파일:

- 완료 기준:

- 목표: "보이는 것"과 "동작"을 함께 합격
- 작업:

  - pass/fail + 증빙 캡처 + 회귀 여부 기록
- 산출물:
  - 신규 QA 문서 1종 + 런로그 1종
- 완료 기준:
