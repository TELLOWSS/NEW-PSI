# PSI 자동 문자/MMS 발송 설정

근로자 개별 리포트 화면의 자동문자 기능은 SOLAPI MMS 연동으로 동작합니다.

## 설치

1. 의존성 설치
   `npm install`
2. Supabase SQL Editor에서 [supabase_report_message_logs_migration.sql](supabase_report_message_logs_migration.sql) 실행

## 필수 환경변수

`.env.local` 또는 배포 환경에 아래 값을 설정합니다.

- `SOLAPI_API_KEY`
- `SOLAPI_API_SECRET`
- `SOLAPI_SENDER` : 솔라피에 등록된 발신번호, 예: `01012345678`

호환용으로 아래 이름도 함께 인식합니다.

- `COOLSMS_API_KEY`
- `COOLSMS_API_SECRET`
- `COOLSMS_SENDER`

## 동작 방식

- 개별 리포트 화면에서 전화번호 입력 후 자동문자 클릭
- `worker_uuid` 또는 이름 기준으로 등록 근로자 전화번호를 먼저 자동 조회 시도
- 서버가 양면 리포트 이미지를 JPG(200KB 이하)로 자동 압축
- SOLAPI MMS 2건 발송
  - 1/2: 요약 페이지
  - 2/2: 상세 해설 페이지
- 리포트에 `worker_uuid`가 있으면 `workers.phone_number`도 함께 갱신 시도

## 관리자 이력 탭

- 관리자 근로자 관리 화면에 `문자 발송 이력` 탭이 추가되었습니다.
- 근로자별 최근 문자 발송 이력, 상태, 발송 페이지 수, 메모를 확인할 수 있습니다.
- 리스트에서 `문자이력` 버튼을 누르면 해당 근로자의 이력 화면으로 즉시 이동합니다.
- 로그는 `report_message_logs` 테이블 기준으로 표시됩니다.
- 최근 7일 / 30일 / 90일 / 전체 기간 필터와 성공/실패 필터를 지원합니다.
- 메모/상태/전화번호 기준의 즉시 검색 필터를 지원합니다.
- 현재 화면 기준 이력은 CSV, JSON, Excel 형식으로 바로 내려받을 수 있습니다.
- 기본 30건씩 불러오며, 이력이 더 많으면 `이력 더 보기`로 추가 로드할 수 있습니다.
- 현재 필터 기준 주요 발송 공급자 통계도 함께 표시됩니다.
- 성공률 카드와 실패 이력 강조 표시로 재확인 대상을 빠르게 식별할 수 있습니다.
- 최신 리포트 원본이 연결된 경우 관리자 이력 탭에서 곧바로 상세 리포트를 다시 열 수 있습니다.
- 최근 최대 6개월 기준 월별 성공/실패 발송 추이 차트가 함께 표시됩니다.
- 현재 캐시에 적재된 팀 기준 성공률 비교 차트가 제공됩니다.
- 실패/보류 로그는 메시지 문구를 바탕으로 주요 실패 사유 분류 차트로 요약됩니다.
- 실패 빈도가 높은 원인은 `재발송 우선순위 가이드` 카드로 정리되어 현장 대응 순서를 바로 확인할 수 있습니다.
- 최신 마이그레이션 적용 시 실패 로그에는 `failure_category`가 저장되어 더 안정적인 실패 원인 통계가 가능합니다.
- 성능 최적화를 위해 `report_message_logs`에는 `(worker_id, created_at desc)`, `(worker_name, created_at desc)` 복합 인덱스를 권장/포함합니다.
- 최신 마이그레이션에는 운영 대시보드 확장을 위한 집계 뷰도 포함됩니다.
  - `report_message_monthly_summary`
  - `report_message_team_summary`
  - `report_message_failure_summary`
- 최신 실패 상태만 모아보는 `report_message_retry_queue` 뷰도 포함됩니다.
- 관리자 문자 이력 탭 상단에는 위 집계 뷰를 읽는 전체 운영 대시보드 패널이 표시됩니다.
- 운영 대시보드 패널은 CSV, JSON, Excel 형식으로도 바로 내보낼 수 있습니다.
- 재시도 큐는 `전체`, `재시도 가능만`, `차단 대상만` 필터로 빠르게 좁혀볼 수 있습니다.
- 재시도 큐 카드에서는 `이력 열기`, `리포트 열기`, `근로자 수정`으로 바로 후속 조치를 이어갈 수 있습니다.
- 재시도 큐 카드에는 전화번호 누락, 등록 근로자 미매칭, 리포트 원본 부재 같은 차단 요인이 함께 표시됩니다.

## Vercel 무료 플랜 대응

- 관리자 이력 탭은 전체 로그를 한 번에 불러오지 않고, 선택한 근로자 1명 기준으로만 조회합니다.
- 조회 결과는 브라우저에서 5분간 캐시되어 반복 클릭 시 불필요한 API 호출을 줄입니다.
- `vercel.json`에서 관리자 문자/로그 관련 함수의 `maxDuration`을 명시해 무료 플랜 기본 설정에서의 예측 가능성을 높였습니다.
- 현재 앱 내부에서 Vercel 실제 사용량 수치를 직접 읽어오지는 않으며, 대신 호출 절감형 UI로 한도 소모를 줄이는 방식입니다.

## 유의사항

- SOLAPI Node.js SDK는 서버 환경(Node.js 18+)에서만 동작합니다.
- MMS 업로드 이미지는 JPG 형식 / 200KB 이하만 허용됩니다.
- 관리자 인증이 된 상태에서만 자동 발송 API를 사용할 수 있습니다.
- 문자공유 버튼은 기기 공유시트를 이용한 수동 첨부 방식이고, 자동문자 버튼은 서버가 직접 MMS를 발송하는 방식입니다.
