## 👨‍💻 1. 개발자 정보 패키지

### 목적
기능 구현, 버그 수정, 코드 리뷰, 테스트 자동화

### 필요 문서 구조
```
_DOCS_DEV/
├─ README.md (개발자 빠른 시작)
├─ SETUP.md (로컬 환경 설정)
├─ ARCHITECTURE.md (프로젝트 구조)
├─ COMPONENT_REFERENCE.md (공유 컴포넌트 카탈로그)
├─ TYPE_DEFINITIONS.md (데이터 타입 정의)
├─ API_INTEGRATION.md (API 연결 가이드)
├─ MOBILE_SPECS.md (모바일 구현 스펙)
├─ PC_SPECS.md (PC 구현 스펙)
├─ TESTING.md (테스트 작성 방법)
└─ TROUBLESHOOTING.md (일반적 문제)
```

### 포함할 정보

**❌ 개발자가 알 필요 없는 것**
- 실무자 사용 시나리오 (→ 실무자 문서로)
- PM 일정 관리 (→ PM 문서로)
- 마케팅 메시지 (→ 설계자 문서로)
- 과거 회의록 (→ 아카이브로)

**✅ 개발자가 알아야 할 것**
```markdown
### 예: 9번 수기 데이터 입력 화면 스펙

**파일 위치**
- 컴포넌트: pages/JudgmentTaggingInput.tsx
- 타입: types.ts (WorkerRecord.judgmentTags)
- 자동화 스크립트: npm run check:judgment-tagging:full

**구현 요구사항**
1. 필드 입력
   - 필드 배열: judgmentTags[0..9] (10개 문항)
   - 각 필드 타입: string, 최대 500자
   - 필수 필드: 모두 필수

2. 상태 관리
   - 로컬 state: current step / filled fields
   - 저장: localStorage + IndexedDB

3. 검증 규칙
   - 빈 값 불허
   - 특수문자 화이트리스트
   - 중복 입력 경고

4. 테스트 케이스
   - npm run check:judgment-tagging:full
   - 예상 출력: reports/judgment-tagging-quality.json
   - 합격 기준: PASS
```

## 👨‍💼 2. 실무자 정보 패키지

### 목적
현장에서 앱 사용, 데이터 입력, 리포트 해석

### 필요 문서 구조
```
_DOCS_USER/
├─ README.md (사용자 빠른 시작)
├─ MOBILE_USER_GUIDE.md (화면별 가이드)
├─ PC_OPERATION_GUIDE.md (PC 관리자 가이드)
├─ SCENARIO_WORKFLOWS.md (일반적 작업 흐름)
├─ FAQ.md (자주 묻는 질문)
├─ TROUBLESHOOTING_USER.md (사용 중 오류 해결)
└─ VIDS/ (선택) 화면 녹화 튜토리얼
```

### 포함할 정보

**❌ 실무자가 알 필요 없는 것**
- 코드 구조 (→ 개발자 문서)
- API 상세 (→ 개발자 문서)
- 기술 용어: TypeScript, React, IndexedDB
- 구현 원리

**✅ 실무자가 알아야 할 것**
```markdown
### 예: 9번 화면 사용 가이드

## 📱 9번: 수기 데이터 입력

### 이 화면에서 하는 일
근로자가 종이 기록지에 작성한 내용을 앱에 한 글자씩 입력하는 단계입니다.

### 시작하기 전에
1. 근로자 이름 확인 (화면 상단)
2. 종이 기록지 준비
3. 펜/마우스 준비

### 단계별 진행

**1단계: 첫 번째 문항 읽기**
- 화면 상단 "질문 1/10"을 봅니다
- 종이 기록지에서 같은 번호 찾기

**2단계: 입력**
- 텍스트 박스에 클릭
- 입력 시작 (영문, 숫자, 한글 모두 가능)

**3단계: 다음**
- "다음 문항" 버튼 클릭
- 2번 반복 (~9번까지)
- 10번: "완료" 버튼

**4단계: 검증**
- 첫 번째 화면으로 돌아가기
- 확인 표시(✓) 개수 확인: 10/10 완료?

### 주의사항
- 한 글자도 빠지면 안 됨
- 오타는 뒤로가기로 수정 가능
- 브라우저 닫으면 저장 안 됨 (완료까지)

### 예상 시간
- 10개 문항: 약 3-5분

### 문제 발생 시
Q: "입력된 글자가 사라졌어요"
A: 브라우저 새로고침 전까지는 저장됨. 새로고침 후 문제 반복되면 화면1(경보)로 돌아갔다가 다시 들어오세요.
```
