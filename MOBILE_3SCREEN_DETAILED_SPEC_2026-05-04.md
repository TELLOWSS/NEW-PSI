# MOBILE 3SCREEN DETAILED SPEC (Dashboard / OCR / AI Risk)

- 작성일: 2026-05-04
- 기준 목업: 다크 / 라이트+다크 혼합 / 완전 라이트
- 적용 범위: 모바일 우선(320x568, 360x800, 375x812, 390x844)
- 목적: 현장 사용성 최적화 + 동일 정보구조 기반 테마 전환 + QA 자동 점검 가능 구조화

---

## 0) 공통 원칙

### 0-1. UX 원칙
1. 3초 내 핵심 상태 인지
2. 1탭 내 핵심 액션 도달
3. 오터치 최소화(터치 타겟 최소 44x44)
4. 색상 의존 금지(색 + 텍스트 + 아이콘 동시 제공)

### 0-2. 레이아웃 원칙
- 상단: 상태/알림
- 본문: 핵심 KPI + 시각화 + 최근 작업
- 하단: 글로벌 탭 바 + 화면별 고정 CTA(필요 시)
- 모든 화면에서 스크롤 1회 이내에 핵심 정보/행동 80% 노출

### 0-3. 반응형 규칙
- 320: 단일 열, 숫자 우선, 설명 축약
- 360/375: 표준 레이아웃
- 390: 카드 간격 확장, 터치 여유 강화

### 0-4. 테마 규칙
- 정보 구조/컴포넌트 순서는 테마와 무관하게 동일
- 다크/라이트는 색상 토큰만 전환
- 혼합 모드는 핵심 경고/분석 카드에 다크 강조, 배경은 라이트 유지

## 1) Screen A — Dashboard (홈)

### 1-1. 사용자 목표
- 당일 위험 상태를 즉시 파악
- 현재 우선 조치 대상을 빠르게 확인
- 분석 화면으로 즉시 이동

### 1-2. 정보 구조 (위→아래)
1. 헤더: 햄버거/로고/알림
2. 인사/상태 문구: “오늘도 안전한 현장 운영”
3. KPI 3카드: `전체 근로자`, `위험군`, `무위험`
4. 위험 분포 블록: 도넛 + 범례(적/황/녹)
5. 최근 분석 리포트 리스트(최신 3개)
6. 1차 CTA: `시작하기`
7. 2차 CTA: `둘러보기`
8. 하단 탭 바: 홈/분석/리포트/더보기

### 1-3. 컴포넌트 스펙
- KPI 카드(3개)
  - 필수: 라벨, 수치, 증감(▲/▼), 보조설명
  - 금지: 수치 없는 상태 문구만 표기
- 위험 분포
  - 도넛 중앙: 총 대상 수
  - 우측 범례: 색상 + 텍스트(건수/비율)
- 최근 리포트 아이템
  - 제목(1줄), 일시, 상태칩(위험/주의/양호)
  - 터치 영역: 카드 전체

### 1-4. 상태/예외
- 로딩: KPI 스켈레톤 3개 + 도넛 스켈레톤
- 빈 데이터: “오늘 분석 기록이 없습니다” + `분석 시작`
- 오류: “데이터를 불러오지 못했습니다” + `다시 시도`

### 1-5. 이벤트 추적
- `dashboard_viewed`
- `dashboard_kpi_card_tapped`
- `dashboard_start_cta_tapped`
- `dashboard_recent_report_tapped`

## 2) Screen B — OCR 분석

### 2-1. 사용자 목표
- 이미지/문서 OCR 분석을 빠르게 시작
- 분석 전 체크 항목 확인
- 처리 가능/불가 조건을 사전 인지

### 2-2. 정보 구조 (위→아래)
1. 상단 바: 뒤로가기 + 화면 제목
2. 히어로 아이콘 + 제목(`수기 기록지 OCR 분석`)
3. 보조 설명(1~2줄)
4. 체크리스트(최대 4개)
   - 다국어 OCR 지원(한국어/영어/중국어 등)
   - 손글씨 인식 최적화
   - 이미지 보정/왜곡 보정
   - 자동 위험 매핑
5. 고정 CTA: `분석 시작`
6. 하단 탭 바

### 2-3. 컴포넌트 스펙
- 체크리스트 아이템
  - 왼쪽 체크 아이콘 + 오른쪽 한 줄 설명
  - 긴 문구는 2줄까지 허용, 3줄 금지
- 분석 시작 버튼
  - 전체폭 버튼 1개
  - 로딩 시 스피너 + 중복탭 방지

### 2-4. 상태/예외
- 카메라 권한 거부: 권한 안내 모달 + 설정 이동
- 파일 형식 오류: 허용 형식 안내 + 재선택 버튼
- OCR 실패: 실패 사유 + `재시도` + `수동 입력 전환`

### 2-5. 이벤트 추적
- `ocr_screen_viewed`
- `ocr_start_tapped`
- `ocr_permission_denied`
- `ocr_failed`
- `ocr_retry_tapped`

## 3) Screen C — AI 리스크 분석

### 3-1. 사용자 목표
- 리스크 점수와 위험 분포를 즉시 이해
- 위험군별 건수 확인 후 조치 화면 이동

### 3-2. 정보 구조 (위→아래)
1. 상단 바: 뒤로가기 + 제목(`AI 리스크 분석`)
2. 점수 게이지(원형): `78/100` 형식
3. 위험군 요약 카드 3개: 적/황/녹 건수
4. 액션 리스트 3개
   - `경향 분석`
   - `개별 점검 영역`
   - `AI 인사이트`
5. 필요 시 하단 CTA: `조치 시작`
6. 하단 탭 바

### 3-3. 컴포넌트 스펙
- 점수 게이지
  - 숫자/최대점/텍스트 라벨 고정
  - 색만으로 상태 전달 금지(라벨 필수)
- 요약 카드(3개)
  - 카드마다 건수 + 구분 라벨 + 강조 라인
- 액션 리스트
  - 좌측 아이콘, 중앙 제목, 우측 화살표
  - 한 줄 제목 원칙

### 3-4. 상태/예외
- 분석 전: “분석 결과가 없습니다” + `분석 실행`
- 데이터 지연: 마지막 갱신 시각 표시
- 모델 오류: “AI 점수 산출 실패” + `재산출`

### 3-5. 이벤트 추적
- `risk_screen_viewed`
- `risk_gauge_viewed`
- `risk_bucket_tapped`
- `risk_action_item_tapped`
- `risk_recompute_tapped`
