## 2) 오늘 작업 요약
1. 직접등록 추가 등록 미반영 이슈 수정
   - 대상: `bulk-upload-workers` 매칭 로직
   - 원인: 이름/공종/팀이 같고 후보가 1명이면 신원정보가 달라도 강제 매칭되어 UPDATE로 처리됨
   - 조치: 전화번호/생년월일/여권번호 신원정보 충돌 시 기존 매칭을 막고 신규 INSERT로 진행

2. 중복 등록자 삭제 실패 이슈 수정(단건/일괄)
   - 대상: `delete-worker`, `delete-workers`
   - 원인: 일부 Supabase/RLS 환경에서 delete 후 `select` 반환값이 비어도 실제 삭제는 수행되는데, 기존 로직이 실패로 처리
   - 조치: delete 에러가 없으면 성공으로 간주하고, 반환 데이터가 비어있을 때는 요청 id(ids)를 성공 결과로 반환

3. 입력 UX 보정(연계 반영)
   - 대상: 근로자 직접등록/인라인수정 전화번호 입력
   - 조치: 하이픈 포맷 표시 대신 숫자만 입력/표시되도록 정리
