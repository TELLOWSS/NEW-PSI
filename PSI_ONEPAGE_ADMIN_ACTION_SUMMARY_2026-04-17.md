## 1) 지금 바로 할 일
재분석 배치를 실행한 뒤 아래 5개 숫자를 전/후로 반드시 기록하세요.

1. 서버 성공
2. OCR 처리 실패
3. 서버 라우트 실패
4. 사전 검증 실패
5. UNKNOWN 비중

---

## 2) 5분 점검 루틴
1. 정상 스캔 샘플 3~5건 선정
2. 재분석 실행 전 수치 기록
3. 재분석 실행
4. 실행 후 수치 기록
5. 개선/정체/악화 판정 후 조치

판정 기준:
- 개선: 서버 성공↑, 처리 실패↓, UNKNOWN↓
- 정체: 변화 미미
- 악화: 라우트 실패↑ 또는 UNKNOWN↑

---

## 3) 이상 징후별 즉시 조치
- 서버 라우트 실패 증가
  - API 경로/배포 상태/환경변수 점검
- 사전 검증 실패 증가
  - 이미지 포맷/손상/길이(업로드 품질) 점검
- OCR 처리 실패 증가
  - 모델 응답 품질, 실패 코드 분포, 폴백 성공률 점검
- UNKNOWN 증가
  - 실패 메시지 원문 수집 후 코드 매핑 보강 요청

## 5) 운영 원칙 (중요)
- 실패는 숨기지 말고 코드로 분류해 조치한다.
- 정상분류는 근거 문장(왜 정상인지)을 반드시 남긴다.
- 재분석은 대량보다 우선순위(실패코드/영향도) 기준으로 실행한다.
- 근로자 안내 문구는 비난 금지, 보완 중심으로 유지한다.

## 6) 이번 개선의 현장 의미
- 정상 스캔이 문구 때문에 실패로 분류되던 오탐 가능성이 낮아졌습니다.
- 점수 누락 응답이 즉시 실패로 떨어지던 경로가 완화되었습니다.
- 따라서, 앞으로는 숫자(5개 KPI) 기반으로 운영 품질을 판단해야 정확합니다.
