# NEW-PSI 수정보강 진행 체크리스트

기준일: 2026-06-16  
목적: 대우건설 공모 제출 및 대외 공개 가능한 제품 신뢰도 기준으로, 수정보강 계획의 완료/진행/대기 상태를 고정 관리한다.

## 1. 클리어 완료

- [x] 프로그램 정체성 재정리
  - 근로자 처벌/채점 중심 인상을 줄이고, `응답품질`, `품질 판단`, `보완 반영`, `보호 조치` 중심 문구로 전환
  - 개인 순위/감점 공개가 아닌 관리자 개선 추적과 익명화 교육 자료 중심으로 정리

- [x] 유명무실한 근로자 코드/사번 표현 정리
  - 화면 표시에서 `사번`, `식별 불가` 표현을 `관리 식별`, `식별 보완 필요` 흐름으로 정리
  - 근로자가 기억해야 하는 코드가 아니라 관리자가 확인 가능한 식별정보라는 방향으로 전환

- [x] CSV 기반 등록 흐름 정리
  - 대량 등록 안내를 Excel 중심 표현에서 CSV 중심 표현으로 정리
  - 직접 등록과 CSV 등록의 역할을 분리해 비전문 관리자도 이해하기 쉽게 개선

- [x] 관리자 PIN 흐름 제거
  - 브라우저에 저장하는 관리자 PIN 방식 제거
  - 운영자 확인, 관리자 비밀번호, 서버 운영 비밀키 중심의 보안 문구로 정리
  - 운영 체크리스트와 배포 체크리스트의 PIN 잔여 문구 제거

- [x] 운영 화면의 데모/샘플 노출 정리
  - 근로자 교육 화면의 개발용 세션은 운영 환경에서 숨김 처리
  - 개입 추천 화면의 가짜 기본 카드 제거
  - 실데이터가 없을 때 가상 차트 데이터를 만들지 않고 데이터 대기 상태로 표시
  - `샘플표시`, `데모 모드`처럼 외부 심사자가 미완성으로 느낄 수 있는 표현 정리

- [x] 검증 및 배포
  - 타입 검사 통과
  - 관리자 인증 보호 테스트 통과
  - 빠른 전체 검증 통과
  - GitHub main 동기화 완료
  - Vercel production 배포 완료
  - 운영 주소 200 OK 확인
  - 최근 production 오류 로그 없음 확인

## 2. 현재 진행 중

- [x] 잔여 용어 오염 재스캔 1차
  - `사번/QR ID`, `PIN`, `샘플`, `데모`, `임시`, `테스트용` 등 대외 신뢰를 낮출 수 있는 표현 재확인
  - 실제 운영 기능명과 개발자 내부 표현을 분리

- [x] 설문 인텔리전스 데모 수치 제거
  - 실데이터가 없을 때 임의의 설문 수치가 표시되지 않도록 정리
  - 빈 상태는 `현장 데이터 대기`로 표시

- [x] QA 시드 운영 노출 제한
  - 다국어 QA 검증용 데이터 주입은 개발 환경 또는 명시 허용 환경변수에서만 동작하도록 제한
  - 콘솔 로그의 `샘플 데이터` 표현을 `검증용 데이터`로 정리
