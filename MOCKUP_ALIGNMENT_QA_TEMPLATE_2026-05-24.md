## 1) 공통 검증 규칙

- 검증 단위: 화면 1개당 `구조`, `동선`, `상태`, `증빙` 4항목
- 판정 기준: `PASS` / `FAIL` / `BLOCKED`
- 증빙 필수: 캡처 파일명 또는 재현 로그 1개 이상
- FAIL 처리 기준: 화면은 보이지만 동선 미작동, 상태 표시 불일치, 핵심 텍스트 오표기
- BLOCKED 처리 기준: 데이터 부재/권한 미충족/외부 연동 장애로 검증 불가
