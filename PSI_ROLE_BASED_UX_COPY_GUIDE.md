# PSI 역할별 UX 카피 가이드

## 1. 목적
본 문서는 PSI의 역할 적응형 표현 정책을 실제 화면 카피로 일관되게 적용하기 위한 기준이다.

## 2. 의미 상태 체계
- **보완 권고**: 기본 정보는 있으나 안전 포인트 보강 필요
- **추가 확인 필요**: 해석 신뢰도 또는 근거가 부족함
- **우선 검토 필요**: 관리자가 먼저 봐야 하는 상태
- **확정 가능**: 현재 기준에서 승인 가능한 상태

## 3. 근로자용 카피
### 사용
- “안전 포인트를 조금 더 적어주세요.”
- “현장 설명을 보강하면 더 정확한 판단이 가능합니다.”
- “추가 확인이 필요한 항목이 있습니다.”

### 금지
- “점수 미달입니다.”
- “작성 상태가 불량합니다.”
- “오답입니다.”

## 4. 관리자용 카피
### 사용
- “재검토 우선순위 상향”
- “반복 위험 패턴 확인”
- “승인 전 근거 보강 필요”
- “정정 후 재분석 권장”

### 금지
- “문제 직원”
- “부적합 근로자”
- “불량 문서”

## 5. 경영진용 카피
### 사용
- “현장별 위험 분포 편차 확대”
- “반복 리스크 유형 집중 관리 필요”
- “예방 개입 효과가 개선 추세로 확인됨”

## 6. 화면별 우선 문구
### 업로드 화면
- “작성 내용을 바탕으로 현장 위험 신호를 정리합니다.”

### 결과 화면
- “현재 상태” → “판단 근거” → “보완 행동” 순서 유지

### 관리자 검수 화면
- “왜 이 항목이 우선 검토 대상인지”를 한 문장으로 제시

### 대시보드
- 점수보다 추세, 반복 패턴, 조치 이행률 우선

## 7. 구현 메모
- 상태명은 `semantic_state` 기준으로 렌더링
- 메시지는 `message_policy_version` 기준으로 관리
- 역할별 카피는 공통 템플릿에서 파생

## 8. 코드 매핑 규칙
- 공통 상태어는 [utils/brandLabels.ts](utils/brandLabels.ts)에서 관리한다.
- 역할별 화면은 아래 우선순위로 문구를 결정한다.
	1. 공통 브랜드 상수 확인
	2. 역할별 보조 문장 결합
	3. 화면 특화 설명 추가
- 관리자 화면
	- 기본 상태명: `확인 필요`, `보완 검토`, `즉시 확인 필요`
	- 운영 액션: `스마트 재분석`, `직접 재분석`, `다시 확인`
- 근로자 화면
	- 상태명은 가능하면 `확인 필요`, `보완 필요`, `안내`처럼 심리 저항이 낮은 표현을 우선한다.
- 새 상태어가 필요하면 화면에서 임시 문구를 늘리지 말고 상수 파일에 먼저 등록한다.
