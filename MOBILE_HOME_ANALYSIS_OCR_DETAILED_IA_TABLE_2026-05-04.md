| 구역 | 우선순위 | 내용 | 표시 규칙 | 액션 |

| 탭바 | 최상 | 홈/분석/리포트/근로자/더보기 | 항상 하단 고정 | 탭 이동 |

### 2-3. 컴포넌트 규칙
- KPI 카드는 3개 고정, 4개 이상 금지
- 위험 분포는 색상 + 라벨 + 건수 동시 표기
- 최근 리포트는 3건까지만 노출, `전체 보기` 별도 분리
- 홈 첫 화면에서 긴 표/리스트 금지

### 2-4. 접기/숨김 규칙
- 320폭에서는 보조 설명 축약
- 최근 리포트 설명 텍스트 제거 가능
- 알림 카운트만 유지, 텍스트형 알림 배너는 홈 기본면에서 숨김

### 2-5. 성공 기준
- 사용자가 홈 진입 후 5초 내 `시작하기`를 인지 가능
- 위험 분포와 최근 리포트를 한 스크롤 내 확인 가능

## 3) Screen B · 분석(Analysis Hub)

### 3-1. 사용자 질문
- 분석 메뉴 중 무엇을 먼저 써야 하나?
- OCR, AI 리스크, 리포트가 어떻게 이어지나?
- 지금 필요한 분석은 무엇인가?

### 3-2. 역할 정의
`분석` 탭은 여러 분석 기능의 나열이 아니라, 사용 목적별 분기 허브여야 한다.

### 3-3. IA 표

| 구역 | 우선순위 | 내용 | 표시 규칙 | 액션 |
| --- | --- | --- | --- | --- |
| 상단 바 | 상 | 제목 `분석` + 알림/도움말 | 단순 헤더 | 도움말 이동 |
| 현재 추천 배너 | 최상 | `지금 필요한 작업` 1건 추천 | 우선순위 1건만 | 해당 분석 이동 |
| 분석 카드 1 | 최상 | OCR 분석 | 아이콘 + 설명 1줄 | OCR 이동 |
| 분석 카드 2 | 최상 | AI 리스크 분석 | 점수/위험군 요약 포함 가능 | AI 리스크 이동 |
| 분석 카드 3 | 중 | 최근 분석 결과 보기 | 최근 업데이트 시간 | 최근 결과 이동 |
| 진행중 작업 | 중 | 분석중/대기중 작업 | 최대 2건 | 이어보기 |
| 빠른 진입 | 중 | `최근 파일 재분석`, `리포트 보기` | 버튼형 | 즉시 이동 |
| 탭바 | 최상 | 고정 | 공통 | 탭 이동 |

### 3-4. 구성 원칙
- 분석 허브는 메뉴판이 아니라 추천 기반이어야 함
- 카드 설명은 1줄 원칙
- 분석 기능이 늘어나도 모바일에서는 3~4개 카드만 노출
- 고급 분석은 `더보기` 또는 PC로 이관

### 3-5. 카드 우선순위 규칙
1. OCR 분석
2. AI 리스크 분석
3. 최근 결과/진행중 작업
4. 고급 도구(후순위)

### 3-6. 성공 기준
- 사용자가 `분석` 탭에서 1번 탭으로 OCR 또는 AI 리스크에 진입 가능
- 진행중 작업이 있으면 즉시 이어보기 가능

## 4) Screen C · OCR 분석(OCR)

### 4-1. 사용자 질문
- 지금 바로 무엇을 업로드하면 되나?
- 분석 전에 뭘 확인해야 하나?
- 실패하면 어떻게 다시 시도하나?

### 4-2. IA 표

| 구역 | 우선순위 | 내용 | 표시 규칙 | 액션 |
| --- | --- | --- | --- | --- |
| 상단 바 | 상 | 뒤로가기 + 제목 `OCR 분석` | 단순 | 이전 이동 |
| 히어로 | 상 | 아이콘 + `수기 기록지 OCR 분석` | 강한 중심 정렬 | 없음 |
| 설명 문구 | 중 | 1~2줄 설명 | 2줄 이내 | 없음 |
| 체크리스트 1 | 상 | 다국어 OCR 지원 | 체크 아이콘 | 없음 |
| 체크리스트 2 | 상 | 손글씨 인식 최적화 | 체크 아이콘 | 없음 |
| 체크리스트 3 | 상 | 이미지 보정/왜곡 보정 | 체크 아이콘 | 없음 |
| 체크리스트 4 | 중 | 자동 위험 매핑 | 체크 아이콘 | 없음 |
| 업로드 상태 | 상 | 파일 선택 여부/권한 상태 | 상태 텍스트 | 재선택/권한 |
| 모드 토글 | 상 | `빠른분석 / 상세검수` | 640px 미만에서만 노출, 기본 `빠른분석` | 모드 전환 |
| 1차 CTA | 최상 | `분석 시작` | 전체폭 하단 고정 | 분석 실행 |
| 실패 안내 | 중 | 실패 사유 + 재시도 | 오류 시만 표시 | 재시도 |
| 수동 전환 | 중 | `수동 입력 전환` | 실패 시 노출 | 수동 입력 |
| 탭바 | 최상 | 고정 | 공통 | 탭 이동 |

### 4-3. 상태 정의
- 기본: `빠른분석` + 핵심 KPI + CTA
- 파일 선택 완료: CTA 활성
- 분석 중: 버튼 로딩 + 중복탭 금지
- 실패: 실패 사유 + 재시도 + 수동 입력 전환
- 상세검수: 재분석/백업/관리자 일괄 처리 패널 노출
- 성공: 결과 보기 또는 AI 리스크로 이동

### 4-4. 모바일 최적화 원칙
- 업로드 영역 과대화 금지
- 설명보다 CTA 우선
- `빠른분석`에서는 운영/백업 패널 기본 숨김
- 카메라 권한 안내는 모달 1회로 충분
- 파일 형식 오류는 즉시 한 줄 메시지로 처리

### 4-5. 성공 기준
- 사용자가 페이지 진입 후 1스크롤 이내에 `분석 시작` 도달
- `빠른분석` 상태에서 긴급 현황과 1차 행동이 첫 화면에 같이 보여야 함
- 실패 시 재시도 경로를 텍스트 없이도 인지 가능

## 10) 승인 체크리스트 (배포 전)

- 첫 화면에서 `위험등급 + 즉시조치 + 담당/기한`이 한 스크린에 보이는가
- 스크롤 없이 1차 CTA가 보이는가
- P2 정보가 기본 접힘 상태로 시작하는가
- OCR에서 `빠른분석` 기본 진입 후 `상세검수` 전환이 1탭으로 가능한가
- AI 리스크에서 텍스트 요약만으로도 상위 위험 대상과 다음 행동을 이해할 수 있는가
- 같은 작업을 30초 이내에 완료 가능한가
- 오터치/재탐색이 반복되는 요소가 제거되었는가
