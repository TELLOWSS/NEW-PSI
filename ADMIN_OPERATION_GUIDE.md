# PSI 관리자 운영 매뉴얼 (모드 분리/링크 보안 기준)

## 1) 운영 모드 개요

- 관리자 모드
  - 전체 관리 기능 접근 가능
  - 수동 잠금 및 유휴 자동잠금(10분) 적용

- 근로자 모드
  - QR 링크로 진입
  - 서명 화면만 노출 (관리자 셸 비노출)
  - 단순화 UI 적용

## 2) 일일 운영 루틴 (권장)

1. 관리자 잠금 해제 후 `관리자 음성교육` 페이지 진입
2. 안내 문구 입력 및 언어 선택 (필요 시 말레이시아어 `ms-MY` 포함)
3. `생성` 클릭 → QR/공유 텍스트 발급
4. 필요 시 `풀스크린 표시`로 QR 가독성 확인 후 메신저로 공유 텍스트 전달
5. 만료 임박/만료 시 `링크 재발급`
6. 필요 시 테스트 세션 정리(삭제)

## 3) 링크 정책

- 링크는 `sessionId + exp + sig` 형태의 서명 링크로 발급됨
- 기본 만료: 12시간 (`TRAINING_LINK_TTL_MINUTES`로 조정 가능)
- 만료 시 근로자 페이지에서 자동 차단
- 관리자 페이지에서 재발급 가능

## 4) 재발급 운영 절차

1. `관리자 음성교육`에서 대상 세션 불러오기
2. QR 영역의 `링크 재발급` 클릭
3. 신규 URL/QR 공유
4. 하단 `링크 생성/재발급 이력`에서 시간/만료/세션 확인

## 4-1) QR 접속 불가 근로자 대응

1. `관리자 음성교육`의 QR 영역에서 하단 공유 텍스트를 복사
2. 근로자 개인 메신저(문자/카카오톡/WhatsApp 등)로 링크 직접 전달
3. 근로자 휴대폰 카메라 인식이 어려우면 관리자 휴대폰 브라우저에서 링크를 직접 열어 전달
4. 링크 만료 또는 접속 실패 시 `링크 재발급` 후 최신 링크만 다시 전달
5. 공용 기기 사용 시 제출 후 성공 화면을 관리자에게 확인시키고 브라우저를 종료

## 5) 근로자 제출 정책

- 유효 링크만 제출 허용
- 만료 링크 제출 차단
- 동일 세션 + 동일 이름 중복 제출 차단
- 제출 완료 후 근로자 화면 재제출 버튼 비활성

## 6) 현장 사고 예방 포인트

- 근로자에게는 관리자 링크 공유 금지
- QR은 작업구역별로 최신 링크만 노출
- 근무 교대 시 만료 여부 확인 후 필요 시 즉시 재발급
- 제출 오류 발생 시 이름 오탈자/중복 제출 여부 먼저 확인

## 7) 장애 대응 체크

- "유효하지 않은 링크"
  - URL에 `exp`, `sig` 존재 여부 확인
  - 재발급 후 최신 링크 재배포

- "링크 만료"
  - 관리자 화면에서 재발급 후 재전달

- "이미 제출된 이름"
  - 동일인 재제출로 판단
  - 관리자에서 세션/대상자 확인 후 필요 시 신규 세션 발급

## 8) 관리자 보안 운영 수칙

- 관리자 PIN은 주기적으로 변경
- 공용 PC 사용 시 업무 종료 즉시 `관리자 잠금`
- 브라우저 자동완성/비밀번호 저장 비활성 권장
- 실운영 도메인 외 테스트 링크 공유 금지

## 9) 2026-03-24 운영 반영 사항

- 근로자 상세 리포트 운영 원칙
  - 외국인: 모국어 우선 표기 + 한국어 [KO] 병기
  - 한국인: 한국어 단일 표기
- 코칭/채점근거에서 모국어가 비어 있는 과거 데이터는 재분석으로 보완
- 외국인 리포트는 핵심 텍스트 가독성 확보를 위해 수기원본 영역을 기본 비노출
