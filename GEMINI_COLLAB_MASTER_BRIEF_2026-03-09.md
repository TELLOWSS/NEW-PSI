- 작성일: 2026-03-09
- 기준 버전: PSI v2.1.0
- 목적: 현재까지 업그레이드 사항을 한 번에 정리하고, Gemini와 향후 추가/수정보강 사항을 체계적으로 논의하기 위한 기준 문서
- 최근 반영(이번 세션): 피드백 Webhook 실전송, 실패 Outbox 보관 및 재전송 UI, 보고서 진행률 퍼센트 UI, 설정 연동 필드 추가

---

## 1) 프로그램 개요

PSI(Proactive Safety Intelligence)는 건설 현장 안전 데이터를 수집·분석·예측·보고하는 통합 안전 AI 시스템입니다. OCR 기반 기록지 디지털화, 근로자 위험 분석, 예측 관리, 보고서 자동 생성, 운영 정책 관리까지 하나의 워크플로우로 연결합니다.

핵심 가치:
- 선제적 안전(사고 이후 대응이 아닌 사전 예방)
- 현장 실사용성(모바일/대규모 인원/다국어 환경 대응)
- 공신력(특허출원 제10-2026-0039151호 기반 신뢰 체계)

---

## 2) 현재까지 업그레이드 핵심 요약

### A. 안정성/품질 (v2.1 중심)
- 300+ 근로자 처리 안정성 강화 (Progressive Rendering, 메모리 여유 확보)
- 무한 재시도/무한 대기 위험 제거 (재시도 상한, 타임아웃 적용)
- 보고서 생성 실패 추적 강화 (성공/실패 건수 및 실패 대상 표시)
- Null 참조 방어 강화 (PDF 생성/저장 시 안전 체크)
- 할당량(Quota) 대응 로직 고도화 및 백오프 정책 개선

### B. 기능/운영
- 소개/피드백 탭에 버전 업그레이드 히스토리 반영
- 근로자 안전모 스티커/스마트 사원증 대량 발급 시나리오 검증
- 보고서 필터(공종/등급/복합 조건) 동작 검증
- 무결성/심리 분석 기반 보조 지표 체계 보강

### C. 모바일/반응형
- 모바일 네비게이션(햄버거, 오버레이, ESC 닫기) 구축
- Sidebar/대시보드/설정/검수 모달의 반응형 입력/버튼 UX 개선
- 터치 타겟/가독성/레이아웃 스택 구조 최적화

### D. 이번 세션 구현 완료 (2026-03-09)
- 피드백 실전송 모드 구현: Settings의 Webhook URL 설정 시 POST 전송
- 전송 실패 안전장치: 실패 건 자동 Outbox 저장(`psi_feedback_outbox`, 최대 100건)
- Outbox 운영 기능 구현: 단건 재전송, 전체 재전송, 항목 삭제/전체 비우기 UI
- 피드백 전송 채널 설정 추가: Webhook URL / Timeout(ms) / 메타데이터 포함 여부
- 보고서 생성 UX 보강: 진행률 퍼센트 + 프로그레스 바 + 처리 건수 표시
- 소개/피드백 협업 문구 보강: Gemini 논의 카테고리와 협업 아젠다 가시화

## 3) 카테고리(페이지)별 기능 정리

- 대시보드: 현장 핵심 지표 요약, 카드형 KPI, 추세 시각화
- OCR 분석: 이미지/문서 기반 텍스트 추출 및 분석 파이프라인
- 근로자 관리: 근로자 정보 관리, 스티커/ID 발급, 대량 처리 최적화
- 예측적 안전 관리: 위험 예측 관점의 선제 대응 지원
- 성과 추이 분석: 성과/안전 등급 변화 추세 확인
- 안전 이행 점검: 점검 항목 관리 및 실행 상태 확인
- 현장 지적사항 관리: 이슈 관리/후속 조치 추적
- 보고서: 필터링 기반 개별/통합 리포트 생성, 실패 추적
- 시스템 설정: 현장 설정, 가중치 정책, 이력/복원, 승인정책
- 피드백: 현장/운영/디자인/버그/Gemini 협업 제안 접수
- 소개: 시스템 철학, 신뢰 근거(특허), 버전 진화 히스토리, 향후 협업 아젠다

## 4) 디자인 전체 기준 (현재 적용 상태)

### 톤앤매너
- 기본 팔레트: Slate + Indigo 중심, 강조색(emerald/amber/rose) 목적형 사용
- 카드 중심 UI: 큰 라운드(2xl~[48px]) + 부드러운 그림자 + 경계선
- 상태 전달: 배지/타임라인/아이콘/컬러를 통한 즉시 인지 구조

### 주요 디자인 패턴
- Hero 섹션: 그라디언트/블러/라이트 글로우로 메시지 강조
- 타임라인: 버전 이력 및 개선사항 스토리텔링
- 정보 카드: 검증 결과/개선사항/가이드의 블록형 분리
- 모달/폼: 모바일 세로 스택, 액션 버튼 전체폭 우선
- 특허/신뢰 배지: 여러 화면에서 일관 표시

### 반응형 원칙
- Mobile-First + Tailwind 브레이크포인트(sm/md/lg) 기준
- 작은 화면에서 입력/버튼/카드 가독성 우선
- 데스크톱에서는 정보 밀도 확장, 모바일에서는 동선 최소화

## 6) Gemini와 논의할 추가/수정보강 항목 (우선순위)

### P0 (즉시)
1. 피드백 접수 실서버 연동 고도화 (완료: Webhook 기본 연동 / 논의: Slack·Email 다중 채널, 인증, 재전송 정책)
2. 보고서 생성 진행률/실패 재시도 UX 고도화 (완료: 퍼센트·진행바 / 논의: 실패 건별 재시도 버튼, 백그라운드 큐)
3. OCR 실패 원인 분류(품질/해상도/언어/레이아웃) 자동 태깅

### P1 (단기)
1. 저무결성/고위험 근로자 자동 후속조치 템플릿
2. 현장별 KPI 비교 대시보드(월/주 단위)
3. 설정 변경 승인 플로우(권한·감사 로그) 강화
4. 모바일 OCR/근로자관리 페이지 추가 최적화

### P2 (중기)
1. E2E 자동화 테스트 시나리오 구축
2. 번들 최적화(코드 스플릿, 지연 로딩)
3. 다국어/현장 특화 용어 사전 기반 번역 정확도 개선
4. PWA(오프라인) 운영성 강화

## 7) 피드백 카테고리 운영 기준 (개편안 반영)

현재 피드백 유형은 다음과 같이 목적별로 분리 운영:
- 긴급: 현장 위험 요소 즉시 제보
- 품질: 시스템 버그 / 번역·OCR 오류
- 기능: 현장 맞춤 기능 제안
- Gemini 협업: 다음 버전 기획/개선
- UX/디자인: 모바일 최적화 / 가독성·표기 개선
- 공신력: 특허·법무·권리화 문의
- 운영: 우수 사례 공유

운영 원칙:
- 긴급/버그는 우선 triage
- 디자인/UX는 월간 리뷰로 묶음 개선
- Gemini 협업 항목은 문서 기반 의사결정 로그로 관리
