# 도면 설명 및 도면부호(출원용)

## 도 1. 시스템 블록도
- 100: 전체 시스템
- 110: 이미지 입력 모듈
- 120: OCR 모듈
- 130: 정규화 모듈
- 140: 개인 프로파일링 모듈
- 150: 안전등급 산정 모듈
- 160: 관리자 대시보드 모듈
- 170: 증빙 저장 모듈
- 180: 보호구 표지 연동 모듈
- 190: 리포트 생성 모듈

도 1은 수기 위험성평가 기록지로부터 개인 안전등급 산정 및 증빙 생성까지의 기능 블록 연계를 나타낸다.

## 도 2. 방법 흐름도
- S110: 기록지 이미지 수집
- S120: OCR 및 항목 구조화
- S130: 근로자 식별 매칭
- S140: 개인 안전역량 산정
- S150: 안전등급 부여
- S160: 관리자 경보/개선 권고
- S170: 조치·재평가 이력 저장
- S180: 증빙 리포트 생성

도 2는 시스템의 기본 처리 순서를 도시하며, 구현에 따라 일부 단계 순서는 변경될 수 있다.

## 도 3. 안전등급 산정 상세도
- 210: 지표 수집부
- 220: 가중치 적용부
- 230: 점수 계산부
- 240: 등급 판정부
- 250: 변동 사유 기록부
- 260: 표지 갱신 트리거부

도 3은 심리 지표, 업무 이해도, 위험성평가 이해도, 숙련도 및 개선이행도 등을 결합해 안전등급을 산정하는 내부 로직을 나타낸다.

## 도 4. 증빙 패키지 구조도
- 310: 원본 문서 저장부
- 320: OCR 결과 저장부
- 330: 정정/검증 이력 저장부
- 340: 조치/교육 이력 저장부
- 350: 승인/검토 이력 저장부
- 360: 무결성 검증값 저장부
- 370: 검색/집계 인덱스부

도 4는 감사 추적 가능한 증빙 패키지의 데이터 구성요소를 나타낸다.

## 도면 작성 가이드
1) 블록도는 기능 중심으로 단순화하고 UI 세부 화면은 제외한다.
2) 각 도면부호는 명세서 본문에서 동일 번호로 일관되게 사용한다.
3) 박스 간 화살표는 데이터 흐름(입력→처리→저장→출력)을 명확히 표기한다.
4) 도 2의 단계명(S110~S180)은 청구항 단계와 용어를 동일하게 유지한다.
