> 적용일: 2026-03-02

## 1. 관점 전환 원칙

### 1.1 코드 레벨 디테일의 아키텍처화
- 개발 문서의 표현(예: 무한 루프 방지, Null 참조 방지, GC 대기시간 조정)은 그대로 청구항 요소가 되기 어렵다.
- 특허 문서에서는 이를 아래와 같이 상위 개념으로 치환한다.
  - 무한 루프 방지 → 제어흐름 한정(재시도 횟수/대기시간/종료조건 포함)
  - Null 참조 방지 → 입력 유효성 검증 단계 및 예외 격리 단계
  - GC 대기시간 조정 → 처리 안정화 대기 제어 또는 자원 해제 제어 단계
- 핵심은 특정 언어/프레임워크가 아닌, 데이터 처리의 안정성을 보장하는 시스템 구성과 단계적 방법을 기술하는 것이다.

### 1.2 UI/UX 개선의 권리화 범위 정리
- 패딩, 폰트, 햄버거 메뉴, 반응형 브레이크포인트 등은 원칙적으로 청구항 핵심으로 두지 않는다.
- 해당 요소는 필요 시 명세서의 실시예(구체적 구현 예) 또는 사용자 인터페이스 보조 설명으로만 간략 기재한다.
- 청구항은 기능적 효과를 내는 처리 단계/모듈 간 상호작용 중심으로 구성한다.

## 2. 문장 치환 규칙 (개발표현 → 특허표현)

| 개발/QA 표현 | 특허 명세서 권장 표현 |
|---|---|
| API 무한 재시도 방지 | 재시도 한계값과 종료 조건을 포함하는 제어단계 |
| Null 체크 추가 | 입력 데이터 유효성 검증 및 예외 처리 분기 단계 |
| setTimeout(500)으로 안정화 | 처리 안정화를 위한 대기 제어 단계 |
| 컴포넌트 렌더링 최적화 | 대량 데이터의 단계적 처리/출력 제어 모듈 |
| 모바일 UI 개선 | 단말 환경별 표시 변형이 가능한 사용자 출력부 |

## 3. 필수 파이프라인(방법 청구항 대응)

특허 관점에서 최소한 다음 흐름이 도면 및 본문에 일관되게 나타나야 한다.

1) 수기 기록지 이미지 업로드(수신)
2) OCR 텍스트화 및 항목 구조화
3) 사용자 정정(검수) 입력
4) 정정 반영 AI 재분석
5) 분석 결과 및 정정 이력 저장

상기 1)~5)는 도 2(방법 흐름도), 명세서 본문(S단계), 청구항 독립항에 동일 용어로 반영한다.

## 4. 청구항 재작성 틀

### 4.1 방법 독립항(골격)
- A 데이터를 수신하는 단계
- 수신 데이터에 대하여 B 알고리즘으로 구조화/분석하는 단계
- 분석 결과에 대한 사용자 정정 입력을 수신하는 단계
- 정정 입력을 반영하여 재분석하는 단계
- 재분석 결과와 이력 데이터를 저장하고 C 결과를 출력하는 단계

### 4.2 시스템 독립항(골격)
- 수신부, 분석부, 정정부 입력부, 재분석부, 저장부, 출력부를 포함하고,
- 상기 구성요소들이 방법 청구항의 각 단계를 수행하도록 결합되는 시스템.

### 4.3 종속항 강화 포인트
- OCR 신뢰도 임계치 및 검증 대기열
- 가중치 세트 버전관리 및 복원 가능성
- 승인권자 역할 기반 차단/허용 정책
- 정정 전/후값 저장 및 증빙 패키지 해시 무결성

## 5. 도면 작성 기준

### 5.1 블록도(도 1)
- 입력부 → OCR/분석부 → 정정/재분석부 → 저장부 → 출력부의 기능 블록 연결
- UI 화면 캡처가 아닌 기능 블록과 데이터 흐름 중심으로 도시

### 5.2 순서도(도 2)
- 결정 노드(신뢰도 임계치, 개입 필요 여부)를 포함
- "사용자 정정 → AI 재분석" 루프를 명시
- 단계 번호(S110~)와 명세서 단계명 일치

## 6. 제출 전 점검 체크
- [ ] 청구항 본문에서 구현 언어/프레임워크 고유 용어를 제거했는가
- [ ] 독립항이 "수신-분석-정정-재분석-저장/출력" 구조를 포함하는가
- [ ] 도면 단계명과 명세서 본문 단계명이 일치하는가
- [ ] UI/UX 내용이 권리범위가 아닌 실시예 수준으로 정리되었는가
- [ ] 기술효과가 안정성·재현성·입증성 관점으로 기술되었는가

## 7. 적용 범위
- 본 가이드는 본 저장소의 개발·검증 보고서(`FEATURE_UPDATE_REPORT.md`, `IMPROVEMENTS_SUMMARY.md`, `VERIFICATION.md` 등)를 특허 명세서 체계로 변환할 때 공통 적용한다.
- 최종 제출본은 변리사 검토 결과를 우선한다.
