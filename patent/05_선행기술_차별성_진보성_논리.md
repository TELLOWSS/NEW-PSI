# 선행기술 차별성/진보성 논리서

## 0. 문서 관리 정보
- 발명 및 개발 총괄: 박성훈
- 검토 완료일: 2026-04-09
- 시스템 적용 버전: PSI v2.2.0
- 상태: ✅ 브랜드·권리화 보정 반영 완료
- 최종본 잠금 태그: PSI-v2.2.0-2026-04-09-AMENDED

## 1. 예상 선행기술 축
- A축: 문서 OCR 기반 디지털 전환 기술
- B축: 산업안전 점검/교육 관리 시스템
- C축: 근로자 등급 또는 배지 표시 체계
- D축: 사용자 맞춤형 대시보드 또는 역할 기반 UI 표현 기술

## 2. 본 발명의 결합 차별점
1) 수기 위험성평가 OCR 결과를 단순 문서 저장이 아니라 개인 안전역량 지표로 변환
2) 심리 지표 + 업무 이해도 + 위험성평가 이해도 + 숙련도를 결합한 개인 단위 프로파일링
3) 동일 분석 결과로부터 의미 상태를 생성하고, 이를 근로자·관리자·경영진 역할별 상이한 표현 결과로 변환
4) 관리자 개입 우선순위 제공과 법규 대응 증빙 패키지 생성을 하나의 처리 체인으로 통합
5) 안전모/안전벨트 표지와 연동해 현장 행동 변화를 유도하는 폐루프(Feedback Loop) 구현

## 3. 진보성 주장 포인트
- 기존 OCR 시스템은 인식/저장 중심이며 개인 안전역량 산정까지 직접 연결되지 않음
- 기존 안전관리 시스템은 교육/점검 관리 중심이며 수기 위험성평가 문서의 고신뢰 구조화 및 증빙 패키지 자동화가 약함
- 기존 배지 시스템은 가시화 기능에 머물고, 분석-개입-재평가의 데이터 루프와 결합된 사례가 제한적임
- 기존 역할 기반 UI 기술은 주로 접근 권한 또는 메뉴 노출 차이에 머물고, 동일 분석 결과를 안전 목적의 의미 상태로 변환한 뒤 역할별 언어정책에 따라 상이하게 재구성하고 이를 증빙 이력으로 남기는 구조와는 구별됨
- 본 발명은 안전역량 산정 가중치(w1~w6)의 버전 관리와 이력 복원 기능을 포함하여, 모델 변경의 감사가능성과 재현가능성을 기술적으로 확보함
- 본 발명은 승인권자 역할(예: 안전관리자/현장소장)에 따른 승인 차단 규칙 분기 및 정책 강제를 지원하여, 동일 데이터라도 운영 정책에 따라 통제 가능한 구조를 제공함
- 본 발명은 정정 이력에서 필드별 정정 전/후 값을 구조적으로 저장하고, 증빙 패키지 해시를 생성하여 무결성 검증이 가능한 체인을 구성함
- 본 발명은 메시지 정책 버전 또는 역할별 결과 표현을 저장하여, 동일 입력 데이터가 어떤 정책에 따라 어떠한 코칭형 또는 실무형 표현으로 제공되었는지까지 재현 가능한 구조를 제공함

## 3-1. 역할 적응형 표현 정책의 차별 논리
1) 단순 시각화 변경이 아니라, 분석 결과를 먼저 의미 상태로 구조화한다.
2) 상기 의미 상태를 역할별 용어 사전과 정책 버전에 따라 다른 문장군으로 변환한다.
3) 변환 결과 또는 정책 버전을 증빙 체인에 포함해 사후 감사 가능성을 확보한다.

따라서 본 발명은 단순한 UI 카피 변경이나 사용자 맞춤형 화면 구성과 달리, 안전 판단 결과의 표현 자체를 기술적 처리 대상으로 삼고 있다는 점에서 차별된다.

## 4. 효과 입증 지표(심사 대응용)
- OCR 인식 정확도(문자/항목 단위)
- 관리자 개입 리드타임 단축률
- 반복 위반 항목 감소율
- 교육 후 등급 개선률
- 법규 대응 문서 준비 시간 단축률
- 역할별 표현 수용도 또는 재수정률 감소율
- 동일 정책 버전 기준 표현 재현율

## 5. 반박 대비 문장 템플릿
- 단순 병렬 결합이 아니라, 개인 안전역량 산정 결과가 보호구 표지 및 재평가 정책에 직접 반영되는 기술적 연계가 존재한다.
- 증빙 패키지는 원본-정정-조치-승인-재평가를 단일 체인으로 관리하여 데이터 무결성과 감사가능성을 기술적으로 보장한다.
- 가중치 버전관리, 승인정책 분기, 해시 기반 무결성 검증은 단순 UI/업무 절차의 조합이 아니라 분석결과의 재현성과 법규 대응 신뢰도를 높이는 기능적 결합이다.
- 역할 적응형 표현 정책은 단순 사용자별 문구 선택이 아니라, 동일 분석 결과를 의미 상태로 변환한 후 역할별 언어 정책을 적용하고 그 결과를 증빙 체인에 연결하는 기술적 처리 구조이다.

## 6. 구현 반영 정합성 메모(2026-04-09)
- 안전역량 산식(P = w1·심리지표 + w2·업무이해도 + w3·위험성평가이해도 + w4·숙련도 + w5·개선이행도 - w6·반복위반도)은 설정 기반 가중치와 버전값으로 운영됨
- 승인 단계에서 역할 기반 차단 규칙이 적용되며, 정책에 따라 안전관리자 엄격 검증 모드를 강제 가능함
- 정정 이력은 정정 사유뿐 아니라 필드별 전/후값을 함께 저장하여 감사 추적성을 강화함
- 증빙 패키지 생성 시 해시 무결성 값을 포함하여 사후 검증 가능성을 확보함
- 의미 상태(예: 보완 권고, 추가 확인, 우선 검토, 확정 가능)를 생성한 뒤 역할별 결과 표현으로 변환하는 구조가 명세서/청구항/도면에 반영됨
- 메시지 정책 버전 또는 역할별 결과 표현을 증빙 저장 범위에 포함하는 구조가 명세서/청구항/도면4에 반영됨
