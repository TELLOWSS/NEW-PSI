# PSI OCR·모바일 현장모드 상품화 최종 계획

작성일: 2026-06-29  
대상: NEW-PSI 위험성평가 수기 기록지 OCR, 근로자 수준 분석, 교육·추적관리 폐루프

## 1. 결론

현재 프로그램은 상품화 가능한 뼈대가 이미 있다. 특히 OCR 결과 구조화, 6대 지표 채점, 모국어 코칭, 모바일 화면 증빙, 점수 일관성 게이트가 존재한다.  
다만 상품화 직전 기준으로는 "실제 수기 기록지 판독률"과 "양식 변화 내성"을 숫자로 고정하는 벤치마크가 아직 부족하다.

따라서 베스트 오브 베스트 전략은 다음이다.

1. Gemini 기반 문서 이해를 유지하되, 수기 OCR 신뢰도 계층을 분리한다.
2. 모든 양식은 "양식 버전/문항 슬롯/답변 좌표/의미 매핑"으로 정규화한다.
3. 모바일은 별도 "현장 스캔 모드"로 독립시킨다.
4. 분석 결과는 즉시 교육 대상, 재촬영 대상, 관리자 검수 대상으로 자동 분기한다.
5. 제품 합격 기준은 감이 아니라 판독률, 필드 정확도, 점수 편차, 재촬영률, 교육완료율로 잠근다.

## 3. 현재 프로그램의 강점

현재 코드에는 이미 상품화에 필요한 핵심 부품이 있다.

- `services/geminiService.ts`
  - PDF/이미지 분석
  - Gemini 모델 폴백
  - 6대 지표 점수 산정
  - 모국어 번역/코칭
  - 우수사례 RAG 기반 코칭 보정
- `api/gateway.ts`
  - 서버 OCR 재시도
  - 이미지 용량 제한
  - OCR 실패 코드 분리
  - 구조 검증 후 결과 반환
- `utils/ocrVerificationLanguageUtils.ts`
  - 문항별 원문 답변, 한국어 해석, 모국어 해석 누락 검사
  - 점수 과대 의심 검사
- `pages/OcrAnalysis.tsx`
  - 실패 유형, 재시도, 상세검수, 원본 대조 UI
- `components/FileUpload.tsx`
  - 이미지/PDF 업로드
  - 카메라 촬영
  - 지원 파일 형식 제한

## 4. 남은 핵심 리스크

### R1. 실제 수기 판독률 미검증

현재 검증은 코드·계약·화면·점수 일관성 중심이다. 실제 근로자가 쓴 이미지 100~300장 기준의 판독률 리포트가 필요하다.

합격 기준:

- 문항 감지율: 98% 이상
- Q1~Q5 답변 슬롯 추출률: 95% 이상
- 이름/공종/날짜/국적 추출 정확도: 95% 이상
- 답변 의미 해석 정확도: 90% 이상
- 재촬영 필요 판정의 정밀도: 90% 이상

### R2. 양식 변화 내성

현재 프롬프트는 문항형 위험성평가표를 읽도록 설계되어 있으나, "기존 양식"과 "다듬어진 양식"을 같은 맥락으로 묶는 명시적 양식 버전 레이어가 더 필요하다.

필수 보강:

- `formTemplateId`
- `formVersion`
- `questionSlots`
- `answerRegions`
- `fieldAliases`
- `canonicalQuestionId`: Q1_TASK, Q2_HAZARD, Q3_LEVEL, Q4_CONTROL, Q5_COMMITMENT

### R3. OCR 신뢰도와 분석 신뢰도가 섞임

현재 `ocrConfidence`는 존재하지만, 상품화 기준에서는 아래를 분리해야 한다.

- 촬영품질 점수
- 양식검출 점수
- 글자판독 점수
- 문항매핑 점수
- 번역 신뢰도
- 안전판정 신뢰도

### R4. 모바일 현장 모드가 아직 "OCR 화면의 일부"

현재 카메라 촬영은 가능하지만, 현장 사용자는 업로드 화면이 아니라 "찍고, 품질 확인하고, 분석하고, 교육 연결"까지 한 흐름으로 끝나야 한다.

11. 교육 추천 및 추적관리 연결

### 5-2. 양식 호환 계층

모든 양식은 화면상 모양이 달라도 아래 공통 구조로 변환한다.

```json
{
  "formTemplateId": "psi-risk-assessment",
  "formVersion": "2026-06-mother-tongue-v2",
  "worker": {},
  "answers": {
    "Q1_TASK": "",
    "Q2_HAZARD": "",
    "Q3_LEVEL": "",
    "Q4_CONTROL": "",
    "Q5_COMMITMENT": ""
  },
  "evidence": {
    "imageQuality": 0,
    "layoutConfidence": 0,
    "ocrConfidence": 0,
    "fieldConfidence": {}
  }
}
```

이 구조가 있으면 기존 양식, 개선 양식, 향후 다른 나라 양식이 와도 같은 품질의 분석자료로 정규화할 수 있다.

## 6. 모바일 현장 스캔 모드

별도 페이지 또는 모드명: `현장 스캔`

첫 화면:

- 큰 버튼 1개: `기록지 촬영`
- 보조 버튼: `PDF/이미지 선택`
- 상태 3개: 오늘 분석, 재촬영 필요, 교육 필요

촬영 후:

- 문서 모서리 자동 가이드
- 품질 판정: 좋음 / 다시 촬영 권장 / 사용 불가
- 바로 분석
- 결과 3분기:
  - 정상: `결과 보기`
  - 낮은 수준: `교육 연결`
  - 판독 불량: `다시 촬영`

분석 완료 후:

- 근로자 현재 수준
- 약점 1~3개
- 오늘 바로 교육할 항목
- 해당 언어 안내문
- 관리자 확인/교육 발송/재평가 예약

## 7. 상품화 합격 게이트

### Gate A. OCR 품질

- 실제 수기 300장
- 11개 언어 각 20장 이상
- 촬영 조건 5종: 정상, 흔들림, 그림자, 사선, 낮은 조도
- 합격: 전체 자동 처리 성공률 90% 이상, 관리자 검수 포함 실사용 성공률 97% 이상

### Gate B. 양식 변화 내성

- 기존 양식 50장
- 개선 양식 50장
- 일부 잘림/회전/스캔 PDF 50장
- 합격: Q1~Q5 공통 매핑 오류 3% 이하

### Gate C. 점수 일관성

- 같은 답변을 다른 양식/다른 촬영조건에서 분석
- 합격: 점수 편차 ±8점 이내 95% 이상

### Gate D. 교육 폐루프

- 위험 발견 → 교육 추천 → 근로자 확인 → 서명 → 재평가
- 합격: 같은 `case_id`로 전 과정 추적 가능

### Gate E. 모바일 현장성

- 360px Android, 390px iPhone 기준
- 촬영부터 결과까지 60초 이내
- 실패 시 재촬영 안내 1탭 이내

## 9. 최종 판단

상품화 방향은 맞다.  
현재 프로그램의 가장 큰 가치는 "문서를 읽는 OCR"이 아니라 "근로자의 위험 인식 수준을 읽고, 즉시 교육과 추적관리로 이어지는 폐루프"다.

따라서 제품 포지셔닝은 다음이 가장 강하다.

> NEW-PSI는 수기 위험성평가 기록지를 현장에서 촬영하면, 근로자의 위험 인식 수준과 교육 필요 항목을 즉시 판정하고, 교육·서명·재평가까지 연결하는 건설현장 안전역량 관리 시스템이다.

이 방향으로 가면 단순 OCR 프로그램이 아니라 현장 안전관리의 운영 엔진이 된다.
