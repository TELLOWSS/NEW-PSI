# PSI 실사용 조작성 검증·보강 계획

작성일: 2026-07-10  
관점: 현장 관리자, 안전담당자, 바이어가 처음 프로그램을 열었을 때 실제로 편하게 조작할 수 있는가

## 1. 결론

현재 PSI는 기능 구성 자체는 충분하다. OCR 분석, 관리자 검증, 개인 보호 리포트, 원페이지 교육자료, 월별 추적자료가 이미 연결되어 있다.

다만 실사용자 입장에서는 아직 다음 문제가 남아 있다.

1. 시작점이 많아 보인다.
2. 메뉴가 현장용보다 개발자용에 가깝게 많이 보인다.
3. `리포트`, `교육자료`, `추적자료`가 각각 따로 있는 기능처럼 보일 수 있다.
4. 관리자 검증 후 “다음에 무엇을 해야 하는지”가 더 강하게 안내되어야 한다.
5. 모바일에서는 핵심 실행 버튼이 있으나, 전체 흐름이 `찍는다 → 확인한다 → 교육한다`로 한 화면에서 완전히 닫히지는 않는다.

따라서 보강 방향은 기능 추가가 아니라 **실사용 동선 단순화**다.

핵심 원칙은 다음과 같다.

**한 화면에는 다음 행동 하나만 강하게 보여준다.**

## 2. 현재 프로그램 확인 결과

### 이미 잘 되어 있는 부분

| 항목 | 확인 결과 | 판단 |
|---|---|---|
| 교육 환류 센터 | 원페이지 교육자료, 개인 보호 리포트, 월별 계도·추적자료 3개 출력물 구조 존재 | 좋음 |
| 2분 원포인트 증명 모드 | 기능 화면 확인 후 다시 증명모드로 돌아오는 배너 존재 | 좋음 |
| 모바일 현장 모드 | OCR 화면에 촬영, 리포트, 추적 빠른 버튼 존재 | 좋음 |
| 보호 중심 표현 | 점수/등급을 보호 신호로 바꾸는 유틸 존재 | 좋음 |
| 메뉴 조정 기능 | 사이드바 표시/숨김과 순서 조정 구조 존재 | 좋음 |

### 불편해질 수 있는 부분

| 항목 | 현재 상태 | 실사용 위험 |
|---|---|---|
| 실무 즉시 모드 메뉴 | `IMMEDIATE_VISIBLE_PAGES`에 매우 많은 페이지가 포함됨 | 처음 쓰는 사람이 무엇부터 해야 할지 모를 수 있음 |
| 사이드바 기본 순서 | 교육 환류가 중간, OCR 분석이 뒤쪽에 위치 | 핵심 흐름과 메뉴 순서가 다르게 느껴질 수 있음 |
| 원포인트 증명 4단계 | `모국어·리포트·추적`, 기본 페이지 `reports` | 환류 구조보다 리포트 기능처럼 보일 수 있음 |
| 교육 환류 카드 | 세 가지 결과물은 명확하지만 검증완료 기준이 더 강하게 보이면 좋음 | AI 자동자료처럼 오해될 수 있음 |
| OCR 화면 | 재분석, 백업, 진단 정보가 많음 | 현장 사용자는 긴급 버튼만 원함 |

## 3. 실사용 기준 사용자 여정

현장 관리자가 실제로 원하는 흐름은 다음 6단계다.

1. **오늘 기록지를 찍는다**
   - 사진/PDF 업로드
   - 분석 시작

2. **AI가 읽은 내용을 확인한다**
   - Q1~Q5 추출 확인
   - 공종, 이름, 날짜, 위험요인 확인

3. **관리자가 검증한다**
   - 틀린 부분 수정
   - 보호 해석 확인
   - 검증완료 처리

4. **검증 결과가 환류된다**
   - 개인 보호 리포트 확인
   - 교육 환류 후보 확인

5. **다음 달 교육자료로 만든다**
   - 원페이지 교육자료 생성
   - 다국어 안내 필요 시 선택

6. **월별로 추적한다**
   - 개인별, 팀별, 공종별, 월별 변화 확인
   - 반복 위험과 개선 흐름 확인

이 여정을 화면에서는 더 짧게 보여줘야 한다.

**찍는다 → 확인한다 → 환류한다 → 교육한다 → 추적한다**

대외/영상용 단순 표현은 계속 다음처럼 유지해도 된다.

**찍는다 → 확인한다 → 교육한다**

## 4. 보강 목표

### 목표 1. 처음 보는 사람도 10초 안에 시작 버튼을 찾는다

기준:
- 첫 화면에서 `기록지 촬영/업로드` 버튼이 눈에 띄어야 한다.
- OCR 분석 화면으로 가는 길이 메뉴 탐색 없이 보여야 한다.

권장:
- 홈/교육 환류 센터 상단에 `기록지 촬영/업로드` 주 버튼 배치
- 보조 버튼은 `검증 필요 기록`, `교육 환류`, `월별 추적`만 노출

### 목표 2. 관리자 검증 후 다음 행동이 자동으로 보인다

기준:
- 검증완료 버튼을 누른 뒤 사용자가 다음 화면을 찾지 않아도 된다.

권장:
- 검증완료 후 하단 또는 팝업에 세 버튼 표시
  - `보호 리포트 확인`
  - `교육 환류로 보내기`
  - `월별 추적에 반영`

최종 추천:
- 기본 주 버튼은 `교육 환류로 이동`
- 보조 버튼으로 `리포트 확인`, `계속 검증`

### 목표 3. 메뉴는 현장용과 개발자용을 확실히 분리한다

현재 실무 즉시 모드도 메뉴가 많다. 상품화 화면에서는 다음 정도가 적절하다.

필수 메뉴:
1. 현장 안전 관제센터
2. 위험성평가 분석
3. 교육 환류
4. 월별 추적자료
5. 시스템 설정

교육 환류 내부 연결:
- 원페이지 교육자료
- 개인 보호 리포트
- QR/음성 파일럿

개발자/관리 확장 메뉴:
- 선행 위험신호 분석
- 안전성과 분석
- 근로자 안전 프로파일
- 현장 위험 이슈 관리
- 재분석 상세
- 백업/검증 패키지

### 목표 4. 모바일은 세 버튼으로 끝낸다

모바일 현장 모드는 다음 세 버튼이 핵심이다.

1. `찍기`
2. `확인`
3. `환류`

세부 동작:
- `찍기`: 카메라/파일 업로드
- `확인`: 확인 필요 기록 또는 방금 분석한 기록 상세
- `환류`: 교육 환류 센터 또는 월별 추적 요약

모바일에서 숨겨야 할 것:
- 대량 재분석
- 백업 내보내기
- 개발자 진단
- API/엔진 세부명
- 긴 설명문

## 5. 화면별 보강 계획

### 5.1 교육 환류 센터

현재 장점:
- PSI의 본질을 가장 잘 보여주는 화면이다.
- 원페이지 교육자료, 개인 보호 리포트, 월별 계도·추적자료 구조가 이미 있다.

보강:
- 상단 문구 수정
  - 현재: `검증된 기록은 세 가지 결과물로 돌아갑니다.`
  - 권장: `관리자가 확인한 기록은 개인 보호 리포트, 다음 달 원페이지 교육자료, 월별 추적자료로 환류됩니다.`

- 3단계 흐름 수정
  - 현재: `찍는다 → 확인한다 → 교육한다`
  - 화면 내부 보강: `찍는다 → 검증한다 → 환류한다`
  - 하단 슬로건: `찍는다 → 확인한다 → 교육한다` 유지 가능

- 카드 버튼 문구 조정
  - `교육자료 만들기`
  - `보호 리포트 확인`
  - `월별 추적 보기`

### 5.2 2분 원포인트 증명 모드

현재 장점:
- 기능 화면을 확인하고 다시 증명모드로 돌아오는 배너가 있다.

보강:
- 4단계 기본 페이지를 `reports`에서 `education-return`으로 변경
- 4단계 제목을 `모국어·리포트·추적`에서 `검증 결과 환류`로 변경
- 빠른 버튼을 다음 4개로 정리
  - `OCR 시연`
  - `검증 결과 환류`
  - `교육자료`
  - `월별 추적`

완료 기준:
- 시연자가 어느 화면에 들어갔다 와도 “다음에 무엇을 눌러야 하는지”가 원포인트 증명 패널에 표시되어야 한다.

### 5.3 OCR 분석 화면

현재 장점:
- 모바일 현장 모드와 빠른 실행 버튼이 있다.
- 오류/재분석/검증 흐름이 매우 상세하다.

실사용 위험:
- 정보가 많아 처음 쓰는 사용자는 대량 재분석, 보조작업, 백업 버튼을 먼저 볼 수 있다.

보강:
- 상품화/현장 모드에서는 상단을 3영역으로 단순화
  1. `새 기록지 분석`
  2. `확인 필요`
  3. `검증완료/환류 가능`

- 재분석, 백업, 진단은 접힘 영역으로 이동
- 방금 업로드한 기록은 상단에 `방금 분석한 기록`으로 고정
- 실패 시 메시지는 기술 용어 대신 행동으로 표시
  - `다시 촬영`
  - `관리자 확인`
  - `재분석 대기`

### 5.4 관리자 상세 판단 화면

보강:
- 화면 하단에 고정 액션바 제공
  - `수정 저장`
  - `검증완료`
  - `환류로 이동`

- 검증완료 전 체크리스트
  - 이름/공종 확인
  - Q1~Q5 확인
  - 보호 해석 확인
  - 교육 환류 여부 확인

완료 기준:
- 관리자가 “수정했는데 어디를 눌러 끝내야 하지?”라고 느끼면 안 된다.

### 5.5 월별 추적 화면

보강:
- 필터 기본값을 `최근월 + 전체 공종`으로 고정
- 상단에 네 가지 탭 배치
  - 개인별
  - 팀별
  - 공종별
  - 월별

- 반복위반 설명 문구 추가
  - `반복위반은 현재 기록지 안의 반복 문장이 아니라 다음 달 동일 위험 재발 또는 약속 행동 미이행이 확인될 때 반영됩니다.`

## 6. 실사용 조작성 검증 시나리오

### 시나리오 A. 신규 기록 1장 분석

목표:
- 현장 관리자가 기록지 1장을 넣고 검증완료까지 갈 수 있는지 확인

측정:
- 시작 버튼 찾기: 10초 이내
- 업로드부터 분석 시작: 3클릭 이내
- 분석 결과 상세 확인: 1클릭
- 검증완료 처리: 2클릭 이내

합격 기준:
- 사용자가 메뉴를 헤매지 않는다.

### 시나리오 B. 확인 필요 기록 처리

목표:
- OCR 오류 또는 보정 필요 기록을 찾아 수정할 수 있는지 확인

측정:
- 확인 필요 목록 찾기: 10초 이내
- 수정 저장 후 검증완료: 3클릭 이내
- 검증완료 후 교육 환류 이동: 1클릭

합격 기준:
- 사용자가 재분석과 관리자 검증 중 무엇을 해야 하는지 이해한다.

### 시나리오 C. 다음 달 교육자료 만들기

목표:
- 검증완료 기록에서 원페이지 교육자료를 생성할 수 있는지 확인

측정:
- 교육 환류 센터 찾기: 10초 이내
- 기준월/공종 확인: 즉시 이해
- 교육자료 생성: 2클릭 이내
- 다국어 보기: 1클릭

합격 기준:
- 교육자료가 임의 작성물이 아니라 지난달 기록 기반임을 이해한다.

### 시나리오 D. 월별 추적 확인

목표:
- 개인별, 팀별, 공종별, 월별 흐름을 찾을 수 있는지 확인

측정:
- 월별 추적자료 진입: 10초 이내
- 공종별 위험 TOP3 확인: 즉시 이해
- 반복 위험 키워드 확인: 1클릭 이내

합격 기준:
- 추적관리가 단순 차트가 아니라 다음 교육/개선 근거임을 이해한다.

### 시나리오 E. 2분 시연

목표:
- 발표자가 설명 없이도 정해진 순서로 시연 가능해야 한다.

측정:
- OCR 시연 → 복귀 → 검증 결과 환류 → 복귀 → 교육자료 → 복귀 → 월별 추적 흐름이 끊기지 않아야 한다.
- 2분 내 완료 가능해야 한다.

합격 기준:
- 시연자가 메뉴를 찾는 시간이 없어야 한다.

## 7. 구현 우선순위

### 1순위. 화면 동선 보강

수정 대상:
- 2분 원포인트 증명 모드
- 교육 환류 센터 문구
- 4단계 기본 이동 화면

효과:
- 스토리보드와 실제 프로그램 동선이 일치한다.

### 2순위. 현장용 메뉴 단순화

수정 대상:
- 실무 즉시 모드 표시 메뉴
- 기본 사이드바 순서

권장 기본 순서:
1. 현장 안전 관제센터
2. 위험성평가 분석
3. 교육 환류
4. 월별 추적자료
5. 시스템 설정

효과:
- 처음 보는 사용자가 길을 잃지 않는다.

### 3순위. 관리자 검증 후 다음 행동 안내

수정 대상:
- 상세 판단/검증완료 영역
- 검증완료 후 CTA

권장 CTA:
- `교육 환류로 이동`
- `보호 리포트 확인`
- `계속 검증`

효과:
- 분석이 리포트에서 끝나지 않고 교육 환류로 이어진다.

### 4순위. 모바일 원터치 흐름 정리

수정 대상:
- OCR 모바일 현장 모드
- 모바일 하단 고정 버튼

권장 버튼:
- `찍기`
- `확인`
- `환류`

효과:
- 현장 사용성이 좋아진다.

### 5순위. 시연/상품화 모드 추가

수정 대상:
- 소개/교육 환류/원포인트 증명 화면

기능:
- 개인정보 숨김
- 파괴적 버튼 숨김
- 개발자 문구 숨김
- 데모 데이터 고정

효과:
- 공모전, 바이어 미팅, 기술영상 촬영 품질이 안정된다.

## 8. 추천 최종 구조

### 실사용 홈

상단:
- 오늘 할 일
  - 기록지 분석
  - 확인 필요
  - 교육 환류

중단:
- 찍는다 → 확인한다 → 교육한다

하단:
- 최근 분석 기록
- 이번 달 반복 위험 TOP3
- 다음 교육자료 생성 가능 상태

### 교육 환류 센터

상단:
- 검증완료 기록 수
- 확인 필요 기록 수
- 교육자료 생성 가능 상태

본문:
- 원페이지 교육자료
- 개인 보호 리포트
- 월별 계도·추적자료

하단:
- 익숙한 수기 작성 흐름을 유지한 채, 버려지던 종이를 예측 가능한 안전 데이터로 구조화합니다.

## 9. 최종 판단

PSI의 실사용 조작성 보강은 대규모 기능 추가보다 **사용자 여정 정리**가 핵심이다.

지금 필요한 것은 다음 세 가지다.

1. 핵심 메뉴를 줄인다.
2. 각 화면의 다음 행동을 하나로 고정한다.
3. 검증완료 기록이 교육 환류와 추적관리로 이어진다는 사실을 화면마다 반복해서 보여준다.

이 방향으로 정리하면 PSI는 개발자에게만 이해되는 복잡한 시스템이 아니라, 현장 관리자가 “기록지 찍고, 확인하고, 다음 달 교육자료로 쓰면 되는구나”라고 바로 이해하는 상품형 프로그램이 된다.
