# PSI 공모전·출시준비 심층리서치 검증계획

작성일: 2026-07-09

## 1. 검증 결론

현재 PSI는 “아이디어 단계”가 아니라 **현장 수기 위험성평가 기록을 OCR·AI로 구조화하고, 관리자 검증을 거쳐 교육·보호조치·월별 추적관리로 환류하는 운영 가능 단계**에 있다.

다만 공모전과 대외 상품화 기준에서는 아직 그대로 내보내면 안 되는 지점이 있다. 핵심 리스크는 기술 완성도보다 **개인평가처럼 보이는 화면 언어와 리포트 표현**이다.

최종 판단:
- 공모전 참가 준비도: **가능, 단 발표 포지션과 공개 화면 정리가 필수**
- 상품화 공개 준비도: **제한 공개 가능, 개인 리포트/수치 노출 정책 보강 후 권장**
- 현재 가장 큰 리스크: **개인별 수치·초급/중급/고급이 처벌·인사평가로 오해될 가능성**
- 가장 큰 강점: **비식별 실증자료, 관리자 검증, 공식 리포트, 원페이지 교육자료, 월별 추적 루프가 이미 존재**

- PSI는 이들과 같은 축으로 싸우면 불리하다.
- PSI는 **장비형 안전기술이 보지 못하는 근로자의 위험 인식, 언어 장벽, 반복 취약 표현, 교육 환류 단절을 해결하는 Safety Intelligence 보완 모듈**로 포지셔닝해야 한다.

## 3. 현재 프로그램 수준

### 이미 준비된 강점

1. OCR·AI 분석 루프
   - 수기 위험성평가 기록지를 분석 대상으로 삼고, Q1~Q5 답변을 구조화하는 방향이 잡혀 있다.
   - 변경 양식 대응, Q1 공종 오분류, 대량 OCR 실패 격리 등 운영 이슈를 이미 고도화 대상으로 반영했다.

2. 관리자 검증 루프
   - 관리자 수정이 공식 리포트, 보호 해석, 개인 안전역량 세부지표 재계산으로 이어지는 검증 장치가 있다.
   - 확인 결과 `check:manager-review-sync` 통과.

3. 비식별 증빙 패키지
   - 원본 이미지·서명·전체 수기문장 등을 제외한 검증용 비식별 요약 저장 기능이 있다.
   - 확인 결과 `check:ocr-verification-package` 통과.

4. 상품화 UI 방향
   - OCR 목록과 근로자 상세 화면이 `보호 판단`, `교육 리포트`, `원본 재판독` 중심으로 정리되기 시작했다.
   - 확인 결과 `check:product-ui-surgery` 통과.

5. 점수 일관성
   - 유사 맥락 점수 편차를 제한하는 게이트가 있다.
   - 확인 결과 `check:score-consistency:strict8` 통과.

6. 대외 기술자료의 방향
   - 기존 대외자료에는 이미 `개인 처벌이 아닌 교육 우선순위`, `비식별 집계`, `교육자료 환류` 문장이 들어 있다.
   - 심층리서치의 권고와 방향이 대체로 일치한다.

### 아직 부족한 지점

1. 개인평가 오해 방지 언어가 완전히 통일되지 않았다.
   - 일부 화면에는 `초급`, `중급`, `고급`, `점수`, `LEVEL`, `SIGNAL`이 직접 노출된다.
   - 내부 계산에는 필요하지만 대외 발표와 근로자 전달 화면에서는 오해 가능성이 크다.

2. 개인 리포트 출력 검증이 오늘 기준 불안정하다.
   - 과거 산출물에는 운영 URL 기준 개인 리포트 11/11 검증 통과 기록이 있다.
   - 그러나 2026-07-09 로컬 재검증에서는 `현재 보고서 내보내기` 다운로드 이벤트가 30초 내 감지되지 않았다.
   - 최근 UI 변경과 검증 스크립트 또는 PDF 출력 버튼 구조가 어긋났을 가능성이 있다.

3. 공모전용 한 장 증명 화면이 더 강해져야 한다.
   - 현재 기능은 많지만 심사자가 30초 안에 “종이 기록이 데이터가 되고 교육으로 돌아간다”를 보는 전용 화면은 더 압축할 필요가 있다.

4. 사고 감소 효과를 직접 주장하면 위험하다.
   - 현재 입증 가능한 것은 사고율 감소가 아니라 위험 인식 변화, 반복 취약점 추적, 보호 우선 대상 선별, 교육 환류 가능성이다.
   - 발표 문장은 이 범위를 넘지 않아야 한다.

## 4. 개인평가 우려에 대한 판단

### 현재 점수·등급 체계는 내부적으로 필요하다

개인별 안전역량 점수와 초급/중급/고급 단계는 완전히 없애면 안 된다. 이유는 다음과 같다.

- 월별 추적을 하려면 정량 기준이 필요하다.
- 관리자 수정 후 점수 재계산과 무결성 검증을 하려면 내부 기준이 필요하다.
- 교육 우선순위와 보호 우선 대상을 선별하려면 상대적 신호가 필요하다.
- 공모전에서 “AI가 어떤 기준으로 판단했는가”를 설명하려면 내부 루브릭이 있어야 한다.

### 다만 외부 표현은 바꿔야 한다

현재 표현을 그대로 노출하면 `근로자 평가`, `인사자료`, `처벌 대상 분류`로 오해될 수 있다.

권장 원칙:
- 내부 데이터명: `safetyScore`, `safetyLevel` 유지 가능
- 관리자 화면 표현: `위험인식 신호`, `지원단계`, `보호신호`
- 근로자 전달 표현: `오늘 확인할 내용`, `다시 설명할 내용`, `잘 작성된 내용`
- 대외 발표 표현: `비식별 교육 우선순위`, `공종·월별 위험인식 신호`
- 금지 표현: `개인 평가`, `개인별 수치 공개`, `순위`, `낙제`, `인사`, `징계`

### 초급·중급·고급 사용 기준

내부 관리자 모드에서는 유지 가능하다. 하지만 근로자 전달본과 공모전 공개 화면에서는 다음처럼 바꾸는 것이 안전하다.

| 현재 표현 | 권장 표현 | 사용 위치 |
|---|---|---|
| 초급 | 우선지원 | 관리자 보호 판단, 내부 검토 |
| 중급 | 확인 | 관리자 보호 판단, 내부 검토 |
| 고급 | 안정 | 관리자 보호 판단, 내부 검토 |
| 점수 | 위험인식 신호 | 관리자 화면 |
| 개인 점수 | 비식별 교육 우선 신호 | 대외 설명 |
| LEVEL | 지원단계 | 상품화 UI |
| SIGNAL | 보호신호 | 상품화 UI |

대외 문장:
> PSI의 점수는 개인을 평가하는 점수가 아니라, 수기 기록에서 드러난 위험 인식과 교육 환류 필요성을 정리한 `위험인식 신호`입니다. 낮은 단계는 처벌 대상이 아니라 `우선지원` 대상입니다.

## 5. 공모전 발표 포지션

가장 강한 한 줄:

> 장비형 스마트 안전기술이 현장을 보는 기술이라면, NEW-PSI는 현장에서 일하는 근로자의 위험 인식과 실천 약속을 읽는 기술입니다.

보완 문장:

> 기존 안전관리 플랫폼을 대체하지 않고, 수기 위험성평가 기록을 AI 기반 교육·보호조치 데이터로 전환하는 Safety Intelligence 보완 모듈입니다.

절대 피해야 할 표현:
- 사고를 자동으로 막는다.
- AI가 근로자를 평가한다.
- 법정 위험성평가를 대체한다.
- 현장 안전관리자를 대체한다.
- 개인별 점수를 공개한다.

권장 발표 흐름:
1. 문제: 기록은 쌓이지만 위험 인식 데이터는 남지 않는다.
2. 기술: A4 수기 기록을 버리지 않고 OCR·AI로 구조화한다.
3. 검증: 관리자가 원본과 대조하고 수정한다.
4. 환류: 공식 리포트, 다음 달 원페이지 교육자료, 월별 추적관리로 돌아간다.
5. 증거: 비식별 기록 수, 다월 추적, 보호 우선 검토, 교육자료 생성 가능성을 보여준다.

## 6. 즉시 수정보강 우선순위

### P0. 공모전 제출 전 반드시

1. 개인평가 오해 방지 UI 문구 정리
   - `초급/중급/고급` 직접 노출을 대외·근로자 전달 화면에서 숨긴다.
   - `점수`는 `위험인식 신호` 또는 `보호 우선 신호`로 바꾼다.
- 개인 리포트에는 “인사·징계 목적이 아님” 고지를 넣는다.

2. 개인 리포트 PDF/이미지 출력 검증 복구
   - `qa:worker-report`가 최신 UI에서 다시 11/11 통과해야 한다.
   - 버튼명이 바뀌었다면 검증 스크립트를 최신 버튼 흐름으로 업데이트한다.
   - 공모전 제출 전 샘플 PDF 1개를 새로 생성해 보관한다.

3. 공모전용 비식별 데모 데이터셋 고정
   - 실명, 서명, 원본 이미지, 전화번호, 사번, 내부 경로 제거.
   - 공종, 월, 위험 키워드, 교육 우선순위, 변화량만 남긴다.

4. 2분 원포인트 증명 화면 정리
   - `기록지 1장 촬영 → OCR → 관리자 보호 판단 → 모국어 교육/월별 추적` 네 장면만 보여준다.
   - 개발자용 상세 지표는 숨긴다.

### P1. 상품화 공개 전

1. 공개/관리자/개발자 모드 분리 강화
   - 공개 모드: 비식별 집계만.
   - 관리자 모드: 보호 우선 대상과 교육 리포트.
   - 개발자 모드: 점수, OCR 신뢰도, 해시, JSON, 상세 루브릭.

2. 개인정보·보복 우려 방지 정책 문구 내장
   - “근로자 처벌·인사평가 목적 사용 금지”
   - “교육·보호조치 우선순위 산정 목적”
   - “관리자 검토 후 사용”
   - “공개 자료는 비식별 집계 기준”

3. 관리자 수정 이력 감사 화면 정리
   - 누가 언제 무엇을 바꿨는지.
   - 변경 후 모국어 안내와 개인 안전역량이 재계산되었는지.
   - 원본 대조가 가능한지.

### P2. 수상 가능성·사업화 강화

1. 3개월 파일럿 KPI 설계
   - 재작성률
   - 교육 리포트 전달률
   - 하락군 감소율
   - 반복 위험 키워드 감소율
   - TBM 참여 확인률

2. 외부 발표용 대시보드 생성
   - 개인명 없는 공종·월별 위험 신호 화면.
   - “사고 감소” 대신 “위험 인식 변화 추적”을 지표로 둔다.

3. 기술설명서 업데이트
   - `Pre-Input Safety Intelligence Module`
   - `Worker Participation Data Loop`
   - `Human-in-the-loop OCR/AI Review`
   - `Multilingual Safety Education Feedback`

## 7. 완료 기준

공모전 제출 전 완료 기준:
- `verify:fast` 통과
- `qa:worker-report` 최신 UI 기준 통과
- 공모전용 비식별 데모 JSON 1개 준비
- 공모전용 샘플 PDF 또는 이미지 3종 준비
- 2분 원포인트 증명 화면에서 개인 실명·점수 직접 노출 없음
- 발표자료 첫 30초 안에 “종이 기록 → 데이터 → 교육 환류”가 보임

상품화 공개 전 완료 기준:
- 개인평가 오해 방지 문구 전 화면 반영
- 공개 모드에서 개인별 수치와 원문 비노출
- 관리자 모드에서만 보호 우선 신호 확인
- 개발자 모드에서만 OCR 신뢰도·JSON·해시·상세 루브릭 확인
- 관리자 수정 후 보호 해석, 공식 리포트 근거, 점수 재계산이 자동 검증됨

## 8. 최종 판단

PSI는 세상 밖으로 나갈 수 있는 방향에 이미 들어섰다. 그러나 지금 그대로 “개인별 수치와 등급이 보이는 분석 프로그램”처럼 보이면 안 된다.

가장 좋은 상품 정체성은 다음이다.

> NEW-PSI는 건설현장의 수기 위험성평가 기록을 AI로 5문항·6지표 구조화하고, 관리자 검증 리포트와 다음 달 원페이지 교육자료, 월별 추적관리로 환류하는 현장형 Safety Intelligence 보완 모듈이다.

이 정체성으로 정리하면 개인평가 논란은 줄고, 공모전에서는 하드웨어형 기술과 다른 차별 축을 만들 수 있다.
