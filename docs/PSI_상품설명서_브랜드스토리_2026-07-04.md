# PSI 상품설명서 및 브랜드 스토리

작성일: 2026-07-04

## 1. 상품 한 문장

PSI는 건설현장의 수기 위험성평가 기록지를 AI로 구조화하고, 관리자의 현장 판단이 근로자 모국어 안내, 개인 안전역량, 월별 교육 환류까지 같은 기준으로 이어지는지 검증하는 현장 안전 인텔리전스 시스템이다.

## 2. 시작 스토리

PSI는 “근로자가 직접 쓴 위험 신호가 현장 관리와 교육으로 제대로 이어지고 있는가?”라는 질문에서 시작했다.

건설현장에서는 위험성평가 기록지가 매일 작성된다. 그러나 종이에 적힌 짧은 문장, 외국인 근로자의 모국어 표현, 현장 관리자의 수정 판단, 다음 달 교육자료는 서로 떨어져 있었다. 기록은 있었지만 추적은 어려웠고, 교육은 있었지만 개인별 변화가 보이지 않았다.

PSI의 출발점은 이 단절을 연결하는 것이었다. 사진이나 PDF로 받은 수기 기록을 OCR로 읽고, Q1 작업, Q2 위험요인, Q3 위험수준, Q4 감소대책, Q5 실천행동을 구조화한다. 그 뒤 관리자가 한국어 보호 해석을 수정하면, 외국인 근로자에게 전달될 모국어 안내와 개인 안전역량 지표가 같은 기준으로 다시 맞춰진다. 마지막으로 월별 계도 리포트와 TBM 교육자료로 환류한다.

PSI는 사람을 평가하려는 시스템이 아니라, 현장의 위험 신호를 보호의 언어로 바꾸는 시스템이다.

## 3. 해결하는 문제

| 현장 문제 | 기존 한계 | PSI 해결 방식 |
|---|---|---|
| 수기 기록이 쌓이지만 분석이 어렵다 | 사진/PDF/종이 자료가 흩어짐 | OCR 분석 후 Q1~Q5와 위험요인을 구조화 |
| 외국인 근로자에게 수정 내용이 전달되지 않는다 | 한국어 수정과 모국어 안내가 분리됨 | 관리자 수정 시 모국어 안내 준비 상태를 검증 |
| 개인별 안전 수준 변화가 보이지 않는다 | 한 번 분석 후 끝나는 기록 중심 | 개인 안전역량 세부지표와 월별 추적관리 |
| 교육자료가 현장 데이터와 약하게 연결된다 | 교육 주제가 경험 의존 | OCR 분석 결과를 월별 TBM/A4/QR 교육으로 환류 |
| 상품 설명 시 증명이 어렵다 | “AI가 분석한다”는 말만 남음 | 검증 패키지, 전후 비교표, 승인 차단 사유를 증빙화 |

## 4. 핵심 기능

| 기능 | 설명 | 상품 가치 |
|---|---|---|
| OCR 위험성평가 분석 | 이미지/PDF/수기 텍스트를 분석해 문항별 답변과 위험 신호를 추출 | 대량 기록을 빠르게 구조화 |
| 관리자 검증 | 한국어 보호 해석, 점수 근거, 위험요인 분류를 관리자가 수정 | AI 판단을 현장 판단으로 보정 |
| 모국어 안내 동기화 | 외국인 근로자에게 전달될 모국어 안내 상태를 확인 | 전달 누락과 설명 책임 리스크 감소 |
| 개인 안전역량 지표 | W1~W5 가중치와 W6 감점 배율로 개인별 역량 점수 계산 | 회사 규칙과 현장 중점에 맞는 평가 기준 |
| 월별 리포트 | 공종/팀/월별 추이, 반복 위험, 교육 우선순위를 집계 | 추적관리와 바이어 보고에 사용 |
| 교육 환류 | 월별 계도, A4 자료, 다국어 QR 교육으로 연결 | 분석이 현장 행동으로 이어짐 |
| 검증자료 저장 | 전후 비교표, 증빙 패키지, 감사 이력으로 남김 | 실증자료와 상품 신뢰 확보 |

## 5. 개인 안전역량 W 지표

| 코드 | 공식 명칭 | 계산 역할 | 운영 해석 |
|---|---|---|---|
| W1 | 심리/작성 안정성 | 점수 비중 | OCR 신뢰도, 답변 압박 신호, 작성 누락을 반영 |
| W2 | 작업 이해도 | 점수 비중 | 공종, 작업 위치, 세부 작업 이해를 반영 |
| W3 | 위험성평가 이해도 | 점수 비중 | 위험요인, 위험수준, 사고 가능 이유의 연결성을 반영 |
| W4 | 현장 숙련도 | 점수 비중 | 실무 행동의 구체성과 경험 기반 판단을 반영 |
| W5 | 개선 이행도 | 점수 비중 | 작업 전·중·후 실행 행동의 명확성을 반영 |
| W6 | 반복지적 감점 배율 | 감점 배율 | 반복위반 또는 상투어 반복 신호가 있을 때 감점 강도를 정함 |

운영 원칙:

- W1~W5 합계는 1.00을 권장한다.
- W6은 합계에 포함하지 않는다.
- 회사 규칙상 특정 지표를 제외해야 하면 해당 W값을 0으로 둘 수 있다.
- 프리셋은 기본 균형형, 위험인지 강화형, 개선이행 강화형, 숙련/리더십 강화형으로 시작하고 현장별로 조정한다.

## 6. 바이어 시연 흐름

1. 실제 수기 위험성평가 이미지 1장을 OCR에 넣는다.
2. Q1~Q5가 문항별로 분리되는지 보여준다.
3. 관리자가 한국어 보호 해석 또는 점수 근거를 수정한다.
4. 모국어 안내 상태와 개인 안전역량 재계산 결과를 확인한다.
5. 승인이 막히거나 통과되는 사유가 감사 이력에 남는 것을 보여준다.
6. 월별 리포트와 다음 TBM 교육자료로 연결되는 것을 보여준다.
7. 전후 비교표를 내보내 “실증자료로 남는다”는 점을 증명한다.

## 7. 상품화 메시지

### 현장 관리자용

“사진으로 찍은 위험성평가 기록지를 바로 분석하고, 보강이 필요한 근로자와 공종을 한눈에 확인합니다.”

### 안전팀/본사용

“현장별 수기 기록을 같은 기준으로 집계하고, 월별 위험 신호와 교육 우선순위를 데이터로 설명합니다.”

### 바이어/의사결정자용

“PSI는 단순 OCR이 아니라 관리자 검증, 모국어 전달, 개인역량 재계산, 교육 환류가 연결된 현장 검증형 안전관리 상품입니다.”

## 8. 실증자료 환류 구조

| 단계 | 남기는 자료 | 환류 목적 |
|---|---|---|
| 분석 전 | 원본 이미지, 기존 점수, 기존 해석 | 원본 증빙 보존 |
| 관리자 수정 | 수정 항목, 수정 전후 점수, 수정 사유 | 현장 판단 근거 확보 |
| 동기화 확인 | 모국어 안내 상태, 개인 지표 재계산 여부 | 외국인 전달 누락 방지 |
| 승인 판단 | 승인 가능 여부, 차단 사유 | 감사 대응과 운영 책임 |
| 월별 환류 | 공종/팀/국적/월별 표 | 교육자료와 개선 추적 |

## 9. 다음 현장 검증 계획

1. 변경 양식 정상 분석 건 중 외국인 기록 5~10건을 선정한다.
2. 관리자 수정 전 상태를 저장한다.
3. 한국어 보호 해석, 점수 근거, 위험요인 해석 중 하나를 수정한다.
4. 저장 후 모국어 안내 상태와 개인 안전역량 지표 변화가 생겼는지 확인한다.
5. `templates/psi_field_validation_before_after_2026-07-04.csv`에 결과를 입력한다.
6. `npm run report:field-validation-feedback`로 표와 요약 리포트를 생성한다.
7. 생성된 표를 바이어 발표자료와 상품 검증자료로 사용한다.
