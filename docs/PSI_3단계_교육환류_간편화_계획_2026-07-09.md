# PSI 3단계 교육 환류 간편화 계획

작성일: 2026-07-09

## 1. 확인검증 결론

첨부 이미지의 핵심 문장과 구조는 PSI 정체성과 잘 맞는다.

> 익숙한 수기 작성 흐름을 유지한 채, 버려지던 종이를 예측 가능한 데이터로 구조화한다.

현재 프로그램에도 이 구조의 각 기능은 이미 존재한다.

| 이미지 흐름 | 현재 구현 위치 | 확인 결과 |
|---|---|---|
| 1. 찍는다 | `위험성평가 분석`, OCR 이미지/PDF 분석 | 구현됨 |
| 2. 확인한다 | OCR 추출, AI 분석, 관리자 검증, 점수 보정 | 구현됨 |
| 3. 교육한다 - 다국어 전파 안내 | `위험성평가 교육자료`, `QR/음성 파일럿` | 구현됨. 단 QR/음성은 파일럿 보조 채널로 유지해야 함 |
| 3. 교육한다 - 개인 리포트 | `근로자 리포트 (관리자 분석)` | 구현됨 |
| 3. 교육한다 - 월별 계도 자료 | `월별 계도 리포트` | 구현됨 |

부족한 점은 기능 자체가 아니라 **한눈에 보는 구조**다. 현재는 메뉴가 나뉘어 있어 개발자는 흐름을 알지만, 처음 보는 사람은 “OCR 다음에 무엇을 해야 하는지”가 바로 들어오지 않는다.

## 2. 최적화 방향

새로운 큰 기능을 추가하기보다, 기존 기능을 묶는 **교육 환류 3출력 패널**을 만든다.

이 패널은 `찍는다 → 확인한다 → 교육한다` 중 3단계인 `교육한다`를 한 화면에서 보여주는 출구 역할을 한다.

### 권장 화면명

`교육 환류`

### 권장 한 문장

> 검증된 기록은 세 가지 결과물로 돌아갑니다. 현장 공통교육, 개인 보호 리포트, 월별 계도·추적자료입니다.

## 3. 3가지 결과물 분류

| 구분 | 화면 표현 | 역할 | 연결 화면 |
|---|---|---|---|
| 1. 현장 공통교육 | 원페이지 교육자료 | 다음 달 위험성평가 교육에 바로 쓰는 A4 한 장 자료 | `위험성평가 교육자료` |
| 2. 개인 보호 안내 | 개인 보호 리포트 | 근로자별 이해 신호와 보호 안내를 관리자 관점에서 확인 | `근로자 리포트` |
| 3. 월별 추적 | 월별 계도·추적자료 | 개인식별 없이 공종/월별 위험 신호를 교육자료와 보고자료로 환류 | `월별 계도 리포트` |

`QR/음성 파일럿`은 1번 현장 공통교육의 보조 전달 채널로 둔다. 별도 핵심 축으로 세우면 다시 정체성이 흔들릴 수 있다.

## 4. 화면 구성안

### A. 대시보드 상단 3단계 바

첫 화면에 아래 흐름을 고정한다.

1. 찍는다  
   수기 기록지 사진/PDF를 넣는다.

2. 확인한다  
   OCR·AI 분석 결과를 관리자가 원본과 대조한다.

3. 교육한다  
   검증 결과를 세 가지 출력물로 만든다.

### B. 교육한다 카드 3개

세 번째 단계 안에는 3개의 큰 카드만 둔다.

1. `원페이지 교육자료`
   - 상태: 생성 가능 / 초안 있음 / 확정됨
   - 주요 값: 기준월, 공종, 중점 위험 TOP3
   - 버튼: `교육자료 만들기`

2. `개인 보호 리포트`
   - 상태: 확인 필요 N명 / 생성 가능 N명
   - 주요 값: 보호 우선 신호, 응답 품질 신호
   - 버튼: `리포트 확인`

3. `월별 계도·추적자료`
   - 상태: 월별 집계 가능 / 데이터 부족 / 추적 중
   - 주요 값: 전월 대비, 반복 위험, 개선 이행률
   - 버튼: `월별 자료 보기`

### C. 보조 채널

`원페이지 교육자료` 카드 안에 작은 보조 버튼으로만 둔다.

- `필요 언어본 보기`
- `QR/음성 파일럿`

이렇게 해야 다국어/QR이 핵심처럼 보이지 않고, 확정된 교육자료의 전달 수단으로 보인다.

## 5. 구현 순서

### 1단계: 구조 정리

- `components/IntegratedWorkBoard.tsx`에 `교육 환류 3출력 패널` 추가
- 기존 `월별 계도 리포트`, `위험성평가 교육자료`, `근로자 리포트`, `QR/음성 파일럿` 버튼을 한 그룹으로 묶음
- 모바일에서는 `교육자료` 탭을 누르면 3출력 패널이 먼저 보이도록 정리

### 2단계: 상태 요약 로직

새 유틸을 만든다.

`utils/educationReturnSummary.ts`

계산 항목:

- OCR 분석 완료 건수
- 관리자 확인 필요 건수
- 원페이지 교육자료 생성 가능 여부
- 개인 보호 리포트 대상 수
- 월별 계도자료 생성 가능 여부
- QR/음성 파일럿 연결 가능 여부

### 3단계: 검증 장치

새 계약 검증을 추가한다.

`scripts/check-education-return-panel-contract.cjs`

검증 항목:

- `찍는다`, `확인한다`, `교육한다` 3단계 문구가 첫 화면에 존재
- 교육한다 아래에 `원페이지 교육자료`, `개인 보호 리포트`, `월별 계도·추적자료`가 존재
- `QR/음성 파일럿`이 독립 핵심축이 아니라 보조 채널로 표시
- 개인 화면에는 점수/등급 중심 표현이 노출되지 않음

### 4단계: 브라우저 확인

검증 화면:

- 데스크톱 첫 화면
- 모바일 폭 첫 화면
- OCR 완료 후 이동 흐름
- 교육자료 카드 클릭
- 개인 보호 리포트 카드 클릭
- 월별 계도자료 카드 클릭

확인 기준:

- 처음 보는 사람이 10초 안에 “OCR 결과가 어디로 가는지” 이해 가능해야 함
- 버튼은 3개 중심이어야 함
- QR/음성은 보조로 보여야 함

## 6. 발표용 설명 문장

> PSI는 종이 기록지를 없애는 프로그램이 아닙니다. 근로자가 익숙하게 쓰던 수기 기록지를 그대로 유지하면서, 그 안의 위험 인식을 OCR·AI와 관리자 검증으로 구조화합니다. 검증된 결과는 현장 공통교육, 개인 보호 리포트, 월별 계도·추적자료 세 가지로 돌아가 다음 달 위험성평가 교육과 추적관리의 근거가 됩니다.

## 7. 최종 권장안

가장 좋은 구조는 별도 복잡한 페이지를 추가하는 것이 아니라, 기존 대시보드와 모바일 첫 화면에 `교육 환류 3출력 패널`을 배치하는 것이다.

즉, 사용자에게 보이는 흐름은 단순해야 한다.

```text
찍는다
  ↓
확인한다
  ↓
교육한다
  ├─ 원페이지 교육자료
  ├─ 개인 보호 리포트
  └─ 월별 계도·추적자료
```

뒤에서는 OCR, AI 분석, 관리자 검증, 점수 보정, 리포트 생성, 월별 추적이 돌아가지만, 바이어와 현장 사용자는 이 3단계와 3가지 결과물만 보면 된다.
