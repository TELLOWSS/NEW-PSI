# Reports 경보 CTA 로그 동기화 실검증 체크리스트 (2026-05-20)

## 목적
- Reports의 경보 CTA 로그가 `서버 우선 + 로컬 폴백`으로 정상 동작하는지 검증
- `전체 초기화`가 서버/로컬 모두에서 일관되게 동작하는지 검증
- Introduction QA RUNLOG 최신 상태가 Reports에서 요약/점프 버튼으로 연동되는지 검증

---

## 사전 준비
1. Supabase SQL Editor에서 `supabase_ops_alert_click_logs_migration.sql` 실행
2. 앱 실행 후 `PSI 정밀 보고서 센터` 진입
3. 경보 CTA 로그 카드에서 동기화 상태 문구 확인

기대값:
- 정상 환경: `서버 연결`
- 스키마 미적용/서버 오류: `로컬 폴백`

---

## 시나리오 A: 서버 저장/복원 검증
1. `8번 개입 화면으로 이동` 또는 `10번 태깅 검증으로 이동` 버튼 1회 클릭
2. Reports로 복귀하여 로그 1건 증가 확인
3. 브라우저 새로고침
4. 동일 로그가 유지되는지 확인

합격 기준:
- 새로고침 후에도 클릭 로그가 유지됨
- 동기화 상태가 `서버 연결` 유지

---

## 시나리오 B: 기간/액션 필터 정합성 검증
1. 액션 필터를 `8번 개입 이동`으로 선택
2. 기간 프리셋을 `오늘`, `최근 7일`, `최근 30일` 순서로 변경
3. KPI 카드(총 클릭수/8번 이동률/10번 이동률/경보활성 클릭 비율) 값 변화 확인
4. 필요 시 `사용자 지정` 기간으로 교차 검증

합격 기준:
- 필터 변경 시 로그 리스트와 KPI가 같은 기준으로 함께 변함
- 증감률 문구가 비교 기준(`전일 대비`, `이전 7일 대비`, `이전 30일 대비`, `직전 동일 기간 대비`)과 일치함

---

## 시나리오 C: 전체 초기화 일관성 검증
1. `전체 초기화` 버튼 클릭
2. 로그 리스트가 즉시 0건으로 비워지는지 확인
3. 브라우저 새로고침
4. 로그가 다시 나타나지 않는지 확인

합격 기준:
- 초기화 직후/새로고침 후 모두 0건 유지
- 동기화 상태 문구가 실패 없이 갱신됨

---

## 시나리오 D: 폴백 동작 검증
1. 서버 접근이 어려운 환경(또는 스키마 미적용)에서 클릭 로그 생성
2. 동기화 상태가 `로컬 폴백`으로 표시되는지 확인
3. 새로고침 후 로컬 로그 유지 여부 확인

합격 기준:
- 서버 실패 시 앱이 중단되지 않고 로컬 로그로 동작 지속
- 경고/상태 문구로 현재 모드를 인지 가능

---

## 시나리오 E: Introduction QA RUNLOG 연동 검증 (2026-05-21 추가)
1. Introduction 화면에서 `12 SCREEN FEATURE CHECK` 섹션 확인
2. QA 상태를 1회 이상 생성(정상/경고)
3. Reports의 `경보 CTA 클릭 로그` 카드에서 `Introduction QA RUNLOG` 요약 표시 확인
4. 경고 페이지 버튼이 보이면 클릭해 대상 화면으로 이동 확인

합격 기준:
- Reports에 최신 점검시각/연결/데이터/경고 건수가 표시됨
- 경고 페이지 버튼 클릭 시 해당 페이지로 즉시 이동됨
- Introduction에서 상태가 바뀌면 Reports 재진입(또는 storage 갱신) 시 최신 상태로 반영됨

---
