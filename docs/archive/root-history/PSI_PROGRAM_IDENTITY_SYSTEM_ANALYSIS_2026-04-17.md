# PSI 프로그램 전체 정체성·작동원리·보강전략 분석

작성일: 2026-04-17  
분석 범위: NEW-PSI 전체 프로그램(제품/운영/아키텍처/거버넌스)

---

## 0) 이 문서의 관점

이 문서는 특정 릴리즈 변경점 보고가 아니라, PSI 프로그램 전체를 하나의 운영 시스템으로 해석한 구조 분석서다.  
핵심 질문은 다음 5가지다.

1. PSI는 본질적으로 무엇인가?
2. PSI는 어떤 원리로 작동하는가?
3. 현재 강점과 구조적 한계는 무엇인가?
4. 무엇을 보강해야 실제 가치가 커지는가?
5. PSI가 지향해야 할 최종 상태는 무엇인가?

---

## 1) PSI의 아이덴티티 (정체성)

## 1.1 한 줄 정의

**PSI는 건설 현장의 위험 신호를 디지털화하고, 사람 중심 언어로 해석하여, 관리자 조치와 조직 학습까지 연결하는 안전 운영 체계다.**

## 1.2 PSI를 구성하는 정체성 4요소

1. **Signal Translator (신호 번역기)**  
   수기 기록/현장 문서를 OCR·AI로 읽고 위험 신호를 구조화한다.

2. **Decision Harness (판단 하네스)**  
   AI 출력을 그대로 확정하지 않고 상태 머신·승인 흐름·가드레일로 통제한다.

3. **Action Orchestrator (조치 오케스트레이터)**  
   재분석, 보완 검토, 관리자 정상분류, 승인/보류를 운영 동선으로 연결한다.

4. **Trust Ledger (신뢰 원장)**  
   수정·승인·재분석·근거를 이력으로 남겨 감사 가능성과 재현성을 확보한다.

## 1.3 PSI가 “OCR 솔루션”과 다른 이유

- OCR 정확도가 목적이 아니라, **안전 의사결정 품질**이 목적이다.
- 출력 결과가 목적이 아니라, **현장 행동 전환**이 목적이다.
- 단일 분석이 목적이 아니라, **검증 가능한 반복 운영**이 목적이다.

## 2) PSI 작동원리 (Operating Mechanics)

## 2.1 전체 작동 루프

PSI는 다음의 6단계 루프로 작동한다.

1. **Capture**: 문서/이미지 입력 수집
2. **Extract**: OCR·파싱으로 텍스트/구조화 데이터 생성
3. **Interpret**: 위험·점수·근거·코칭 생성
4. **Govern**: 실패코드/상태전이/인간승인으로 통제
5. **Act**: 관리자 조치 실행(재분석, 분류, 승인, 보완)
6. **Learn**: 이력/분포/패턴으로 운영정책 개선

## 2.2 시스템의 3계층 원리

### A. 경험 계층 (UX Layer)
- 근로자: 심리저항을 낮추는 코칭형 언어
- 관리자: 근거 중심의 즉시 조치 언어
- 경영진: 리스크 집중도/추세 중심 요약

### B. 판단 계층 (Logic Layer)
- WorkerRecord 중심 데이터 모델
- 실패코드 체계(QUOTA, KEY, FORMAT, PARSE, PAYLOAD, NETWORK, UNKNOWN)
- 상태 머신(업로드→검토→승인/재분석→완료)

### C. 통제 계층 (Governance Layer)
- 하네스 기반 전이 제약
- 승인 정책과 근거 코멘트 요구
- 감사 추적(수정/승인/재분석 이력)

## 2.3 PSI의 핵심 제어 변수

- 입력 신뢰도: OCR confidence, 이미지 품질
- 분석 신뢰도: 점수-등급 정합성, 근거 충실도
- 운영 신뢰도: 실패코드 분포, 재분석 회복률, 승인 사유 품질
- 조직 신뢰도: 이력 재현성, 정책 일관성

## 3) PSI의 현재 강점

## 3.1 제품 구조 강점

- 기능이 분절되지 않고 하나의 운영 흐름으로 연결됨
- 관리자 액션(재분석/정상분류/승인)이 명시적
- 실패를 단일 에러가 아닌 코드 체계로 관리
- 역할별 톤·문구 원칙이 정의되어 있음

## 3.2 아키텍처 강점

- 프론트/서비스/API/하네스 계층이 분리되어 확장 가능
- 상태와 이력 모델이 types 중심으로 정규화됨
- OCR 경로에 폴백/재시도/쿨다운 등 운영 복원력 장치 존재

## 3.3 운영 강점

- 실무자가 즉시 볼 수 있는 집계 지표(성공/실패/라우트/폴백)
- 사후 감사에 필요한 변경 이력/근거 축적 구조
- 문서화 문화가 강해 온보딩/인수인계 비용이 낮음

---

## 4) PSI의 구조적 한계 (현재 보강 필요)

## 4.1 의미 해석의 불안정성

- UNKNOWN은 운영자에게 “원인 미상” 부담을 남긴다.
- 일부 실패는 메시지/휴리스틱 의존이 높아 오탐 가능성이 생긴다.

## 4.2 운영 가시성의 불완전성

- provider/latency/attempt/fallback depth 같은 추적 필드가 상시 표준화되지 않으면,
  장애 분석이 경험 의존으로 흐를 수 있다.

## 4.3 정책-실행 간 간극

- 브랜드 보이스/운영 원칙은 문서화되어 있으나,
  모든 화면/모든 액션에서 동일 강도로 강제되는지는 주기적 점검이 필요하다.

## 4.4 조직 확장 리스크

- 팀/현장 수가 늘수록 “사람 기억 기반 운영”은 급격히 한계에 부딪힌다.
- KPI와 정책버전이 자동 연동되지 않으면 개선 주기가 느려진다.

---

## 5) 보강해야 할 사항 (우선순위)

## 5.1 P0: 운영 안정성 즉시 보강

1. **UNKNOWN 2차 분류 도입**
   - UNKNOWN을 network-like / parse-like / policy-like / uncategorized로 세분
2. **Trace 표준화**
   - providerUsed, latencyMs, attempts, fallbackDepth, finalCode 저장
3. **전/후 비교 패널 상시화**
   - 서버성공, 처리실패, UNKNOWN 비중을 24h/7d로 자동 비교

## 5.2 P1: 거버넌스 고도화

1. **정책 엔진 설정화**
   - 임계값·승인조건·예외룰을 하드코딩에서 정책 데이터로 전환
2. **승인 사유 품질 게이트**
   - 근거 길이보다 근거 구조(원인-조치-검증)를 점검
3. **하이브리드 OCR 오케스트레이션 본격화**
   - provider chain 전략을 운영 데이터 기반으로 최적화

## 5.3 P2: 가치 확장

1. **월간 리스크 인텔리전스 자동화**
   - 반복 실패 패턴, 공종별 취약구간, 조치 효과를 자동 리포트
2. **현장별 정책 편차 관리**
   - 현장 특성 반영 정책과 공통 정책의 분리 운영
3. **교육-평가-행동 연동**
   - 교육 이수 상태와 위험 신호를 결합한 선제 개입 체계 구축

---

## 6) PSI의 지향점 (North Star)

## 6.1 목표 상태

PSI의 최종 지향점은 다음 3가지를 동시에 달성하는 것이다.

1. **정확성**: 위험 신호를 놓치지 않는다.
2. **실행성**: 운영자가 바로 조치할 수 있다.
3. **증빙성**: 나중에 검증해도 판단의 맥락이 남아 있다.

## 6.2 북극성 지표 제안

- 운영 정확도: 처리 실패율, UNKNOWN 비중, 재분석 회복률
- 운영 속도: 실패 감지→조치 완료까지 소요시간
- 운영 품질: 승인 사유 충실도, 재발률 감소
- 조직 효과: 현장 리스크 지표의 월간 개선 추세

---

## 7) PSI가 제공하는 핵심 가치

## 7.1 근로자 관점

- 점수 통보보다 행동 보완 중심 피드백 제공
- 비난형 시스템이 아닌 보호형 시스템 경험

## 7.2 관리자 관점

- 실패를 코드와 조치로 분해해 판단 피로를 낮춤
- 재분석/승인/분류를 하나의 동선에서 처리 가능

## 7.3 경영진/조직 관점

- 안전 리스크를 숫자와 이력으로 설명 가능
- 감사/법규 대응에 필요한 재현 가능성 확보
- 데이터 기반 정책 개선(학습 조직) 가능

---

## 8) 최종 진단

PSI의 본질은 “AI가 잘 읽는가”가 아니라, **조직이 안전을 더 잘 운영하게 만드는가**다.  
현재 PSI는 이미 단순 기능형 제품을 넘어 운영체계의 형태를 갖추고 있다.  
다음 도약의 핵심은 정확도 개선 그 자체보다, **거버넌스 표준화·추적성 강화·정책 자동화**를 통해 시스템 신뢰를 제도화하는 것이다.

---

## 9) 실행 권고 (90일)

- 0~30일: UNKNOWN 세분화 + trace 필드 상시 저장 + 전/후 비교 패널 고정
- 31~60일: 정책 엔진 설정화 + 승인 사유 품질 게이트 적용
- 61~90일: 하이브리드 체인 성능 최적화 + 월간 리스크 인텔리전스 자동화

이 90일 계획이 완료되면 PSI는 “분석 도구”가 아니라, **현장 안전 운영 표준 플랫폼**으로 자리 잡을 가능성이 높다.
