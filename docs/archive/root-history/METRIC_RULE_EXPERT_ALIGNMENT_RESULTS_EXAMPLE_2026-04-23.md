## 4) 원인 유형별 분석

### A1. 앵커 해석 차이 (3건: W001, W003, W009)
**현상**: 같은 위험 상황을 보고 판정 등급이 다식나
- W001: A/시스템 "A급(28)", B "B급(26)" → "안전대 미착용"을 A가 더 심각하게 봄
- W003: A "C급(20)", B "B급(22)", 시스템 "C급(18)" → 임시지보 기준 해석 차이
- W009: A "B급(21)", B "C급(19)", 시스템 "C급(20)" → 배수 기준 해석 차이

**조치**: 앵커 판정 가이드에 사진/사례 추가

### A2. 입력 정보 부족 (1건: W004)
**현상**: 근로자 기술 내용이 모호해서 앵커 판정이 어려움
- W004 "높이 탈락물" → 높이가 명시 안 됨, 탈락 물체가 뭔지 불명확
- 시스템: 정보 부족으로 보수적 평가(22점)
- A/B: 추측으로 채워 24~23점

**조치**: 입력 템플릿에 필수 항목 강제 (높이, 질량, 물체 종류)

### A3. 기준 엄격도 (1건: W005)
**현상**: 평가자 성향의 차이 (A가 조금 더 관대)
- A "우수한 예방 기술(31)" vs B/시스템 "확실한 기술(30)" → 1점 차이
- 범위 내 편차이므로 운영상 무방

**조치**: 추가 조치 없음 (정상 범위)

## 5) 구체적 불일치 사례 분석

### 사례 1: W003 (굴착, 토사 붕괴)
```
근로자 기술:
"깊이 4m 굴착 중에는 토사가 붕괴될 수 있으니까 
임시지보를 설치해야 한다. 우리는 사다리도 놨다."

평가자 A (20점): C级 (구호형)
- 근거: "임시지보"와 "사다리" 언급만 있고, 
  - 설치 기준(거리, 각도, 재료)이 없음
  - 점검 주기가 없음
  - 붕괴 사전 신호가 없음

평가자 B (22점): B급 (조건 제시 있으나 기준 부족)
- 근거: "임시지보 설치"라는 행동이 명시되어 있으므로 
  - B급(조건만 충족)으로 높게 평가

시스템 (18점): C급 (보수적)
- 근거: scoreBreakdown에서 
  - "위험조건" 인식 O (토사붕괴)
  - "행동" O (임시지보, 사다리)
  - "검증기준" X (설치 기준 누락) → C급

개선안:
- 근로자: "사다리는 45도 각도로 설치하고, 
  임시지보는 1m 간격으로 점검한다" 추가 → A급 가능
- 시스템: B급과 C급의 경계를 "행동만 있어도 B" vs "검증기준까지"로 명확히
```

### 사례 2: W004 (거푸집, 탈락물)
```
근로자 기술:
"높이에서 떨어지면 위험하니까 조심해야 한다."

평가자 A (24점): B급
- 근거: "높이에서 떨어짐" = 위험조건 언급

평가자 B (23점): B중하
- 근거: "조심해야 한다" = 구호형이므로 B 하단

시스템 (22점): C급
- 근거: 
  - "위험조건" 모호 (높이가 얼마인가?)
  - "행동" 없음 (뭘 해야 하는가?)
  - "검증기준" 없음

문제점:
- 근로자 기술이 너무 일반적
- 입력 필드에 "높이", "탈락 물체 종류" 등이 없어서 구체성 부족

해결책:
- 입력 템플릿 보강: 
  - "현재 작업 높이는?" (m 단위)
  - "탈락 가능 물체는?" (자재/도구/구조재)
  - "하단에 근로자가 있는가?"
```

## 7) 평가자별 피드백

### 평가자 A (관리자)
- 강점: 현장 실무 관점에서 실행 가능성을 중심으로 평가
- 약점: 기준 엄격도가 낮아 조금 과평가 경향
- 권장: strict8 게이트가 있으므로 현재 수준 유지 가능

### 평가자 B (감시원)
- 강점: 법적/기준 관점에서 엄격하게 평가
- 약점: 실제 현장 행동의 효과성 판단이 보수적
- 권장: A와의 균형을 맞추기 위해 현장 교육 추가

### 시스템 규칙
- 강점: 구체성 부족한 기술을 객관적으로 낮게 평가
- 약점: 경계(B/C급) 사례에서 편차 가능
- 권장: 입력 정보 품질 개선으로 신뢰도 향상 가능
