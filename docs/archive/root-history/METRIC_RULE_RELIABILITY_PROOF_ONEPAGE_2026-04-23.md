# 지표 규칙 신뢰성 입증 원페이지 (2026-04-23)

## 0) 목적
이 문서는 6대 지표 및 운영 규칙을 **근로자별로 적용했을 때**, 해당 규칙이
- 실무자에게는 공정하고 이해 가능하며,
- 평가자에게는 일관되고 감사 가능하고,
- 경영자에게는 재현 가능하며 성과로 연결된다는 점을
짧고 명확하게 입증하기 위한 1페이지 기준서입니다.

---

## 1) 한 줄 결론
규칙의 신뢰성은 **설명 가능성 + 일관성 + 재현성 + 현업 성과** 4축으로 입증합니다.

---

## 2) 입증 프레임워크 4축

### A. 설명 가능성
질문: "왜 이 근로자가 이 점수를 받았는가?"

증빙 기준:
- 각 지표별 `scoreReasoning` 기록 존재
- 앵커(A/B/C) 판정 근거 최소 1개 이상 존재
- 점수보다 먼저 행동 근거가 설명 가능

합격 기준:
- 표본 점검 시 근거 없는 점수 0건
- 점수 변경 건은 변경 사유 100% 기록

---

### B. 일관성
질문: "비슷한 근로자에게 비슷한 점수가 부여되는가?"

증빙 기준:
- 유사 공종/유사 위험/유사 행동 수준 비교쌍 구성
- `strict8(±8)` 게이트 통과 여부 확인
- 경미 수정(오타/문장정리) 시 점수 급변 없음

합격 기준:
- `strict8` PASS
- 최대 편차 허용 범위 내 유지
- 경미 수정 전후 급변 사례 0건

---

### C. 재현성
질문: "같은 데이터를 같은 규칙으로 다시 보면 같은 결과가 나오는가?"

증빙 기준:
- 규칙 버전 고정
- 실행 시각, 브랜치, 커밋, dirty 상태 기록
- 릴리즈 리포트 보관

합격 기준:
- 동일 입력 재실행 시 결과 동일
- 리포트 메타데이터 누락 0건
- 배포 전 `verify:release` 통과

---

### D. 현업 성과
질문: "이 규칙이 실제 운영 의사결정에 도움이 되는가?"

증빙 기준:
- 고위험군 선별 정확도 개선
- 관리자 수동 재조정률 감소
- 후속조치 연결률 상승
- 비용 절감과 품질 유지 동시 확인

합격 기준:
- 고위험군 조치 연결률 상승 추세
- 수동 수정률 또는 번복률 감소
- 비용/품질/재현성 3축 동시 관리 가능

---

## 3) 역할별 입증 포인트

| 대상 | 가장 먼저 묻는 질문 | 반드시 보여줄 증빙 | 핵심 KPI |
| --- | --- | --- | --- |
| 근로자 | 내 평가는 공정한가? | 점수 근거 문장, 앵커 판정, 변경 사유 | 이의제기율, 점수설명 가능률 |
| 평가자 | 같은 기준으로 적용됐는가? | `strict8`, 비교쌍 리포트, 감사로그 | 일관성 PASS율, 변경사유 기록률 |
| 경영자 | 운영 가능한가? 성과가 나는가? | 릴리즈 리포트, 추세표, 비용/품질/재현성 3축 | 비용 절감률, 검증 통과율, 조치 연결률 |

---

## 4) 실무 입증 절차(권장 순서)

### 1단계. 표본 검증
- 근로자 샘플 30~50건 추출
- 각 건에 대해 `scoreReasoning`, 앵커, 점수 변경 이력 확인
- 근거 없는 점수 여부 점검

### 2단계. 전문가 합치도 검증
- 관리자/평가자 2명 이상이 같은 샘플을 독립 판정
- 시스템 결과와 비교
- 차이 발생 시 유형 분류:
  - 기준 해석 차이
  - 문장 정보 부족
  - 규칙 보정 필요

### 3단계. 일관성 게이트 검증
- 유사 맥락 비교쌍으로 `strict8(±8)` 실행
- 최대 편차, 위험 비교쌍 수, PASS/FAIL 기록

### 4단계. 재현성 검증
- 동일 입력 기준 재실행
- `generatedAt`, `git.branch`, `git.commit`, `git.dirty` 보존 확인
- 릴리즈 리포트 보관

### 5단계. 운영성과 검증
- 고위험군 후속조치 연결률
- 수동 수정률/번복률
- 비용 절감률
- 품질 게이트 유지 여부

---

## 5) 운영 대시보드에 바로 올릴 신뢰성 KPI

### 필수 KPI
1. `strict8` PASS율
2. 최대 편차
3. 점수 근거 기록률
4. 점수 변경 사유 기록률
5. 수동 수정률
6. 수동 수정 후 번복률
7. 고위험군 후속조치 연결률
8. 비용 절감률
9. `verify:release` 통과율

### 권장 보조 KPI
- 전문가 합치율
- 근로자 이의제기율
- 공종별 편차 상위군
- 국적별 설명 누락률

---

## 6) 신뢰성 판정표(간단)

| 항목 | 기준 | 상태 |
| --- | --- | --- |
| 설명 가능성 | 근거 없는 점수 0건 | PASS / FAIL |
| 일관성 | `strict8(±8)` 통과 | PASS / FAIL |
| 재현성 | 동일 입력 재실행 동일 결과 | PASS / FAIL |
| 감사 가능성 | 변경 사유/감사로그 100% | PASS / FAIL |
| 운영 성과 | 비용/품질/재현성 3축 관리 | PASS / FAIL |

**최종 판정 원칙**
- 5개 중 5개 PASS: 대외 설명 가능
- 5개 중 4개 PASS: 내부 운영 가능, 대외 설명은 조건부
- 3개 이하 PASS: 규칙 고정 보류, 보정 후 재검증

---

## 7) 역할별 설명 문구

### 근로자용
- 이 평가는 단어 개수로 채점하지 않고, 작업 맥락과 실행 행동 기준으로 판단합니다.
- 오타나 문장 정리만으로 점수가 크게 바뀌지 않도록 설계되어 있습니다.
- 점수 변경이 발생하면 이유가 기록됩니다.

### 평가자용
- 동일 작업 맥락은 `strict8(±8)` 기준으로 편차를 통제합니다.
- 모든 점수는 앵커 근거와 함께 검토하며, 변경 시 감사로그를 남깁니다.
- 유사 사례 편차와 예외 변경 건은 리포트로 추적 가능합니다.

### 경영자용
- 이 규칙은 비용 절감, 품질 유지, 재현성 확보의 3축으로 운영됩니다.
- 릴리즈마다 실행 시각과 커밋이 남아 외부 검증 가능성을 확보합니다.
- 규칙 신뢰성은 현장 조치 연결률과 운영 효율 개선으로 최종 확인합니다.
