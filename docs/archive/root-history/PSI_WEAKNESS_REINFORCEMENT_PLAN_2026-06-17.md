## 1. 이번 1차 개선 완료 사항

| 구분 | 상태 | 반영 내용 | 기대 효과 |
| --- | --- | --- | --- |
| 동일 근로자 기준 통합 | 완료 | `공종 + 한글관리명 + 국적`을 1순위 식별 기준으로 공용화 | 같은 근로자의 월별 기록이 화면마다 다르게 묶이는 문제 감소 |
| 코드/QR/UUID 보조화 | 완료 | 근로자 코드·QR·UUID는 보조 기준으로 유지 | 현장에서 코드를 기억하지 못해도 누적 추적 가능 |
| 동명이인 오병합 방지 | 완료 | 같은 이름이어도 공종 또는 국적이 다르면 다른 근로자로 분리 | 동명이인과 공종 변경에 따른 오분류 위험 감소 |
| OCR 분석 누적 카드 기준 정비 | 완료 | 근로자별 누적 보기 기준을 공용 식별 기준으로 변경 | JSON 불러오기 후 누적 카드의 일관성 강화 |
| 근로자 히스토리 기준 정비 | 완료 | 히스토리 모달도 같은 식별 기준 사용 | 상세 기록·월별 기록·누적 보기의 기준 통일 |
| 대시보드 집계 기준 정비 | 완료 | 대시보드 근로자 집계도 같은 식별 기준 사용 | 심사·보고 시 화면별 숫자 불일치 위험 감소 |
| 실증 증빙 준비도 패널 | 완료 | OCR 분석 화면에 비식별 집계 패널 추가 | 전체 기록, 관리 단위, 다월 추적, 개선/하락군, 증빙 보유율을 화면에서 설명 가능 |
| 날짜 이상치 경고 | 완료 | 미래일자·날짜 해석 실패 건수 표시 | 월별 추적 왜곡을 관리자 확인 대상으로 분리 |
| 기술능력자료 업데이트 | 완료 | 약점 섹션을 “약점과 실질 보완 방향”으로 재작성 | 심사자에게 약점 관리 능력과 개선 흐름을 함께 제시 |
| 검증 테스트 추가 | 완료 | 근로자 식별 기준 테스트 3건 추가 | 향후 수정 시 동일성 기준이 깨지는 것을 자동 감지 |

## 2. 아직 남은 핵심 보완

| 우선순위 | 보완 항목 | 이유 | 추천 방식 |
| --- | --- | --- | --- |
| 1 | 백업 JSON 전용 불러오기/실증 요약 화면 강화 | 실제 실증자료가 앱 화면에 바로 보이지 않으면 심사 설득력이 약함 | “실증자료 불러오기 → 비식별 요약 → 누적 보기 자동 갱신” 흐름으로 별도 버튼 구성 |
| 2 | 국적·언어·공종 표준 사전 완성 | 표기가 흔들리면 집계 신뢰도가 떨어짐 | 대한민국/한국/KO, vi/Vietnamese/베트남어 등 표준 매핑 테이블화 |
| 3 | 공개용 데모 데이터셋 생성 | 원본 실명·서명·이미지를 그대로 보여줄 수 없음 | 이름 A-001, B-014 방식 마스킹, 원본 이미지 흐림 처리 |
| 4 | 실증 캡처 패키지 구성 | 공모 제출 시 “말”보다 “화면 증거”가 강함 | 실증 요약, 누적 추적, 상세 판단, 다국어 TBM 환류 4장 캡처 |
| 5 | 3개월 파일럿 KPI 설계 | 사고 감소율은 아직 주장하기 어려움 | 저점군 감소율, 하락군 재교육 후 회복률, TBM 참여율, 재작성률 추적 |

## 3. 현재 주장 가능한 문장

NEW-PSI는 현재 단계에서 “사고 감소율을 입증한 시스템”이라고 말하기보다는, 아래 문장으로 제안하는 것이 정확하고 강하다.

> NEW-PSI는 실제 월별 현장 수기 위험성평가 기록을 AI로 구조화하여, 근로자 관리 단위별 위험 인식 변화와 교육 우선순위를 추적하고 다음 달 다국어 TBM 교육으로 환류하는 현장 실증형 안전 인텔리전스 시스템이다.
