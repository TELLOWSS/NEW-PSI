## 1) 문서 목적

이 문서는 두 가지를 한 번에 해결하기 위한 실무 문서다.

1. **OCR 재분석 개선 적용 후**, 운영자가 현장에서 바로 확인할 수 있는 **전/후 비교 점검표** 제공
2. PSI 프로그램의 **아이덴티티, 작동원리, 보강 필요사항, 지향점, 핵심 가치**를 공통 기준으로 정리

## 2) 재분석 전/후 즉시 점검표 (운영용 5개 항목)

### 2.1 사용 방법 (5분)

1. 동일한 성격의 샘플(정상 스캔 이미지 기준) 3~5건을 선정
2. 개선 적용 전 기준값(또는 직전 배치 기록) 확보
3. 개선 적용 후 동일 조건으로 재분석 실행
4. 아래 5개 항목을 같은 시점/같은 배치 기준으로 기록
5. 증감과 비율을 보고 즉시 판정

### 2.2 점검 항목 표

| No | 항목 | 확인 위치(화면/집계) | 계산식 | 기대 방향 |
|---|---|---|---|---|
| 1 | 서버 성공 건수 | 재분석 집계 카드 `서버 성공` | 절대건수 | **증가** |
| 2 | OCR 처리 실패 건수 | 재분석 집계 카드 `OCR 처리 실패` | 절대건수 | **감소** |
| 3 | 서버 라우트 실패 건수 | 재분석 집계 카드 `서버 라우트 실패` | 절대건수 | **감소** |
| 4 | 사전 검증 실패 건수 | 재분석 집계 카드 `사전 검증 실패` | 절대건수 | **유지/감소** |
| 5 | UNKNOWN 비중 | 실패 코드 분포(`failedFailureCodeSummary`) | `UNKNOWN 건수 / 전체 실패 건수 × 100` | **감소** |

### 2.3 운영 판정 기준 (권장)

- **정상 개선 판정**
  - 서버 성공 ↑
  - OCR 처리 실패 ↓
  - UNKNOWN 비중 ↓
- **주의 판정**
  - 서버 성공 변화 없음 + OCR 처리 실패 정체
- **즉시 원인분석 필요**
  - 서버 라우트 실패 급증 또는 UNKNOWN 비중 급증

### 2.4 현장 기록 템플릿 (복붙)

## 3) PSI 프로그램 아이덴티티 분석

## 3.1 한 줄 정체성

**PSI는 "현장 위험 신호를 사람 중심 언어로 번역해, 관리자 의사결정과 근로자 행동 전환을 동시에 만드는 안전 운영 시스템"이다.**

## 3.2 정체성의 구성 요소

- **안전 AI 엔진**: OCR + 해석 + 점수화 + 코칭
- **운영 워크플로우**: 재분석, 승인/보류, 근거 기록, 이력 추적
- **역할 기반 전달**: 근로자/관리자/경영진에게 다른 표현으로 같은 사실 전달
- **감사/증빙 지향**: 기록의 연속성과 재현성을 운영 자산으로 축적

## 3.3 PSI가 단순 OCR 도구가 아닌 이유

- OCR 결과를 그대로 보여주지 않고 **판단 가능한 상태(확인 필요/추가 확인/보완 검토)**로 변환
- 오류를 단일 실패로 끝내지 않고 **실패 코드 체계(QUOTA/KEY/FORMAT/PARSE/PAYLOAD/NETWORK/UNKNOWN)**로 운영 가능 상태로 환원
- 결과를 점수로만 끝내지 않고 **다음 행동**(재분석, 관리자 정상분류, 사유 보강)으로 이어지게 설계

## 4) PSI 작동원리 분석 (운영 관점)

## 4.1 입력 → 해석 → 판정 → 조치의 4단 구조

1. **입력 수집**
   - 이미지/문서 데이터 수신, 포맷/무결성 검증
2. **해석/구조화**
   - OCR/모델 응답을 WorkerRecord 구조로 정규화
3. **판정/분류**
   - 신뢰도, 실패 코드, 오류 타입, 워크플로우 상태를 결합해 운영 상태화
4. **조치 실행**
   - 스마트/직접 재분석, 관리자 정상분류, 승인/보류, 이력 저장

## 4.2 핵심 설계 철학

- **모델 성능만 믿지 않는다**: 실패를 코드로 표준화해 운영 가능성을 확보
- **실패도 데이터다**: 실패코드 분포와 재분석 회복률을 성능 지표로 사용
- **사람이 최종 책임을 가진다**: 자동 처리 + 관리자 판단을 분리/연결

## 4.3 최근 개선의 의미 (2026-04-17 반영)

- 실패 판정이 단어 매칭 중심에서 **운영 장애 신호 중심**으로 정밀화됨
- 점수 누락 응답의 과도한 실패 전이를 완화하도록 **서버 보정 로직** 추가
- 효과: 정상 스캔에 대한 오탐 실패 누적 가능성 완화

## 5) 보강해야 할 사항 (정확도/운영성 기준)

## 5.1 P0 (즉시)

1. **재분석 결과 추적성 강화 (trace 표준 필드 저장)**
   - 필요 필드: providerUsed, fallbackUsed, latencyMs, attemptCount
   - 이유: 실패 원인 재현/분석 시간 단축

2. **UNKNOWN 세분화 룰 추가**
   - UNKNOWN을 바로 종결하지 말고 2차 분류(네트워크 의심/응답 품질/미분류)
   - 이유: 운영 조치 정확도 향상

3. **운영 대시보드에 전/후 비교 고정 블록 추가**
   - 서버 성공/처리 실패/UNKNOWN 비중의 최근 24h 비교
   - 이유: 개선 효과를 즉시 확인 가능

## 5.2 P1 (단기)

1. **Hybrid OCR 체인(`ocr.hybrid`) 단계적 전환**
   - local_ocr+gemma → server_gemini → client_gemini
   - 이유: 비용/속도/성공률 균형

2. **실패코드별 자동 가이드 정밀화**
   - 코드별 권장 조치 문구를 현장 실행 단계로 세분화
   - 이유: 관리자 피로도 감소

3. **테스트 시나리오 고정 세트 운영**
   - 정상 스캔/경계 스캔/손상 스캔 3세트
   - 이유: 회귀(Regression) 조기 탐지

## 5.3 P2 (중기)

1. **운영 정책 엔진화**
   - 임계값/자동전이/예외승인 룰을 설정화
2. **감사 리포트 자동화**
   - 월간 실패코드 추세 + 개선조치 이행률 자동 생성
3. **역할별 UI 최소행동 원칙 강화**
   - 관리자: 3클릭 내 조치 완료
   - 근로자: 이해 중심 1스크린 요약

## 6) PSI의 지향점 (North Star)

1. **정확한 판독**: 문서 인식 정확도 향상
2. **낮은 운영 피로**: 실패를 조치 가능한 정보로 변환
3. **높은 재현성**: 같은 입력에 일관된 판단/이력 보장
4. **사람 중심 안전성**: 비난이 아닌 보호/보완 중심
5. **감사 가능성**: 법규·내부 감사 대응 가능한 기록 체계

---

## 7) PSI의 핵심 가치 정리

- **현장 실행성**: 보기 좋은 리포트보다 바로 쓸 수 있는 조치 우선
- **운영 신뢰성**: 성공뿐 아니라 실패를 관리 가능한 체계로 축적
- **사람 보호 관점**: 점수보다 행동 전환을 촉진
- **조직 학습성**: 누적 데이터로 재발 방지/정책 개선을 가능하게 함

---

## 8) 최종 결론

PSI는 "OCR 앱"이 아니라 **현장 안전 의사결정 운영체계**다.  
이번 개선으로 정상 스캔의 오탐 실패 전이를 줄일 기반이 만들어졌고, 다음 단계는 **UNKNOWN 축소 + trace 표준화 + 전/후 비교의 상시 운영화**다.  
이 3가지를 고정 루틴으로 만들면 PSI의 가치는 "분석 결과"를 넘어 "현장 위험을 줄이는 실행 체계"로 더 선명해진다.
