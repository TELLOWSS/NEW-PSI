## 2️⃣ 문제점 1: 정보 혼잡도 (너무 다양한 정보)

### 2.1 현상
**현재 상태:**
```
📁 프로젝트 루트
├─ 📄 MOBILE_12SCREEN_EXECUTION_PLAN_2026-05-16.md (기획, 기술 혼합)
├─ 📄 MOBILE_MOCKUP_REALITY_AUDIT_2026-05-20.md (감사, 상태 보고)
├─ 📄 IMPLEMENTATION_MASTER_PLAN_2026-05-21.md (마스터 계획)
├─ 📄 END_OF_DAY_HANDOFF_2026-05-24.md (일일 인수인계)
├─ 📄 ADMIN_OPERATION_GUIDE.md (운영 가이드)
├─ 📄 APP_MENU_RESTRUCTURE_PROPOSAL_2026-05-04.md (메뉴 재구성)
├─ 📄 ... (80+ 추가 문서)
└─ 💻 /pages, /components (코드)
```

**문제:**
- 📌 같은 정보를 여러 문서에서 반복 설명
- 📌 계획, 검증, 로그, 가이드가 분류 없이 혼합
- 📌 최신 상태와 과거 상태 문서가 구분 안 됨
- 📌 개발자가 "어디서부터 읽어야 하는지" 불명확
- 📌 실무자는 마크다운 기술 문서만 봐야 함

### 2.2 근본 원인
| 원인 | 영향 |
|-----|------|
| **문서 관리 체계 부재** | 각 세션마다 새 문서 생성 → 누적 |
| **정보 계층 미구분** | 마스터 정보 vs 일일 로그 vs 코드 주석 혼합 |
| **역할별 뷰 부재** | 모든 정보가 동등 수준 |
| **아카이빙 정책 부재** | 완료된 정보가 남아 있음 |

### 2.3 체감 영향
```
개발자: "다 읽어야 하나?" → 생산성 ⬇️
실무자: "기술 내용까지 봐야 하는 건가?" → 혼란 ⬆️
관리자: "현재 상태가 뭐지?" → 명확성 ⬇️
```

## 3️⃣ 문제점 2: 정보 혼재 (개발자 vs 실무자)

### 3.1 현상
같은 문서 내에 **3가지 정보가 섞여 있음:**

**예: MOBILE_12SCREEN_EXECUTION_PLAN_2026-05-16.md**
```markdown
# 개발자 정보
- [pages/WorkerTraining.tsx] ✅ 9번: 입력 진행도
- npm run check:judgment-tagging:full
- 의존성 재설치, 컴파일 오류 0

# 실무자 정보
- 12화면 IA 정렬안 (홈/경보/프로파일/진단/컨텍스트...)
- 화면당 1핵심 행동 (CTA 고정 원칙)
- 현장 사용성 최적화

# 설계 정보
- 모바일 우선(320x568 ~ 390x844)
- 위험 점수보다 이유 우선 (UI 원칙)
- 기존 데이터 자산 100% 재사용
```

**문제:**
- 개발자는 기술 내용에만 집중 필요 → 다른 내용 스킵
- 실무자는 기술 용어가 방해 → 이해 저하
- 설계자는 세 정보를 모두 조율 필요 → 부담 ⬆️

### 3.2 각 역할이 알아야 할 정보

| 역할 | 필요 정보 | 예시 | 현재 상태 |
|-----|---------|------|---------|
| **개발자** | 구현 스펙, 코드 위치, 로직, 의존성 | `pages/WorkerTraining.tsx`, 태입 입력 필드 구조 | ✅ 코드 주석 OK, 문서 미분류 |
| **실무자** | 사용 가이드, 화면 설명, 행동 흐름 | 9번 화면에서 어떻게 입력? 완료 버튼은? | ❌ 기술 문서만 있음 |
| **설계자** | 목업, IA, UX 원칙, 컴포넌트 구조 | 모바일 12화면 정보구조, CTA 위치 | 🔄 부분 혼합됨 |
| **PM/운영** | 일정, 상태, 리스크, 다음 액션 | 어디까지 완료? 다음은? | ✅ 일부 명확 |

### 3.3 현재 정보 흐름 (문제)
```
❌ 현재 흐름:
코드(TypeScript) ← → 기술 문서(기술자)
                  ↓
          통합 작업 문서
          (모든 정보 혼합)
                  ↓
           실무자/PM (혼란)
```

**개선 흐름:**
```
✅ 개선 흐름:
코드 ──→ [개발자 문서] ──→ 개발자
 ↓
[설계 문서] ──→ 설계자/PM
 ↓
[사용자 가이드] ──→ 실무자
 ↓
[상태 리포트] ──→ 관리자
```

## 4️⃣ 문제점 3: PC/모바일 목업 미완성

### 4.1 현황: "보이는 것 vs 실제 기능"

**문서: MOBILE_MOCKUP_REALITY_AUDIT_2026-05-20 (공식 감사 보고서)**
```
핵심 결론:
"현재 반영된 것은 **소개 화면(Introduction)의 목업 프리뷰 레이어**가 중심이다.
즉, '목업처럼 보이는 화면'은 되었지만, 
 각 화면의 기능이 모두 목업 수준으로 완성된 것은 아니다."
```

### 4.2 12화면 구현 상태 (2026-05-20 기준)

| 번호 | 화면명 | 현재 상태 | 문제점 |
|-----|------|---------|-------|
| 1 | 홈 대시보드 | 🟠 부분완료 | 기존 대시보드 기반 (목업 UX 일부만 반영) |
| 2 | 경보 알림 | 🟠 부분완료 | 목업 전용 경보 UX와 차이 가능 |
| 3 | 개인인지 프로파일 | 🟠 부분완료 | 프로파일 관점 정렬 필요 |
| 4 | 위험인지 진단 | 🟠 부분완료 | 진단 전용 모바일 UX 보강 필요 |
| 5 | 현장 컨텍스트 | 🟡 신규/보강 | 데이터 검증·저장 플로우 미완성 |
| 6 | 행동 패턴 분석 | 🟠 부분완료 | 모바일 요약 카드 강화 필요 |
| 7 | 위험 예측 | 🟠 부분완료 | 액션 연결(8번) 고도화 필요 |
| **8** | **개입 추천** | **🟡 신규/보강** | **실무용 UX 미완성 (P0 우선)** |
| **9** | **수기 데이터 입력** | **🟡 신규/보강** | **입력 검증 메시지 미완성 (P0)** |
| 10 | 태깅 검증 | ✅ 거의완료 | QA 표시·동선 안정적 |
| 11 | 분석 리포트 | ✅ 거의완료 | CTA 로그 동기화는 SQL 적용 후 QA 필요 |
| 12 | 메뉴/설정 | 🟠 부분완료 | 모바일 전용 설정 UX 정리 필요 |

**현황:**
- ✅ 2/12 화면 거의 완료 (8개: 태깅, 검증, 리포트)
- 🟠 7/12 화면 부분 완료 (기존 페이지 기반)
- 🟡 3/12 화면 신규 미완성 (5, 8, 9번)

### 4.3 PC 화면 현황
| 영역 | 의도 | 현재 상태 |
|-----|------|---------|
| **PC 대시보드** | 운영자 종합 현황 | 🟠 기본 정보만 (고급 기능 부족) |
| **PC 분석 섹션** | 오류 관리·QA 중심 | ✅ 비교적 완성 |
| **PC 리포트** | 경영진 보고 중심 | ✅ PDF 내보내기 완성 |
| **PC 설정** | 시스템 설정 | 🟠 기본 UI만 |
| **모바일과 구분** | PC vs Mobile 이원화 | 🟡 프리뷰만 분리, 기능은 미분리 |

**핵심 문제:**
- 모바일 12화면과 PC가 "같은 코드베이스"를 공유
- PC의 역할(운영/관리/보고) vs 모바일의 역할(현장 행동)이 UI에 명확히 구분 안 됨
- 목업 프리뷰는 분리했으나, 실제 구현은 여전히 혼합

## 5️⃣ 핵심 원인 분석

### 5.1 왜 이런 일이 발생했나?

| 원인 | 설명 |
|-----|------|
| **스코프 팽창** | 100+ 개의 새 기능·검증·최적화를 동시 진행 |
| **기술 자산 재사용** | 기존 DashboardPages를 "그대로" 모바일로 재사용 시도 |
| **문서와 코드 분리** | 설계 목표(문서)와 구현(코드)이 별도로 진행됨 |
| **역할 혼재** | 개발자, 설계자, PM이 같은 마크다운 파일에서 협업 |
| **우선순위 재정의 부재** | 모바일 우선? PC 우선? 불명확하게 시작 |

### 5.2 결과
- 📌 코드는 기술적으로 완성되었으나, UX는 "목업과 다름"
- 📌 문서는 풍부하나, 정보 계층이 없어서 "혼란스러움"
- 📌 팀원들이 "지금 뭘 해야 하는지" 불명확함
