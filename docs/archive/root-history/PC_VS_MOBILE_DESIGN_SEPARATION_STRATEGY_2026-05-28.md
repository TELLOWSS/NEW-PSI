# 📱 PC vs 모바일 UI/UX 분리 설계안 (2026-05-28)
**목적**: PC와 모바일의 역할을 명확히 하고, 구현 가능한 디자인 분리 전략 제시  
**상태**: 구현 준비 완료  
**기간**: 2-3주 예상  

---

## 🎯 핵심 전략

### 현재 문제
```
❌ 현재:
같은 컴포넌트 → PC 축소 → "모바일처럼 보이는 화면"
                        (기능은 여전히 PC 중심)

✅ 개선:
모바일 화면 → 모바일 로직 → 모바일 UX
+ 
PC 화면    → PC 로직    → PC UX
(별도 라우팅 통해 분리)
```

### 설계 원칙
1. **모바일은 행동 중심** (Do: 한 가지 일)
2. **PC는 운영 중심** (Monitor: 많은 정보, 관리 기능)
3. **코드는 가능한 공유** (데이터 모델, API, 비즈니스 로직)
4. **UI만 분리** (라우팅, 레이아웃, 정보 계층)

```tsx
// App.tsx (현재)

  <Route path="/dashboard" component={Dashboard} />        ← PC용
  <Route path="/ocr-analysis" component={OcrAnalysis} />   ← PC용
  <Route path="/worker-management" component={...} />      ← PC용

// 모바일 컨텍스트에서도 같은 컴포넌트

```

### 개선 구조 (제안)
```tsx
// App.tsx (개선)

// 1. 단말 감지
const isMobile = useMediaQuery('(max-width: 768px)');

// 2. 라우팅 분리
<Switch>

    // 모바일 라우팅 (12화면)

      <Route path="/analysis" component={PCAnalysis} />
      <Route path="/operations" component={PCOperations} />
      ...
    </>
  )}

```

## 📱 3. 모바일 구체 설계 (12화면)

### 원칙
- **화면 1개 = 행동 1개**
- **상단 고정 CTA 1개** (핵심 버튼)
- **하단 글로벌 탭 바**
- **정보 3초 이해**

### 7. 위험 예측 → 8. 개입 추천 예시

#### 모바일 7번 화면 (위험 예측)
```
┌─────────────────────────────┐
│ [이전]              [메뉴]  │  ← 헤더
├─────────────────────────────┤
│                             │
│ 📊 위험 예측                │
│    김○○ (ID: W-0123)       │
│                             │
├─────────────────────────────┤
│  [ 고위험 ]  ⛔ 68%         │  ← 큰 점수 (1순위)
│  다음 2주 내 사건 가능     │
│  근로자: 안전교육 필요      │
│                             │
├─────────────────────────────┤
│ [근거 보기] [개입하기]      │  ← CTA 2개
│                             │
├─────────────────────────────┤
```

**특징:**
- 점수 큼 + 간단한 문장
- "개입하기" 버튼 명확
- 다음 화면으로 1탭

#### 모바일 8번 화면 (개입 추천) ← **[P0 우선 완성]**
```

│                             │
```

**P0 체크리스트 (이번 주 완성):**
- [ ] "지정하기" 클릭 → 상태 변경 (미착수 → 진행중)
- [ ] "완료" 클릭 → 상태 변경 (진행중 → 완료)
- [ ] 상태별 배경색 변경 (빨강/노랑/초록)
- [ ] 데이터 저장 확인
- [ ] 화면 8번에서 화면 1번으로 돌아가면 "개입 0건" → "개입 1건" 카운트 증가

## 🖥️ 4. PC 운영 화면 구체 설계

### 원칙
- **좌측 네비 (고정)** + 주요 콘텐츠
- **정보 밀도 높음** (표, 차트, 다중 카드)
- **배치/조회 중심**
- **관리 기능 노출**

### PC 대시보드 예시
```
┌──────────────────────────────────────────────────────────┐
│ [SideBar] 🏢 PSI                    [사용자] [로그아웃]  │

│ 🔍 분석  │  ┌────────┬────────┬────────┐               │
│ ⚠️ 경보  │  │ 전체   │ 위험   │ 안전   │  (좌측 정보백) │
│ 📋 리포트 │  │ 245명  │ 31명   │ 214명  │  (오른쪽: 차트)│
│ ⚙️ 설정  │  └────────┴────────┴────────┘               │
```

- 좌측 메뉴: 큰 영역 (관리자/일반사용자 드롭다운)

- 색상은 모바일과 동일 (브랜드 일관성)
