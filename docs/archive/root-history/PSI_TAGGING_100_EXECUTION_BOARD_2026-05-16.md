# PSI 100건 태깅 실행보드 (20건 × 5라운드)

- 작성일: 2026-05-16
- 목적: 표본 100건 태깅을 배치 단위로 안정적으로 완료하고, 합의/검증/전조분석까지 끊김 없이 연결한다.

---

## 1) 운영 원칙
- 한 라운드(20건) 완료 전 다음 라운드로 넘어가지 않는다.
- 각 라운드는 `1차 태깅 → 2인 합의 → QA 검증 → 요약 기록` 순서로 종료한다.
- 라운드 종료 시 `check:judgment-tagging:full` 실행 결과를 반드시 남긴다.

---

## 2) 라운드 계획

| Round | 대상 recordId | 목표 | 산출물 |
| --- | --- | --- | --- |
| R1 | 001~020 | 태깅 기준/코드북 적용 안정화 | 1차 태깅 + 합의 + QA 리포트 |
| R2 | 021~040 | 불일치 유형 축소 | 합의 반영 + QA 리포트 |
| R3 | 041~060 | 전조 태그 패턴 가시화 시작 | TOP5/ACTION_TOP5 추세 확인 |
| R4 | 061~080 | 태그 정합성 고정 | 오류 0 목표 검증 |
| R5 | 081~100 | 최종 마감 및 인사이트 추출 | 최종 QA + OPS 3줄 + 전조 후보 10개 |

---

## 3) 라운드별 체크리스트

### 공통 체크 (각 라운드)
- [ ] 1차 태깅(20건) 입력 완료
- [ ] 2인 태깅 불일치 합의 완료
- [ ] 코드북 재정렬 완료(`judgmentTagCodes`)
- [ ] `npm run check:judgment-tagging:full` 실행
- [ ] `reports/judgment-tagging-ops-summary.md` 확인
- [ ] 라운드 완료 메모(완료/다음/검증) 기록

---

## 4) 품질 게이트

- R1~R2: 오류 감소 추세 확보
- R3: `ERROR_TOP5`에서 동일 원인 반복 여부 확인
- R4: 오류 0 또는 핵심 오류 1개 이하
- R5: 최종 `ACTION_TOP5` 기준 개선조치 확정

---

## 5) 라운드 종료 기록 템플릿

- 완료: (이번 라운드에서 끝낸 항목)
- 다음: (다음 라운드 첫 작업)
- 검증: (`check:judgment-tagging:full` 결과)

---

## 6) 최종 성공 조건
- 100건 태깅 입력 완료
- 2인 합의 완료
- 최종 QA 리포트 생성 완료
- OPS 3줄 요약 생성 완료
- 전조 시그널 후보 10개 도출 완료
