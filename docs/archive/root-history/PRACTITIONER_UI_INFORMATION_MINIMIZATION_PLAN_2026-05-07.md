# 실무자 중심 UI 정보 최소화 재설계 계획 (2026-05-07)

## 0) 목적
- 모바일/PC에서 실무자가 **지금 해야 할 일**만 먼저 보이도록 재구성한다.
- 과도한 정보는 제거가 아니라 **기본 숨김 + 필요 시 확장** 구조로 전환한다.
- 기존 기능을 유지하면서, 의사결정 속도와 실행 완료율을 높인다.

---

## 1) 적용 범위
- 핵심 3화면
  1. `Dashboard` (팀/공종 비교 포함)
  2. `OcrAnalysis` (운영 탐색/실패 큐/고급 필터)
  3. `PredictiveAnalysis` (우선 개입/실행계획/공종 조치율)

---

## 2) 정보 노출 원칙 (실무자 모드)
1. 한 화면 1결정: 첫 화면에서 선택 가능한 핵심 행동은 최대 2개
2. 기본 접힘: 분석/설명/히스토리/프리셋/고급설정은 기본 숨김
3. 0건 자동 숨김: 대상이 없으면 카드·패널 자체를 노출하지 않음
4. 우선순위 고정: `긴급 → 오늘 마감 → 진행중 → 참고` 순서 유지
5. 동일 의미 통합: 같은 의미 KPI/카드는 1개로 합침

---

## 3) 화면별 정밀 재설계

## 3-1) Dashboard

### A. 모바일(실무자)

#### 즉시 노출(유지)
- 상단 핵심 KPI 3개(현장지수/활동근로자/고위험)
- 빠른 행동 2개(신규 분석, 리포트 생성)
- 팀 비교 진입용 최소 컨트롤
  - 공종 퀵칩
  - 팀 선택 상태(선택 n/3)

#### 기본 숨김(접기)
- `comparisonCards` 해석 카드 전체
- 프리셋 관리 전체(검색/핀/수정/CSV)
- 고급정렬/고급필터(`isComparisonAdvancedOpen`)
- 팀 내부 국적 드릴다운 설명 박스(문장형 안내)

#### 지연 노출(더보기)
- 팀 비교 상세 카드(점수/위험/미처리/추세)
- 국적 드릴다운 차트(팀 2개 이상 선택 시만 노출)

### B. PC(실무자)

#### 즉시 노출(유지)
- 공종 퀵 접근 + 팀 선택 바
- 메인 비교 테이블(선택 팀 2~3개 기준)

#### 기본 숨김(접기)
- 프리셋 관리 블록
- 해석 카드 2세트(`dashboardSummaryCards`, `operationalFocusCards`)

#### 지연 노출
- `고급 보기` 내부 정렬/필터/국적 보조드릴다운

### C. 구현 규칙
- 공종 변경 시 다음 상태 강제 초기화
  - `selectedTeam = ALL`
  - `selectedTeamsForComparison = []`
- 팀 2개 미만일 때 상세 비교 영역은 비노출 + 1줄 가이드만 표시

---

## 3-2) OcrAnalysis

### A. 모바일(실무자)

#### 즉시 노출(유지)
- 검색 1개 + 핵심 카운터 3개
  - 조회 결과 / 재분석 가능 / 우선 확인
- 우선 조치 버튼 2개
  - 원문 다시 읽기
  - 상세 판단 열기

#### 기본 숨김(접기)
- 관리자 활동 패널 전체
- 사유 품질 QA 상세 전체
- 월간 리스크 인텔리전스 전체
- 2차 AI 재분석 패널 상세(제외사유 분해/미리보기/진단)
- 고급 필터·정렬 패널 전체

#### 지연 노출
- 실패코드 카드 확장(기본 2~3개)
- 최근 재분석 결과 상세 비교

### B. PC(실무자)

#### 즉시 노출(유지)
- 운영 탐색 + 실패 큐 우선 조치
- 실패 우선 정렬, 재분석 상세 열기

#### 기본 숨김(접기)
- 관리자 전용/QA 전용 패널
- 고급 필터 전체(기본 펼침 금지)

#### 지연 노출
- 운영일지/사유품질/리스크 인텔리전스는 "운영 상세" 버튼 뒤로 이동

### C. 구현 규칙
- `showAdvancedOcrControls=false`, `showAdminActivityPanel=false`, `showReasonQaDetailPanel=false`, `showRetryDetailPanel=false`를 실무자 기본값으로 고정
- 핵심 실패 대상이 0건이면 실패 큐 섹션 자체 비노출

---

## 3-3) PredictiveAnalysis

### A. 모바일(실무자)

#### 즉시 노출(유지)
- 위험 점수 원형 + 버킷 3개(위험/주의/안정)
- 우선 개입 대상 리스트(기본 3건)
- 실행 계획(기본 3건, `urgent` 필터)

#### 기본 숨김(접기)
- `predictiveSummaryCards`, `harnessSummaryMetrics` 설명형 카드
- 온톨로지 맵(기본 숨김, 버튼으로 열기)
- 공종별 조치율 카드(모바일 기본 숨김)
- TBM 전파 교육 제안 전체

#### 지연 노출
- 경향 분석: 우선개입 3→5 확장
- AI 인사이트: 실행계획 3→전체 확장
- 중점 관리 안건: 2→전체 확장

### B. PC(실무자)

#### 즉시 노출(유지)
- 우선개입 TOP + 실행계획 + 상태 변경
- 공종별 조치율(상위 저조치 공종만)

#### 기본 숨김(접기)
- 온톨로지 맵 상세 컨트롤(줌/간격)
- 회의용 장문 설명 카드

#### 지연 노출
- 실행계획 히스토리 상세
- 공종별 조치율 전체 펼침

### C. 구현 규칙
- 실무자 기본 상태
  - `executionPlanFilter='urgent'`
  - `showAllRiskInsights=false`
  - `showAllExecutionPlans=false`
  - `showAllMeetingAgenda=false`
- 버튼 클릭 체감 보강
  - 상태칩 표시: `긴급 필터 적용`, `전체 보기 ON`

---

## 4) "나머지 정보" 처리 전략
- 관리자/감사/품질 정보는 "운영 상세" 레이어로 분리
- 실무자 레이어에서는 다음만 유지
  - 오늘 처리 대상
  - 바로 실행 버튼
  - 결과 상태(성공/대기/실패)
- 분석 근거·장문 설명·이력은 Drawer/접힘영역/별도 탭으로 이동

---

## 5) 단계별 실행 계획

### Phase 1 (즉시, 1~2일)
- 기본 접힘 상태 정리
- 모바일 우선 3건 노출 규칙 통일
- 0건 자동 숨김 규칙 적용

### Phase 2 (3~5일)
- Dashboard/OCR/Predictive에 실무자 모드 토글 도입
- 관리자성 패널을 "운영 상세"로 분리
- 버튼 상태칩/행동 피드백 추가

### Phase 3 (1주)
- 사용 로그 기반 미세조정
  - 첫 클릭까지 시간
  - 완료율
  - 확장 버튼 사용률

---

## 6) 검증 체크포인트
1. 모바일 첫 화면에서 핵심 행동 버튼이 2개 이내인지
2. 접힘 없이 노출되는 카드가 3개를 넘지 않는지
3. 0건 섹션이 자동 숨김되는지
4. 공종/팀 전환 시 이전 상태가 잔존하지 않는지
5. 실무자 플로우(조회→판단→조치)가 3탭 이내로 끝나는지

---

## 7) 성공 KPI
- 첫 행동까지 시간(TTA) 30% 단축
- 실무 조치 완료율 20% 향상
- 불필요 스크롤 깊이 25% 감소
- "어디부터 해야 할지 모르겠다" 피드백 50% 감소

---

## 8) 오늘 결론
- 기능 삭제 없이도, **기본 숨김/우선노출/지연확장**만으로 실무자 화면의 과밀 문제를 해소할 수 있다.
- 우선순위는 `Predictive → OcrAnalysis → Dashboard 고급영역` 순으로 적용하는 것이 체감 효과가 가장 크다.
