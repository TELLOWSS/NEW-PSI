## 검사 원칙

- 원본 JSON은 읽기 전용으로 검사하며 수정·삭제·대체하지 않는다.
- 모든 레코드를 스트리밍 검사기로 통과시킨다.
- 원본 JSON, 이미지, 개인정보, 전체 DOM·로그·diff는 대화에 출력하지 않는다.
- 상세 증거는 `artifacts/audit/`에 저장하고 주 스레드에는 통계·핵심 오류·경로만 보고한다.
- 계산 규칙 변경은 데이터와 코드의 근거가 확인된 뒤에만 수행한다.

## 2단계 — 전수검사 프로그램 작성 완료

- 검사기: `scripts/audit-data.mjs`
- 처리 방식: 전체 JSON을 한꺼번에 메모리에 올리지 않고 레코드 단위 스트리밍
- 개인정보 보호: CSV에는 원문 식별자가 아니라 잘린 SHA-256 해시만 기록
- 검사 범위:
  - 전체 건수, 파일 해시, 파일 내부·파일 간 중복
  - 필수값, 날짜, 숫자 범위, enum, 승인·워크플로 참조
  - 근로자 식별키·UUID 연결
  - 점수와 6대 지표, 저장 등급과 기본 임계값의 불일치
  - 백업 세대 간 추가·삭제·변경·동일 레코드
- 검사기 자체 검증:
  - 동일 파일명의 저점수 건수가 누적되던 결함 수정
  - 파일 경로를 고유하게 기록하도록 수정
