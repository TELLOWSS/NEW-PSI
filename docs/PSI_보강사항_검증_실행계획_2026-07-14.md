# PSI 보강사항 확인검증 및 실행계획

작성일: 2026-07-14

## 1. 검증 결론

첨부 보강안의 방향은 타당하다. PSI는 SMART TBM과 별개의 프로그램으로 정리해야 하며, PSI의 고유 영역은 다음 흐름이다.

```text
근로자 자기기입 위험성평가
→ OCR·AI 분석
→ 문항 간 무결성 검증
→ 관리자 확인·수정·승인
→ 공식 리포트와 월별 위험분석
→ 원페이지 위험성평가 교육자료 생성
→ 다음 달 반복위험 변화 확인
```

현재 프로그램은 이 방향으로 상당 부분 구현되어 있다. 특히 OCR 분석, 관리자 검토, 모국어 보호 안내 품질 확인, 반복위반 패널티 조건, 원페이지 교육자료 흐름은 이미 기반이 있다. 다만 대외 설명과 일부 화면·프롬프트에 `TBM` 표현이 남아 있어, PSI가 SMART TBM 또는 별도 TBM 운영 시스템처럼 오해될 수 있다. 상품화 단계에서는 이 표현 정리가 1순위다.

## 3. 보강 우선순위

### P0. PSI 경계 정리와 TBM 표현 제거

목표는 PSI가 SMART TBM과 섞여 보이지 않게 만드는 것이다.

작업 항목:

1. 사용자 화면의 `TBM` 표현을 `위험성평가 전파교육`, `원페이지 위험성평가 교육자료`, `다음 달 교육 환류`로 변경한다.
2. `A4EducationMaterial`의 언어별 제목에서 `PSI TBM Safety Guide` 계열 문구를 제거한다.
3. AI 프롬프트의 `TBM 환류`를 `위험성평가 교육자료 환류`로 변경한다.
4. `routeMeta`의 교육자료 productGroup과 developerLabel에서 TBM 중심 표현을 걷어낸다.
5. 공모전·대외 제출 자료의 핵심 문장을 다음 기준으로 통일한다.

권장 대외 문장:

> NEW-PSI는 건설현장의 수기 위험성평가 기록지를 OCR·AI로 5문항·6지표 구조화하고, 관리자 검증을 거쳐 원페이지 교육자료와 월별 추적관리로 환류하는 현장형 위험성평가 운영 솔루션입니다.

통과 기준:

```text
사용자 화면, 제품 소개, AI 프롬프트, 제출 문서에서
SMART TBM 또는 TBM 운영 시스템으로 오해될 표현이 없어야 한다.
```

예외:

내부 호환 라우트명, 과거 파일명, 번역 품질 검사에서 금지어로 탐지하는 `TBM` 문자열은 남길 수 있다.

### P1. 기록지 기준데이터 단일화

목표는 종이 기록지, 모바일 입력, PDF 출력, OCR 프롬프트, 교육자료가 모두 같은 질문과 선택지를 쓰게 하는 것이다.

작업 항목:

1. `PSI 기록지 기준데이터`를 만든다.
2. Q1~Q5 질문, 작성방법, 예시, 위험등급, 공종, 위험유형, 안전조치, 하단 안내문을 하나의 데이터로 관리한다.
3. 언어별 번역은 기준데이터에 연결하고, 승인 상태를 둔다.
4. 양식 PDF와 모바일 화면은 기준데이터에서 생성되도록 한다.

필수 관리 항목:

| 항목 | 필요 이유 |
---|---|
| 원문 버전 | 질문 수정 이력 추적 |
| 언어별 번역 버전 | 국가별 양식 불일치 방지 |
| 검수 상태 | 미검수 번역 사용 방지 |
| 검수자·수정일 | 대외 증빙 |
| 출력 미리보기 | 글자 넘침·줄바꿈 오류 확인 |
| 모바일 미리보기 | 휴대폰 잘림 방지 |

통과 기준:

```text
질문 하나를 수정하면 PDF, 모바일, OCR 프롬프트, 교육자료 원문에 동일하게 반영되어야 한다.
```

### P1. 문항 간 무결성 검증 강화

목표는 AI가 답을 요약하는 수준을 넘어, 기록지의 앞뒤 맥락이 맞는지 확인하는 것이다.

검증 규칙:

| 검증 항목 | 예시 | 판정 |
|---|---|---|
| Q1 작업과 공종 일치 | 철근공인데 Q1이 도장작업 | 확인 필요 |
| Q2 위험요소와 Q1 작업 연결 | 고소작업인데 소음만 작성 | 확인 필요 |
| Q3 상등급 이유 누락 | 위험등급 상, 이유 없음 | 확인 필요 |
| Q4 감소대책과 위험요소 연결 | 추락 위험인데 귀마개 착용만 작성 | 확인 필요 |
| Q5 실천행동 구체성 | 안전하게 하겠다 | 보강 필요 |
| OCR 원문 신뢰도 | 필기 답변과 추출문 차이 큼 | 원문 재확인 |

화면 표시 방식:

```text
검증결과: 확인 필요

사유
- Q3 위험등급은 상이지만 이유가 부족합니다.
- Q4 감소대책이 Q2 추락 위험과 직접 연결되지 않습니다.
- 원본 이미지 확인 후 관리자 판단이 필요합니다.
```

통과 기준:

```text
AI 결과는 바로 공식 결과가 아니라,
관리자 승인 전 검증 사유와 원문 대조 근거를 함께 보여줘야 한다.
```

### P1. 관리자 수정과 모국어 안내 동기화

목표는 관리자가 한국어 보호 해석을 바꾸면 외국인 근로자에게 전달될 모국어 안내도 반드시 같이 갱신되게 하는 것이다.

현재 프로그램에는 모국어 안내 갱신 필요, 품질 확인, 준비 완료 판정이 이미 있다. 다음 보강은 이 흐름을 더 명확하게 만드는 것이다.

작업 항목:

1. 관리자 수정 전후 차이를 한 화면에서 보여준다.
2. 한국어 보호 해석이 바뀐 경우 기존 외국어 번역을 자동으로 `갱신 필요` 상태로 바꾼다.
3. 승인 버튼은 모국어 안내가 준비되기 전에는 제한한다.
4. 언어별 번역 결과에 한글 혼입, 영문 시스템 용어, 긴 문장, 관리자 평가 문체가 있으면 경고한다.
5. 최종 승인 시 한국어 원문과 언어별 안내문을 같은 승인 묶음으로 저장한다.

통과 기준:

```text
관리자 수정 후 외국어 안내가 옛 내용으로 남아 있는 상태에서
최종 승인·출력이 되면 안 된다.
```

### P2. 6대 지표의 대외 표현과 산정근거 보강

현재 내부 지표는 다음 구조다.

| 내부 지표 | 현재 의미 |
---|---|
| 응답 충실도 | 답변이 성의 있고 문장으로 작성되었는지 |
| 업무이해도 | 공종·작업·자재·도구를 이해했는지 |
| 위험성평가 이해도 | 위험요소와 작업을 연결했는지 |
| 숙련도 | 실효성 있는 대책을 썼는지 |
| 개선이행도 | 구체적 실행계획이 있는지 |
| 반복위반 패널티 | 다음 달 동일 위험 재발·약속 불이행 확인 시 감점 |

첨부안의 `위험인지도, 교육이해도, 실천참여도, 보호구·수칙준수, 소통·보고성, 작업중지 감수성`은 상품 설명력은 좋지만, 현재 기록지 Q1~Q5만으로 모두 직접 측정한다고 말하면 과장될 수 있다. 특히 `교육이해도`, `보호구·수칙준수`, `작업중지 감수성`은 교육 이수, 현장 확인, 작업중지 응답 데이터가 함께 있어야 더 정확하다.

따라서 1차 보강은 내부 계산식을 크게 뒤집지 않고, 대외 표현을 다음처럼 정리하는 것이 적절하다.

| 대외 표시명 | 내부 근거 |
---|---|
| 기록 충실 신호 | 응답 충실도 |
| 작업 이해 신호 | 업무이해도 |
| 위험 인식 신호 | 위험성평가 이해도 |
| 예방조치 구체성 | 숙련도 |
| 실천 약속 명확성 | 개선이행도 |
| 반복위험 변화 | 반복위반 패널티와 월별 추적 |

각 지표에는 반드시 근거를 붙인다.

예시:

```text
위험 인식 신호 78점

근거
- 작업과 위험요소를 정확히 연결한 응답: 31건
- 위험등급 이유를 구체적으로 작성한 응답: 24건
- 추상적이거나 불명확한 응답: 9건
- 문항 간 불일치 응답: 4건

관리자 해석
단부·고소작업의 추락위험 인식은 높지만,
구체적인 발생상황과 원인 설명은 보강이 필요합니다.
```

통과 기준:

```text
점수만 보이는 화면을 줄이고,
점수 옆에 반영 문항, 근거 응답 수, 신뢰도, 전월 대비 변화가 같이 보여야 한다.
```

### P2. 위험유형 표준분류 구축

목표는 근로자의 자유서술을 사고예방에 쓸 수 있는 구조화 데이터로 바꾸는 것이다.

기본 위험유형:

1. 추락
2. 낙하·비래
3. 깔림
4. 끼임
5. 무너짐
6. 부딪힘
7. 감전
8. 화재·폭발
9. 베임·절단
10. 전도
11. 질식
12. 온열질환

구조화 예시:

```text
원문: 단부 이동 중 안전고리를 걸기 어렵다.

공종: 형틀
작업: 단부 이동
위험유형: 추락
위험상황: 단부 이동 중 안전대 체결 곤란
직접원인: 체결 지점 부족
관리요인: 생명줄·부착설비 미흡
근로자 조치: 이동 전 안전고리 체결 확인
필요조치: 생명줄 또는 체결 지점 추가
위험등급: 상
```

통과 기준:

```text
원문은 그대로 보존하고,
분석용 표준분류는 별도 필드로 저장해야 한다.
```

### P2. 월별 안전계도 환류 증빙 강화

목표는 PSI가 분석에서 끝나지 않고 다음 달 교육과 현장조치로 이어졌다는 것을 보여주는 것이다.

환류 흐름:

```text
이번 달 기록 분석
→ 반복 위험언어 탐지
→ 공종·위치·언어별 묶음
→ 관리자 확인
→ 원페이지 교육자료 생성
→ 현장조치 기록
→ 다음 달 동일 위험 변화 확인
```

보고서 필수 항목:

| 항목 | 목적 |
---|---|
| 현장명·교육월 | 공식 문서성 |
| 분석대상 인원·작성 완료율 | 실증 규모 |
| 언어별 참여현황 | 외국인 근로자 대응 |
| 주요 위험유형·상등급 위험 | 사고예방 신호 |
| 반복 위험 | 환류 대상 |
| 6대 지표와 산정근거 | 설명 가능성 |
| 관리자 검토결과 | AI 단독판정 방지 |
| 교육 반영내용 | 다음 달 전파교육 근거 |
| 현장 조치내용 | 실제 개선 연결 |
| 전월 대비 변화 | 추적관리 증빙 |
| AI 분석일·관리자 승인일·문서번호 | 감사 대응 |
| 수정이력 | 무결성 증빙 |

통과 기준:

```text
익명 대외 보고서와 내부 확인용 기명 보고서를 구분해서 출력할 수 있어야 한다.
```

### P3. 근로자 모바일 작성기능

종이 양식은 유지하되, 모바일 작성은 별도 최적화 흐름으로 가야 한다.

권장 흐름:

1. 언어 선택
2. 공종 선택
3. 위험한 작업 선택 또는 직접 입력
4. 위험요소 선택
5. 위험등급 선택
6. 위험하다고 생각한 이유 작성
7. 내가 할 수 있는 조치 선택
8. 실천약속 작성
9. 모국어 최종확인
10. 제출

주의:

모바일 작성기능은 지금 당장 상품성의 핵심이 아니라, 종이 기록지 OCR 실증 품질이 안정된 뒤 확장하는 것이 좋다. 현재 상품화 핵심은 `익숙한 수기 흐름을 유지한 채 버려지던 종이를 예측 가능한 데이터로 구조화한다`는 점이다.

## 4. 검증 방법

### 데이터 검증

| 검증 항목 | 기준 |
---|---|
| Q1~Q5 추출률 | 변경 양식 30~50건 기준 문항별 누락률 측정 |
| 위험유형 분류 일치율 | 관리자 판단과 AI 분류 일치율 확인 |
| 무결성 검증 정확도 | 확인 필요로 잡힌 건의 실제 보정 필요 여부 확인 |
| 점수 일관성 | 유사 공종·유사 답변의 점수차 ±8점 이내 |
| 반복위험 추적 | 다음 달 동일 위험 언급 증감 확인 |
| 모국어 안내 품질 | 한글 혼입, 영문 시스템 용어, 문장 길이, 문체 검사 |
| 원페이지 출력 품질 | A4 한 장 넘침·잘림 0건 |
| 모바일 화면 품질 | 320~430px 주요 화면 넘침·잘림 0건 |

### 자동 검사로 추가할 항목

1. `check:psi-scope-language`
   - 사용자 화면과 제출 문서에서 불필요한 `TBM` 표현을 탐지한다.
2. `check:form-master-sync`
   - Q1~Q5 기준데이터와 PDF·모바일·OCR 프롬프트 문구 불일치를 탐지한다.
3. `audit:native-translation-master`
   - 언어별 승인 번역문, 검수 상태, 출력 넘침 여부를 점검한다.
4. `audit:feedback-loop`
   - 월별 기록 → 위험유형 → 교육자료 → 다음 달 변화 확인 연결 여부를 점검한다.

## 5. 실행 순서

### 1차 실행: 정체성 정렬

1. TBM 표현 제거 및 PSI 고유 표현으로 정리
2. 대외 설명문·브랜드 스토리·상품 설명서 문구 통일
3. AI 프롬프트의 환류 표현 변경
4. A4 교육자료 제목 변경
5. `check:psi-scope-language` 추가

완료 기준:

```text
PSI를 처음 보는 사람이 SMART TBM과 혼동하지 않고,
수기 위험성평가 기록지 기반 OCR·AI 환류 솔루션으로 이해해야 한다.
```

### 2차 실행: 기준데이터와 번역 승인체계

1. Q1~Q5 기준데이터 생성
2. 위험유형·공종·안전조치 표준분류 생성
3. 언어별 승인 번역문 상태관리 추가
4. PDF·모바일·교육자료의 기준데이터 연결
5. 출력 넘침 자동검사 연결

완료 기준:

```text
기록지 문구와 번역문이 화면마다 다르게 존재하지 않아야 한다.
```

### 3차 실행: 무결성 검증과 관리자 승인

1. 문항 간 모순 검증 규칙 추가
2. OCR 원문대조 표시 강화
3. 관리자 승인 전 확인 필요 사유 표준화
4. 관리자 수정 후 모국어 안내 자동 갱신 게이트 강화
5. 승인·수정·제외·현장확인 필요 상태를 통계화

완료 기준:

```text
AI 분석 결과가 공식 결과가 되기 전,
관리자가 왜 승인 또는 수정했는지 증빙이 남아야 한다.
```

### 4차 실행: 지표 근거와 월별 환류 증빙

1. 6대 지표 대외 표시명 정리
2. 지표별 산정근거 카드 추가
3. 전월 대비 변화 원인 표시
4. 반복 위험언어 탐지와 월별 비교 강화
5. 익명 대외 보고서와 기명 내부 보고서 구분

완료 기준:

```text
점수가 평가처럼 보이지 않고,
위험 인식과 교육 환류를 위한 보호 신호로 설명되어야 한다.
```

## 6. 최종 판단

첨부 보강안은 PSI 상품화 방향과 맞다. 다만 한 번에 모든 기능을 추가하는 방식은 위험하다. 지금 가장 큰 상품화 리스크는 기능 부족보다 `정체성 혼선`과 `설명 과장`이다.

따라서 첫 작업은 기능 추가가 아니라 다음 세 가지다.

1. SMART TBM과 PSI의 경계 분리
2. 5문항·6지표·관리자 검증·월별 환류의 기준 통일
3. 개인평가가 아니라 보호·교육 환류 시스템이라는 표현 고정

이 세 가지가 정리되면 PSI는 대외 발표에서 다음 한 문장으로 선명하게 설명할 수 있다.

> PSI는 근로자가 직접 쓴 위험성평가 기록지를 OCR·AI로 구조화하고, 관리자 검증을 거쳐 다음 달 교육자료와 월별 추적관리로 되돌리는 현장형 Safety Intelligence입니다.
