# NEW-PSI 공식 지표 카탈로그

최종 갱신: 2026-06-22

## 공통 운영 지표

계산 소유자: `utils/coreMetrics.ts`  
규칙 버전: `psi-core-metrics-2026-06-22-v1`

| 지표 | 공식 계산 기준 | 주요 사용 화면 |
|---|---|---|
| 근로자 수 | 현재 범위에서 동일 근로자를 한 명으로 집계 | 통합 보드, 대시보드 |
| 평균 응답품질 | 근로자별 최신 유효 점수 1건의 평균 | 통합 보드, 대시보드, 성과 분석 |
| 보호 우선 | 최신 점수가 현재 중급 임계값 미만인 인원 | 통합 보드, 대시보드 |
| 세부 분석 완료 | 최신 기록에 6개 지표 상세점수가 있는 인원 | 통합 보드 |
| 개선 이행률 | 최신 개선이행 점수(0~20)를 백분율로 환산한 평균 | 통합 보드, 성과 분석 |
| 대상 공종 | 최신 근로자 기록의 고유 공종 수 | 통합 보드 |
| 월 추세 | 근로자별·월별 최신 유효 기록 1건으로 계산 | 통합 보드, 성과 분석, 월 추세 차트 |

### 동일 근로자 판정 순서

1. 공종 + 이름 + 국적
2. 사번
3. QR 식별번호
4. 근로자 UUID
5. 기록 ID

월별 백업에서 자동 생성 사번이나 UUID가 달라진 동일 근로자도 공종·이름·국적이 같으면 지표 집계에서는 한 명으로 연결한다.

## 전문 지표별 계산 소유자

| 영역 | 계산 소유자 | 비고 |
|---|---|---|
| 안전등급 임계값 | `utils/safetyLevelUtils.ts` | 설정값을 반영하며 기본 기준은 고급 80점, 중급 60점 |
| 관리자 기준·근로자 체감 차이 | `utils/surveyRiskGap.ts` | 공종별 비교 가능 응답만 사용 |
| 6개 지표 원점수·총점 | `services/geminiService.ts` | 상세점수 합산과 총점 일치 검증 포함 |
| 근로자 누적 이력 | `utils/workerIdentity.ts` | 개인 리포트 기간·기록 수·점수 변화 |
| 보호사건 진행률 | `utils/safetyCase.ts` | 6단계 순차 완료 기준 |

## 변경 원칙

- 화면에서 공통 지표를 직접 다시 계산하지 않는다.
- 계산 기준을 바꾸면 규칙 버전과 테스트를 함께 변경한다.
- 저장된 등급 문자열과 점수가 다르면 보호 우선 판정은 현재 점수 임계값을 따른다.
- 유효하지 않은 점수는 평균과 보호 우선 계산에서 제외하되 근로자 수와 제외 건수는 보존한다.
- 릴리스 전 `npm run check:core-metrics`로 계산 소유권 회귀를 검사한다.
