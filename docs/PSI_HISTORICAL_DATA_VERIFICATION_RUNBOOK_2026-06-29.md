# 기존 운영 기록 기반 PSI 검증 절차

작성일: 2026-06-29  
대상: 작년부터 운영한 기존 위험성평가 기록지 분석자료 100~300명, 기존 양식, 새 모국어 양식 전환 검증

## 1. 결론

기존 운영 기록은 상품화 직전 검증에 가장 중요한 기준 데이터다. 새 양식은 아직 실사용 전이므로, 바로 새 양식만 테스트하면 품질을 판단하기 어렵다. 먼저 기존 운영 기록으로 현재 프로그램의 실제 성능을 숫자로 고정하고, 그다음 새 양식 표본을 같은 기준으로 비교해야 한다.

가장 좋은 순서는 다음과 같다.

1. 기존 운영 기록 전체를 JSON 백업으로 내보낸다.
2. 기존 기록 감사 리포트로 추적관리, 수기답변, Q1~Q5, 점수일관성, 모국어 안내 저장률을 확인한다.
3. 기존 양식 이미지 30~50장과 새 양식 이미지 30~50장을 같은 작업·같은 위험요인 기준으로 재분석한다.
4. 점수 차이, 문항 매핑 오류, OCR 실패율, 교육추천 일관성을 비교한다.
5. 통과하면 모바일 현장 스캔 모드를 상품화 우선 기능으로 확정한다.

## 2. 내가 확인하려면 필요한 파일

필수 파일:

- `reports/records-export.json`
- 프로그램에서 전체 기록 또는 백업 내보내기로 만든 JSON 파일
- 가능하면 `originalImage` 또는 스캔 이미지 데이터가 포함된 백업

권장 표본 폴더:

- `reports/form-comparison/old-form/`
- `reports/form-comparison/new-form/`
- 기존 양식 이미지 30~50장, 새 양식 이미지 30~50장
- 같은 공종과 비슷한 작업 상황이 섞이도록 준비

개인정보가 포함될 수 있으므로, 이 검증은 우선 로컬 PC 안에서만 진행한다.

## 3. 1차 검증 명령

기존 운영 기록 JSON을 `reports/records-export.json`에 둔 뒤 실행한다.

```bash
npm run audit:existing-records -- --input reports/records-export.json
```

생성되는 결과:

- `reports/existing-records-audit.json`
- `reports/existing-records-audit.md`

## 4. 1차 리포트에서 보는 항목

운영 데이터 사용성:

- 전체 기록 수
- 필수항목 충족률
- 날짜 오류와 미래 날짜
- 국적, 공종, 점수 분포

수기 판독 증거:

- 수기답변 저장률
- Q1 작업, Q2 위험요인, Q3 위험수준, Q4 감소대책, Q5 실천확인 슬롯 완성률
- 문항 누락 표본

추적관리 증거:

- 같은 근로자 반복 기록 그룹
- 여러 달에 걸친 추적 그룹
- 점수 개선, 하락, 유지 그룹

교육연계 증거:

- AI 코칭 저장률
- 한국어 해석 저장률
- 모국어 안내 저장률
- 낮은 점수 근로자 비율

새 양식 전환 준비:

- 원본 이미지 보존율
- OCR 실패 신호율
- 유사한 답변인데 점수 차이가 큰 기록

## 5. 합격 기준

1차 기존 기록 감사 기준:

- 필수항목 충족률 90% 이상
- 수기답변 저장률 90% 이상
- Q1~Q5 전체 슬롯 완성률 70% 이상
- AI 코칭 저장률 90% 이상
- 모국어 안내 저장률 80% 이상
- 유사 맥락 점수 차이 초과 0~5쌍 이하
- 반복 근로자 그룹 10개 이상
- 다월 추적 그룹 5개 이상

새 양식 비교 기준:

- Q1~Q5 문항 매핑 오류 3% 이하
- 기존 양식과 새 양식의 유사 사례 점수 차이 ±8점 이내 95% 이상
- 관리자 검토 포함 실사용 성공률 97% 이상
- 촬영부터 분석 결과까지 현장 기준 60초 이내
- 재촬영 안내가 1문장 안에 명확히 표시

## 6. 결과별 다음 단계

`READY_FOR_NEW_FORM_COMPARISON`:

- 기존 운영 기록은 기준 데이터로 사용 가능하다.
- 새 양식 30~50장 표본 촬영 테스트로 바로 넘어간다.

`READY_WITH_WARNINGS`:

- 상품화 검증은 가능하지만 일부 데이터 정리가 필요하다.
- 보통 원본 이미지 부족, 일부 문항 누락, 모국어 안내 누락이 원인이다.

`NEEDS_REVIEW`:

- 기존 데이터만으로 새 양식 전환 판단을 내리기 어렵다.
- 누락 항목을 보강하거나 표본을 다시 구성해야 한다.

## 7. 상품화 관점의 최종 판단 방식

이 프로그램의 핵심 가치는 단순 OCR이 아니다. 현장에서 수기로 쓴 위험성평가 기록지를 모바일이나 PC로 받아 근로자의 현재 위험 인식 수준을 파악하고, 그 결과를 교육과 추적관리로 이어주는 것이다.

따라서 최종 검증은 다음 네 문장으로 증명되어야 한다.

- 기존 실제 기록에서도 근로자별 변화가 추적된다.
- 기존 양식과 새 양식이 같은 Q1~Q5 구조로 안정적으로 정규화된다.
- 같은 위험 인식 수준이면 양식이 달라도 점수가 크게 흔들리지 않는다.
- 분석 결과가 바로 현장 교육, 확인, 재평가로 이어진다.

이 네 가지가 리포트로 확인되면, 새 양식 도입과 모바일 현장 스캔 모드는 상품화 막바리의 핵심 기능으로 확정할 수 있다.
