# PSI 2분 원포인트 증명 영상 스토리보드 검증·보강 계획

작성일: 2026-07-10  
대상: 2분 기술소개 및 시연 영상 스토리보드

## 1. 결론

현재 스토리보드의 큰 맥락은 PSI의 실제 정체성과 잘 맞는다.

핵심 흐름은 다음 한 줄로 정리된다.

**찍는다 → 확인한다 → 검증 결과를 환류한다 → 교육한다 → 추적한다**

다만 4번째 장면의 제목이 `리포트 생성`으로 되어 있으면, PSI가 단순히 개인 리포트를 만드는 프로그램처럼 보일 수 있다. 실제 PSI의 본질은 관리자 검증이 끝난 기록을 개인 보호 리포트, 다음 달 교육자료, 월별 추적자료로 되돌리는 것이다.

따라서 4번째 장면은 `리포트 생성`이 아니라 **검증 결과 환류**로 바꾸는 것이 맞다.

## 2. 현재 프로그램과 스토리보드 매핑 검증

| 1. 찍는다: 수기 기록지 사진/PDF 업로드 | OCR 분석 화면에 모바일/PC 업로드 흐름 존재 | 적합 |
| 2. 확인한다: OCR 추출 + AI 분석 + 관리자 검증 | OCR 분석, 상세판단, 관리자 검증/승인 흐름 존재 | 적합 |
| 4. 리포트 생성 | 실제로는 검증완료 기록이 보호 리포트와 교육 환류로 연결됨 | 제목과 설명 수정 필요 |
| 3. 교육한다: 다음 달 위험성평가 교육 반영 | 교육 환류 센터, 원페이지 교육자료, 다국어 안내 흐름 존재 | 적합 |

- `utils/educationReturnSummary.ts`: OCR 완료, 관리자 확인 필요, 다국어 환류, 월별 추적 요약 계산

- `utils/safetyLevelUtils.ts`: 점수/등급을 보호 신호 중심 표현으로 전환

## 4. 접근상 어려움과 보강 필요 지점

### 어려움 1. 리포트가 개인평가처럼 보일 수 있음

위험:
- 영상에서 점수, 단계, 리포트가 먼저 보이면 근로자 평가 시스템처럼 오해될 수 있다.

보강:
- 영상 자막에서 `개인평가` 대신 `보호 우선 신호`, `추가 안내`, `교육 환류`를 사용한다.
- 개인별 숫자는 작게, 환류 결과는 크게 보여준다.
- 나레이션에 “개인을 벌점화하는 것이 아니라 다음 교육과 보호 우선순위를 찾는 자료”를 반드시 넣는다.

### 어려움 2. 관리자 검증 전 데이터가 교육자료로 바로 넘어가는 것처럼 보일 수 있음

위험:
- AI가 자동으로 공식 결론을 내리는 것처럼 보이면 신뢰성과 책임성이 약해진다.

보강:
- 영상에서 `AI 분석` 다음에 반드시 `관리자 검증` 버튼 또는 확인 화면을 보여준다.
- 교육 환류는 `검증완료 기록 기준`이라는 자막을 넣는다.
- 프로그램 내부에서도 시연 모드에서는 검증완료/확인필요 상태가 명확히 보이게 한다.

### 어려움 3. 3번 교육과 4번 리포트의 순서가 헷갈릴 수 있음

위험:
- 리포트 생성과 교육자료 생성이 별도 기능처럼 보이면 PSI의 환류 구조가 약해진다.

보강:
- 순서는 `찍는다 → 확인한다 → 검증 결과 환류 → 교육한다 → 추적한다`로 설명한다.
- 다만 화면 하단 슬로건은 계속 `찍는다 → 확인한다 → 교육한다`로 단순화해도 된다.
- 4번째 장면은 중간 연결부로 두고, 5번째 장면에서 원페이지 교육자료 생성을 보여준다.

### 어려움 4. 다국어 교육/QR이 완성 기능처럼 과장될 수 있음

위험:
- 실제 현장에서는 다국어 교육/QR이 파일럿 상태인데 완성 기능처럼 보이면 발표 후 질문에서 흔들릴 수 있다.

보강:
- 영상에서는 `다국어 원페이지 안내 가능`, `파일럿 적용 중`으로 표현한다.
- 핵심은 다국어 자체가 아니라 “지난달 기록을 다음 달 교육자료로 바꾼다”는 점에 둔다.

### 어려움 5. 추적관리의 근거가 약해 보일 수 있음

위험:
- 단순 차트처럼 보이면 기존 안전 대시보드와 차별성이 줄어든다.

보강:
- 추적관리 화면에서는 반드시 `개인별`, `팀별`, `공종별`, `월별` 네 축을 보여준다.
- 반복위반은 현재 기록지 내부 반복 문장이 아니라 다음 달 동일 위험 재발/약속 행동 미이행 확인 시 추적 신호로 설명한다.

## 5. 제품 보강 실행 계획

### 1단계. 영상 문구 정리

우선순위: 매우 높음  
난이도: 낮음

수정:
- `리포트 생성` → `검증 결과 환류`
- `개인평가 아닌 보호 우선 리포트` → `검증완료 기록이 보호·교육자료로 전환`
- 나레이션에 “관리자 검증 후”를 명확히 추가

완료 기준:
- 스토리보드만 보고도 리포트가 개인평가가 아니라 교육 환류 자료임을 이해할 수 있어야 한다.

### 2단계. 2분 원포인트 증명 모드 동선 보강

우선순위: 높음  
난이도: 낮음~중간

현재:
- 4단계가 `모국어·리포트·추적`이며 기본 페이지가 `reports`로 연결되어 있다.

권장:
- 4단계 제목을 `검증 결과 환류`로 바꾼다.
- 기본 이동 화면을 `education-return`으로 바꾼다.
- 교육 환류 센터 안에서 `개인 보호 리포트`, `원페이지 교육자료`, `월별 추적자료`로 갈라지게 보여준다.

완료 기준:
- 2분 증명모드에서 각 장면 확인 후 다시 원포인트 화면으로 돌아왔을 때, 다음 순서가 자연스럽게 이어져야 한다.

### 3단계. 교육 환류 센터 표현 보강

우선순위: 높음  
난이도: 중간

현재:
- 교육 환류 센터는 이미 3개 결과물 구조가 잘 잡혀 있다.

보강:
- 상단 문구를 `검증된 기록은 세 가지 결과물로 돌아갑니다`에서 한 단계 더 명확하게 만든다.
- 추천 문구:
  - `관리자가 확인한 기록은 개인 보호 리포트, 다음 달 원페이지 교육자료, 월별 추적자료로 환류됩니다.`

완료 기준:
- 화면을 처음 보는 사람이 “기록 → 검증 → 교육 환류”를 5초 안에 이해해야 한다.

### 4단계. 검증완료 기준 강화

우선순위: 높음  
난이도: 중간

현재:
- 교육 환류 요약에서 점수가 있고 오류가 없으면 완료 기록으로 볼 수 있는 여지가 있다.

권장:
- 대외 시연/상품 화면에서는 `관리자 승인 또는 검증완료`된 기록만 교육 환류 기준으로 쓰는 표현을 강화한다.
- 확인 필요 기록은 교육자료 반영 전 관리자 확인이 필요하다고 분리한다.

완료 기준:
- AI 자동판정이 아니라 관리자 검증 기반 시스템이라는 신뢰가 생겨야 한다.

### 5단계. 영상용 데모 데이터 준비

우선순위: 높음  
난이도: 중간

준비:
- 비식별 수기 기록지 1장
- OCR 분석 결과 10건 이상
- 검증완료 기록과 확인필요 기록이 섞인 샘플
- 다음 달 교육자료 TOP3가 자연스럽게 나오는 샘플
- 개인·팀·공종·월별 추적이 보이는 샘플

완료 기준:
- 영상에서 수치가 서로 맞아야 한다.
- `OCR 완료`, `관리자 확인 필요`, `다국어 환류 가능`, `월별 추적` 수치가 서로 모순되지 않아야 한다.

> PSI는 기록을 끝내는 시스템이 아니라, 수기 기록을 관리자 검증과 교육 환류, 월별 추적관리로 되돌리는 현장형 Safety Intelligence입니다.

## 8. 최종 판단

이 스토리보드 방식으로 접근하는 데 큰 구조적 어려움은 없다. 오히려 현재 프로그램의 정체성을 가장 잘 설명하는 방향이다.

다만 상품화·공모전 발표 기준에서는 다음 세 가지를 반드시 보강해야 한다.

1. `리포트 생성`이 아니라 `검증 결과 환류`로 표현한다.
2. AI 분석 뒤에는 반드시 관리자 검증이 있다는 점을 강조한다.
3. 개인별 결과는 평가가 아니라 보호·교육 환류 신호라는 표현을 유지한다.

이 세 가지만 지키면, PSI는 단순 OCR 프로그램이 아니라 **수기 위험성평가 기록을 교육과 추적관리로 되돌리는 안전지능 시스템**으로 설득될 수 있다.
