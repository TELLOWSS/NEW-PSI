# PSI 실사용 UI 보강 되돌림 가이드

작성일: 2026-07-10  
목적: 실사용 조작성 보강 화면이 기대와 다를 때 빠르게 이전 구성으로 되돌린다.

## 1. 이번 보강의 기준점

이번 화면 보강 전 기준 커밋:

`a15eef3`

이번 보강은 OCR/AI 분석 로직이 아니라 화면 동선과 기본 메뉴 노출을 조정하는 작업이다.

## 2. 이번에 바꾼 범위

1. 2분 원포인트 증명 모드
   - 4단계 명칭을 `리포트·교육자료·추적`에서 `검증 결과 환류`로 변경
   - 4단계 기본 이동 화면을 `근로자 리포트`에서 `교육 환류 센터`로 변경

2. 교육 환류 센터
   - 상단 문구를 관리자 검증완료 기록의 환류 구조로 정리
   - `찍는다 → 검증한다 → 환류한다` 흐름을 더 명확히 표시
   - 버튼명을 `보호 리포트 확인`, `월별 추적 보기`로 정리

3. 현장용 기본 메뉴
   - 기본 사이드바를 현장 사용에 필요한 핵심 메뉴 중심으로 축소
   - 숨겨진 기능은 삭제된 것이 아니라 교육 환류 센터 내부 버튼이나 관리자/개발자 모드에서 접근 가능

## 3. 화면만 이상할 때 먼저 해볼 일

브라우저에 과거 메뉴 설정이 남아 있으면 새 기본 구성이 바로 보이지 않을 수 있다.

이 경우 시스템 설정에서 화면 구성을 초기화하거나, 브라우저 개발자 도구에서 아래 저장값을 지우면 된다.

- `psi_ui_composition_v1`

이번 보강에서는 UI 구성 버전을 올려 기존 기본 메뉴 구성이 자동으로 새 기본값으로 전환되도록 했다.

## 4. 코드 자체를 이전으로 되돌리는 방법

이번 보강 커밋만 되돌리면 된다.

예시:

```bash
git revert <이번_UI보강_커밋>
git push origin main
```

운영 배포까지 되돌릴 경우:

```bash
npx vercel --prod --yes
```

## 5. 되돌려도 유지되는 것

이번 보강은 화면 동선 중심 작업이므로, 되돌려도 다음 로직 보강은 유지된다.

- OCR 분석 로직
- 5문항·6대 지표 보정
- 반복위반 패널티 의미 정렬
- 관리자 검증 로직
- 기존 데이터

## 6. 판단 기준

되돌릴지 여부는 다음 기준으로 판단한다.

1. 현장 관리자가 시작 버튼을 10초 안에 찾는가
2. OCR 후 검증완료까지 자연스럽게 이어지는가
3. 검증완료 후 교육 환류로 가는 길이 명확한가
4. 메뉴가 너무 줄어 업무에 불편한가
5. 바이어/심사위원이 2분 흐름을 더 쉽게 이해하는가

화면이 너무 단순해져 실제 운영자가 불편하면 메뉴 노출만 되돌리고, 원포인트 증명과 교육 환류 문구는 유지하는 방식도 가능하다.
