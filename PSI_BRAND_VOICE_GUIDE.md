# PSI 브랜드 보이스 가이드

## 1. 브랜드 한 줄 정의
PSI는 현장의 신호를 정확하게 읽고 사람을 보호하는 안전 파트너다.

## 2. 핵심 페르소나
- **정체성**: 깐깐하지만 내 편인 현장 안전 코치
- **역할**: 평가자보다 조력자, 감시자보다 보호자
- **태도**: 정확하게 판단하되 사람을 압박하지 않음

## 3. 보이스 원칙
1. **평가보다 해석**
   - 점수만 말하지 않고 왜 그런 결과가 나왔는지 설명한다.
2. **지적보다 보완**
   - 문제를 단정하지 않고 다음 행동을 제안한다.
3. **감시보다 보호**
   - 사람을 판단하는 느낌보다 현장을 지키는 느낌을 준다.
4. **차가움보다 신뢰**
   - 과도하게 감성적이지 않되 딱딱하지 않게 쓴다.

## 4. 금지어
- 불합격
- 미달
- 불량 작성
- 문제 있는 답변
- 오류 문서

## 5. 권장어
- 보완 권고
- 추가 확인 필요
- 안전 포인트 보강 필요
- 우선 검토 필요
- 현장 설명 보강 권장

## 6. 역할별 메시지 원칙
### 근로자
- 목적: 심리적 저항을 낮추고 행동 전환을 유도
- 톤: 짧고 부드럽고 코칭형
- 예시: “위험 신호가 더 잘 드러나도록 안전 포인트를 조금만 보강해 주세요.”

### 관리자
- 목적: 빠른 판단과 우선순위 결정 지원
- 톤: 근거 중심, 실무형, 행동 지시형
- 예시: “추락 위험 항목 설명 부족으로 재검토 우선순위 상향”

### 경영진
- 목적: 운영 리스크와 예방 효과를 한눈에 전달
- 톤: 요약형, 지표형, 비교 가능 문장
- 예시: “반복 위험 패턴이 특정 공종에 집중되어 선제 개입이 필요합니다.”

## 7. UI 적용 원칙
- 결과 화면은 점수보다 의미 상태를 먼저 보여준다.
- 위험은 숨기지 않되 사람 비난처럼 보이지 않게 쓴다.
- 같은 데이터라도 역할별로 다른 표현 구조를 사용한다.
- 상태명은 시스템 내부 코드가 아니라 사용자가 이해할 언어로 변환한다.

## 8. 브랜드 문장 템플릿
- 종합 상태: “현재 현장 기록에서 {상태명}이 확인되었습니다.”
- 근거 설명: “{근거} 때문에 추가 확인이 필요합니다.”
- 행동 제안: “{다음 행동}을 보강하면 더 정확한 판단이 가능합니다.”
- 관리자 알림: “{위험유형} 기준으로 우선 검토 대상입니다.”

## 9. 시각 톤 요약
- 기본 인상: 차분함, 신뢰감, 보호감
- 컬러 감각: 네이비/슬레이트 기반 + 에메랄드/틸 보조
- 경고 표현: 레드 남용보다 의미 상태 중심 표시

## 10. 코드 상수 매핑 원칙
- 사용자 노출 상태명은 화면마다 하드코딩하지 않고 [utils/brandLabels.ts](utils/brandLabels.ts) 기준으로 우선 관리한다.
- `BRAND_STATUS_LABELS`
   - `attention` → 확인 필요
   - `attentionHold` → 확인 필요/보류
   - `attentionPending` → 추가 확인/대기
   - `supplementaryReview` → 보완 검토
   - `immediateAttention` → 즉시 확인 필요
- `BRAND_ACTION_LABELS`
   - `smartReanalyze` → 스마트 재분석
   - `directReanalyze` → 직접 재분석
   - `recheck` → 다시 확인
- 트래픽라이트·지적 상태도 공통 상수에서 파생한다.
- 새 화면을 추가할 때는 먼저 상수를 검토하고, 없으면 상수 파일에 추가한 뒤 화면에 연결한다.
