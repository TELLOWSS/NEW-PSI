# PSI 프로그램 정체성·비전·보강 로드맵 (Gemini / NotebookLM 업로드용)
작성일: 2026-03-17

---

## 1) 프로그램 정체성 (Identity)

### 1-1. 한 줄 정의
**PSI(Proactive Safety Intelligence)는 위험성평가 문서 기반의 현장 안전 이해도 검증·확약·증빙 자동화를 수행하는 운영형 AI 플랫폼**이다.

### 1-2. 존재 이유
- 외국인/다국적 현장에서 발생하는 **언어 장벽, 확인 누락, 형식적 서명** 문제를 줄인다.
- 안전교육이 “전달 완료”를 넘어 **실제 이해·행동 유도·감사 가능성**으로 이어지도록 만든다.
- 관리자는 대규모 인원(300+ 포함)도 **우선순위 기반으로 안정 운영**할 수 있어야 한다.

### 1-3. 현재 제품이 이미 확보한 핵심 성격
- OCR 분석 + 점수/등급/세부지표 정합성 유지
- 재검토 대기/최종확정 트러스트 상태 관리
- 점수 하향 사유코드 및 무결성 이력(해시/리포트 포함)
- API 할당량 제약 대응(스마트 재시도, 우선순위 처리)
- 11개국 다국어 교육(국기 선택, 언어별 MP3, 전자서명 게이트)

## 2) 비전 (Vision)

### 2-1. 단기 비전 (0~3개월)
- “**현장에서 바로 쓰는 안전 운영 콘솔**” 완성
- 관리자: 업로드·배포·모니터링·증빙 추출을 1화면에서 끝냄
- 근로자: 국기 선택 → 듣기/읽기 → 숙지 체크 → 서명까지 이탈 없이 완료

### 2-2. 중기 비전 (3~12개월)
- “**위험성평가 이해도 운영 표준**”으로 확장
- 교육·확약 데이터와 실제 사고/아차사고 지표를 연결해 **예측형 지표** 구축
- 현장/공종/국적별 패턴 분석으로 맞춤형 교육 루프 자동화

### 2-3. 장기 비전 (1~3년)
- “**감사·분쟁 대응 가능한 안전증빙 인프라**”로 고도화
- 단순 체크리스트 시스템이 아닌, **행동 전환율과 사고 저감률**을 입증하는 산업 안전 플랫폼으로 진화

## 3) 핵심 차별성 (Why PSI)

1. **다국어 전달 + 숙지 강제**
   - 언어별 음성/텍스트 제공, 국기 기반 직관 UX
   - 오디오 재생/스크롤 완료 전 서명 차단

2. **무결성 중심 설계**
   - 점수 하향 사유코드 필수화
   - 조정 이력, 감사 트레일, 증빙 해시 기반 추적

3. **현실 운영성**
   - 무료/제한 API 환경 대응 로직
   - 우선순위 큐 기반 재분석으로 자원 효율 극대화

4. **관리자 통제력**
   - 등급 컷오프/배치 크기/정책 설정 가능
   - 대규모 인원 운영시 병목 최소화

## 4) 추가 보강 과제 (우선순위 제안)

###[P0] 즉시 보강 (운영 안정성)
- **업로드/서명 이벤트 감사 로그 통합 뷰**
  - 누가, 언제, 어떤 언어, 어떤 파일 버전으로 배포했는지 추적
- **세션 만료/재발급 정책 명시화**
  - 링크 TTL, 재발급 이력, 무효화 정책 문서화
- **실패 복구 표준 운영절차(SOP)**
  - MP3 미업로드 언어 대응, 번역 fallback 기준, 현장 대응 문구 표준화

###[P1] 1차 고도화 (성과 가시화)
- **교육 완료율·숙지율·확약률 대시보드**
  - 현장/국적/작업반 단위 KPI 시각화
- **리스크 우선순위 추천 엔진**
  - 저숙지/반복위반/저신뢰 문항 자동 플래깅
- **정책 시뮬레이터**
  - 등급 컷오프·가중치 변경 시 분포 변화 예측

###[P2] 전략 고도화 (경영 보고/특허 확장)
- **사고·아차사고 연계 분석**
  - 교육/확약 데이터와 사고 데이터 상관 모델 구축
- **증빙 패키지 자동 생성**
  - 대외 감사용 PDF/CSV/해시 검증 세트 자동 출력
- **멀티 사이트 벤치마킹**
  - 현장 간 성과 비교 및 우수 패턴 전파

## 5) Gemini / NotebookLM 업로드 후 확인 질문 세트

아래 질문을 그대로 복사해 사용:

1. "PSI의 현재 정체성을 3개 키워드로 요약하고, 각 키워드의 근거를 문서에서 찾아 제시해줘."
2. "이 문서 기준으로 PSI의 가장 큰 실행 리스크 5개와 완화 전략을 우선순위로 제안해줘."
3. "P0/P1/P2 로드맵에서 90일 실행계획(주차별 산출물, KPI, 책임역할)을 표로 만들어줘."
4. "다국어 교육-서명 게이트 로직의 법적/감사 관점 강점과 취약점을 점검해줘."
5. "경영진 보고용 1페이지 요약본(문제-해결-성과-다음투자)을 작성해줘."
6. "특허/권리화 관점에서 신규 청구항 후보를 5개 제안해줘."

## 6) 확인 체크리스트 (업로드 후)

- [ ] 정체성 요약이 ‘AI 기능 소개’가 아니라 ‘운영 문제 해결’ 중심인지
- [ ] 비전이 추상적 표현이 아니라 KPI/기간 기준으로 내려왔는지
- [ ] 보강 과제가 기술 과제만이 아니라 운영·감사·법무까지 포함되는지
- [ ] 단기(90일) 실행계획에 담당/일정/측정지표가 있는지
- [ ] 기존 기능(무결성/다국어/서명게이트/쿼터대응)과 충돌 없는지

## 7) 최종 목표 문장 (대외 발표용)

**PSI는 ‘안전교육 전달 시스템’이 아니라, 다국어 현장에서 위험성평가의 이해·확약·증빙을 실제 행동 데이터로 연결하는 운영형 AI 안전 인프라를 지향한다.**

## 8) 경영진 1페이지 요약본 (발표용)

### 8-1. 경영진 메시지 (30초)
PSI는 다국어 현장에서 발생하는 안전교육의 형식화 문제를 해결하기 위해, **전달(음성/텍스트) → 이해확인(행동 게이트) → 확약(전자서명) → 증빙(감사 추적)**을 하나의 운영 체계로 묶은 플랫폼입니다.

### 8-2. 현재 해결한 문제
- 다국어 전달 실패: 11개국 언어 기준 음성/텍스트 동시 제공
- 형식적 확인: 오디오 재생 또는 대본 완독 전 서명 차단
- 감사 대응 부족: 조정 이력·해시·승인 흐름 기반 추적 가능
- 대규모 운영 병목: 실패/고위험 우선 처리 + 배치 분할 운영

### 8-3. 핵심 성과지표 (KPI 제안)
- 교육 완료율 = 제출 인원 / 대상 인원
- 숙지 확약률 = 이해+확약 인원 / 제출 인원
- 다국어 커버리지 = 음성 업로드 언어 수 / 11
- 분석 운영 안정도 = (성공 처리 건 / 전체 처리 건), 재시도 성공률 포함
- 감사 대응 리드타임 = 요청~증빙패키지 제출까지 소요시간

### 8-4. 투자 우선순위 (90일)
1) 운영 안정화: 업로드/서명 이벤트 감사 로그 통합
2) 가시화: 현장·국적·작업반별 KPI 대시보드 구축
3) 확장성: 사고/아차사고 연계 분석으로 예방지표 고도화

### 8-5. 기대 효과
- 현장: 교육 누락·오해 감소, 근로자 행동 일치도 향상
- 관리자: 대규모 인원 운영 자동화 및 리스크 집중관리
- 경영진: 안전활동을 정량 KPI로 모니터링하고 투자효과 검증 가능

## 9) 구현 가이드 (UI/UX/구성도 포함, 문서 기반 즉시 개발 가능)

## 9-1. 제품 정보구조 (IA)
- 관리자 플로우
  - 교육 세션 생성
  - 11개국 MP3 업로드
  - QR/링크 배포
  - 제출 현황/확약 통계 확인
  - 증빙 추출
- 근로자 플로우
  - QR 진입
  - 국기 탭으로 언어 선택
  - 오디오 청취 또는 대본 완독
  - 숙지 체크
  - 전자서명 제출

## 9-2. 화면 구성도 (텍스트 와이어프레임)

### A. 관리자 화면 (AdminTraining)
1) 헤더 영역
- 제목: 관리자 다국어 음성 안내 생성
- 설명: 관리자 화면 한국어 고정 안내

2) 원본 대본 입력 영역
- 대형 텍스트 입력
- placeholder: 한국어 위험성평가 핵심 문구

3) 11개국 MP3 업로드 그리드
- 각 카드: 국기 + 언어명 + 코드 + 파일입력
- 상태 라벨: 선택 파일명 / 업로드 URL 저장 여부
- 입력 제한: MP3만 허용

4) 생성/배포 영역
- 버튼: 세션 생성 및 업로드
- 성공 시: QR, 모바일 링크, 만료시각

5) 운영 모니터링 영역
- 제출 인원, 확약 인원, 확약률, 국적 수
- 최근 세션 목록, 재불러오기, 삭제

### B. 근로자 화면 (WorkerTraining)
1) 기본 정보
- 이름 입력
- 현재 선택 언어/국적 표기

2) 언어 선택 탭(가로 스와이프)
- 11개 국기 버튼
- 선택 시 즉시 오디오/대본 교체

3) 오디오 영역
- MP3가 있으면 audio player 노출
- 없으면 플레이어 숨김 + 대본 완독 안내

4) 대본 영역
- 번역 대본 스크롤 박스
- 스크롤 진척도(%) 표시
- 끝 도달 시 읽기 완료 상태 저장

5) 서명 게이트 영역
- 조건 충족 전: 체크박스/서명 캔버스 비활성
- 조건 충족 후: 체크박스 활성
- 체크 완료 후: 서명 캔버스 활성

6) 제출 영역
- 제출 버튼은 조건 미충족 시 비활성
- 성공/실패 메시지 표시

## 9-3. UI/UX 동작 규칙 (필수)

1) 언어 선택
- 탭 클릭 즉시 selectedLanguageCode 상태 변경
- selectedLanguageCode 기준으로 translated_texts, audio_urls 조회

2) 오디오 표시 규칙
- audio_urls[selectedLanguageCode] 존재 시만 player 렌더링
- 없으면 미업로드 안내 텍스트만 표시

3) 숙지 증빙 규칙
- hasEngagementProof = (audioPlayed OR scrolledToEnd)
- hasEngagementProof가 false면 체크박스/서명 비활성
- 체크박스 true일 때만 서명 캔버스 활성

4) 제출 방어 규칙(클라이언트+서버 동시)
- 이름 미입력 제출 차단
- hasEngagementProof=false 제출 차단
- 숙지 체크 미완료 제출 차단
- 서명 비어있음 제출 차단

## 9-5. API 구성도
- POST /api/admin/create-training
  - 세션 생성, 번역 텍스트 생성, 링크 발급
- POST /api/admin/upload-training-audio
  - 11개국 MP3 업로드 및 audio_urls 저장
- POST /api/admin/reissue-training-link
  - 만료 링크 재발급
- POST /api/training/submit-signature
  - 게이트 검증 후 서명 제출

## 9-6. 운영 시나리오 (E2E)
1) 관리자가 한국어 대본 입력
2) 11개국 MP3 첨부 후 세션 생성
3) 시스템이 QR/모바일 링크 발급
4) 근로자가 QR 접속 후 국기 선택
5) 오디오 청취 또는 대본 완독
6) 숙지 체크 + 전자서명
7) 서버가 게이트 검증 후 저장
8) 관리자는 통계/증빙으로 확인

## 9-7. QA 체크리스트 (개발 직후)
- [ ] 11개 국기 탭 전환 시 오디오/대본이 즉시 바뀌는가
- [ ] MP3 없는 언어에서 player가 숨겨지는가
- [ ] 오디오/완독 이전에 체크박스·서명 입력이 불가능한가
- [ ] 서버 API가 방어 조건 미충족 요청을 거절하는가
- [ ] 세션 삭제 시 storage 파일 정리가 되는가
- [ ] 링크 만료/재발급 동작이 정상인가

## 10) 경영진 발표용 PPT 8장 구성안 (즉시 사용 가능)

## 10-1. 슬라이드 1: 문제정의 (Why Now)
- 제목: 다국어 현장의 안전교육, 전달은 했지만 숙지는 증명되지 않았다
- 핵심 문장: 현재 현장 안전교육은 전달 중심이며, 실제 이해·확약·증빙의 연결이 약하다.
- 시각 요소: 기존 프로세스(전달→종이서명 종료)와 누락 지점(이해확인/감사증빙) 비교 도식
- 발표 멘트(20초): “우리는 교육을 ‘했는가’는 기록하지만, ‘이해했는가’는 증명하지 못하고 있습니다.”

## 10-2. 슬라이드 2: PSI 정의
- 제목: PSI = 전달·이해·확약·증빙을 연결하는 운영형 AI 플랫폼

### Q1. “기존 안전교육 시스템과 무엇이 본질적으로 다른가?”
A. “기존은 전달 완료 중심이고, PSI는 숙지 행동과 확약을 게이트로 검증한 뒤 증빙까지 연결합니다. 즉, 결과 데이터의 품질이 다릅니다.”

### Q2. “법적/감사 대응에 실제로 도움이 되는가?”
A. “예. 언어 선택, 대본 확인, 제출 시점, 서명 등 핵심 이벤트가 이력으로 남아 감사자료 생성 시간을 단축합니다.”

### Q3. “현장 적용 시 가장 큰 장애요인은?”
A. “초기 운영 습관 전환입니다. 이를 줄이기 위해 UI 단순화와 표준 운영절차(SOP) 및 현장 가이드 문구를 함께 제공합니다.”

### Q4. “ROI는 어떻게 측정할 것인가?”
A. “교육 완료율·숙지 확약률·재작업 감소·감사 대응 리드타임 감소를 월 단위로 추적해 투자 대비 효과를 확인합니다.”

### Q5. “확장성은 충분한가?”
A. “언어·현장·정책을 설정 기반으로 분리해두어, 다국적 확장과 멀티 사이트 운영에 맞춰 수평 확장이 가능합니다.”

## 14-8. Slide 8 의사결정 요청
1) 90일 고도화 예산 승인 요청
2) 파일럿 대상 현장 및 책임조직 지정 요청
3) KPI 월간 리뷰 체계 승인 요청
4) 승인 즉시 주차별 실행계획과 담당 매트릭스를 제출합니다.
5) PSI를 데모가 아닌 현장 표준 운영 체계로 전환하겠습니다.
