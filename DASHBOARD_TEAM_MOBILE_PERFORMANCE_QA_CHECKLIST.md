# 대시보드 팀별/모바일/성능 QA 체크리스트

## 1. 팀별 안전숙련도 분석
- [ ] 대시보드에서 `팀별 보기` 드롭다운이 정상 노출된다.
- [ ] `전체` 선택 시 전체 공종/국적 교차 차트가 보인다.
- [ ] 특정 팀 선택 시 해당 팀 데이터만 차트/레이더/개인추이에 반영된다.
- [ ] 팀 전환 시 이전 선택 작업조가 초기화된다.
- [ ] 팀 빠른 전환 칩 클릭 시 동일하게 팀 필터가 적용된다.
- [ ] 형틀 공종 선택 후 팀 대 팀 비교 보드가 표시된다.
- [ ] `이 팀만 보기` 버튼 클릭 시 해당 팀으로 즉시 전환된다.

## 2. 팀 비교 보드
- [ ] 선택 공종 기준 `최취약 팀` 카드가 표시된다.
- [ ] 선택 공종 기준 `최우수 팀` 카드가 표시된다.
- [ ] 정렬 기준 변경이 정상 동작한다.
  - [ ] 평균점수 낮은 순
  - [ ] 평균점수 높은 순
  - [ ] 고위험 인원 많은 순
  - [ ] 인원 많은 순
- [ ] 최신 반영일이 비정상 문자열 없이 표시된다.

## 3. 모바일 UX
- [ ] 모바일에서 분석 영역 탭이 보인다.
  - [ ] 차트
  - [ ] 팀비교
  - [ ] 개인추이
- [ ] 모바일에서 한 번에 한 섹션 중심으로 보여 화면 압박이 줄어든다.
- [ ] 팀 비교 카드가 가로 깨짐 없이 카드형으로 보인다.
- [ ] 개인별 트렌드가 테이블 대신 카드형으로 보인다.
- [ ] 미니 추이가 모바일 카드에도 표시된다.
- [ ] 가로 스크롤이 필요한 영역은 자연스럽게 스와이프된다.

## 4. 개인별 트렌드 추적
- [ ] 검색 입력으로 이름/공종/국적 검색이 된다.
- [ ] 빠른 필터가 정상 동작한다.
  - [ ] 전체
  - [ ] 고위험자
  - [ ] 상승자
  - [ ] 하락자
- [ ] 페이지네이션 `이전/다음` 버튼이 정상 동작한다.
- [ ] 페이지 이동 시 표시 인원 수가 맞다.
- [ ] 근로자 클릭 시 상세 모달이 열린다.
- [ ] 상세 모달에서 ESC 키로 닫힌다.

## 5. 지연 로딩 / 성능
- [ ] 첫 진입 시 대시보드 상단 KPI/요약 영역이 먼저 렌더링된다.
- [ ] 하단 분석 차트는 스크롤 근접 시 로드된다.
- [ ] 리포트/개인보고서 진입 시 템플릿이 지연 로드된다.
- [ ] PDF/ZIP/QR/엑셀 기능은 실제 실행 시점에만 로드된다.
- [ ] 초기 진입 시 브라우저 네트워크 탭에서 외부 스크립트 즉시 호출이 줄어든다.

## 6. 기능별 동작 확인
### 보고서
- [ ] 현재 보고서 PDF 내보내기 정상 동작
- [ ] 대량 ZIP 생성 정상 동작
- [ ] 증빙 ZIP 생성 정상 동작

### 개인 보고서
- [ ] PDF 저장 정상 동작
- [ ] 이미지 저장 정상 동작

### 관리자 교육
- [ ] QR 박스가 처음에는 로딩 placeholder 후 정상 표시된다.
- [ ] 풀스크린 QR도 정상 표시된다.

### 근로자 관리
- [ ] 엑셀 업로드 정상 동작
- [ ] 엑셀 템플릿 다운로드 정상 동작
- [ ] QR 렌더링 정상 동작

## 7. 회귀 점검
- [ ] `npm run build` 성공
- [ ] 콘솔 에러 없음
- [ ] 모바일 Safari/Chrome에서 주요 페이지 진입 가능
- [ ] 대시보드, OCR, 근로자관리, 리포트, 교육관리 페이지 전환 정상

## 8. 권장 점검 환경
- Desktop Chrome
- Mobile Chrome
- iPhone Safari
- 저사양 안드로이드 1대 이상

## 9. 비고
- Chart.js, jsPDF, html2canvas, JSZip, FileSaver, qrcodejs는 사용 시점 로딩 구조로 변경됨
- 초기 로딩보다 기능 실행 시 첫 호출이 약간 늦을 수 있으므로, 첫 실행 체감 확인 필요
