# OCR 운영 쉬운 가이드 (공종/팀 기준)

## 1) 핵심 개념
- 기록 양식: OCR 기록에 어떤 입력 항목을 사용할지 정한 템플릿
	- 예: 이름, 근로자ID(worker_id), 국적, 공종, 위험요인, 통제조치, 확인서명
- 공종/팀 그룹: 같은 양식을 적용할 운영 단위
	- 예: 철근 1팀, 형틀 A조, 미장 외주팀
- 배정: 특정 그룹에 특정 템플릿을 연결하는 설정

즉, **양식 만들기 → 그룹 만들기 → 그룹에 양식 배정** 순서로 사용하면 됩니다.

## 3) 기록 양식 관리 사용법

### 3-1. 언제 쓰는 기능인가?
현장마다 OCR 기록 양식이 다를 수 있기 때문에, 입력 항목 묶음을 템플릿으로 등록해 관리합니다.

예시:
- 위험성평가 전파교육 기본형
- 신규 입장자 확인형
- TBM 특화형

### 3-2. 입력 항목 설명
- 템플릿 이름: 운영자가 알아보기 쉬운 양식 이름
- 버전: 개정 이력 관리용 값
	- 예: `v1.0.0`, `v1.1.0`, `v2.0.0`
- 입력 항목 목록: 쉼표로 구분한 실제 기록 필드 목록

예시 입력:
- 템플릿 이름: `위험성평가 전파교육 기본형`
- 버전: `v1.0.0`
- 입력 항목 목록: `이름, 근로자ID(worker_id), 국적, 공종, 위험요인, 통제조치, 확인서명`

### 3-3. 등록 방법
1. 템플릿 이름 입력
2. 버전 입력
3. 입력 항목 목록 작성
4. [템플릿 추가] 클릭

등록되면 목록에 아래 정보가 보입니다.
- 템플릿 이름
- 버전
- 최근 업데이트 시각
- 입력 항목 요약

### 3-4. 삭제 방법
목록 오른쪽의 [삭제]를 누르면 템플릿이 삭제됩니다.

주의:
- 템플릿 삭제 시 해당 템플릿을 참조하던 배정도 함께 정리됩니다.
- 운영 중인 템플릿은 바로 삭제하지 말고, 새 버전 템플릿을 만든 뒤 교체하는 방식이 안전합니다.

## 4) 공종·팀 그룹 관리 사용법

### 4-1. 그룹이란?
그룹은 "어떤 팀/공종 묶음에 어떤 템플릿을 적용할지"를 정하는 운영 단위입니다.

예시:
- 철근 1팀
- 철근 2팀
- 형틀 A조
- 설비 야간팀

### 4-2. 그룹 추가 방법
1. [공종/팀 그룹명 추가] 입력칸에 이름 입력
2. [추가] 클릭

### 4-3. 그룹명 작성 권장 규칙
- 실제 현장 호칭과 동일하게 작성
- 너무 포괄적인 이름보다 운영자가 바로 이해할 수 있는 이름 사용
- 예: `철근`, `철근 1팀`, `형틀 B조`, `미장 외주팀`

---

## 5) 배정 등록 사용법

### 5-1. 배정이란?
배정은 "이 그룹은 이 양식을 사용한다"라고 연결해 두는 기능입니다.

### 5-2. 배정 저장 방법
1. [공종/팀 그룹 선택]에서 그룹 선택
2. [템플릿 선택]에서 사용할 템플릿 선택
3. [적용일] 선택
4. [배정 저장] 클릭

저장되면 목록에 아래 정보가 보입니다.
- 그룹명
- 템플릿명 / 버전
- 적용일
- 상태(`ACTIVE` / `INACTIVE`)

### 5-3. 상태 전환은 언제 쓰나?
- `ACTIVE`: 현재 사용 중
- `INACTIVE`: 과거 기록 보존용 / 임시 중단

예를 들어:
- 기존 양식을 중단하고 새 양식으로 교체할 때 기존 배정을 `INACTIVE`
- 테스트용 배정은 평소 `INACTIVE`

### 5-4. 배정 삭제는 언제 쓰나?
완전히 잘못 만든 배정이거나 더 이상 운영상 의미가 없는 경우 삭제합니다.

실무에서는 보통:
- 과거 이력 보존이 필요하면 `INACTIVE`
- 정말 불필요한 오배정이면 삭제

## 6) 실무 권장 운영 순서

### 기본 운영 예시
1. 현장에서 필요한 기록 양식 정의
2. 템플릿 1~2개 등록
3. 실제 공종/팀 이름으로 그룹 등록
4. 각 그룹에 적절한 템플릿 배정
5. OCR 분석 진행
6. 운영 변경이 생기면 새 템플릿 버전 추가
7. 이전 배정은 `INACTIVE`, 신규 배정은 `ACTIVE`

### 예시 시나리오
- 철근 1팀 → `위험성평가 전파교육 기본형 v1.0.0`
- 형틀 A조 → `TBM 특화형 v1.0.0`
- 4월부터 철근팀 양식이 바뀌면
	- 새 템플릿 `위험성평가 전파교육 기본형 v1.1.0` 생성
	- 철근 1팀에 새 배정 추가
	- 기존 배정은 `INACTIVE` 처리

## 7) 현재 기능 범위 (중요)
현재 구현 기준으로 이 기능은 **템플릿/그룹/배정 관리 데이터 등록 및 운영 관리 UI**까지 제공됩니다.

즉, 현재 가능한 것:
- 템플릿 생성 / 조회 / 삭제
- 그룹 생성 / 조회 / 삭제
- 배정 생성 / 조회 / 상태 전환 / 삭제

현재 코드 기준으로 아직 자동 연결되지 않은 것:
- OCR 분석 시 특정 근로자의 공종/팀을 읽어 자동으로 템플릿을 선택해주는 완전 자동 적용 로직
- 배정된 템플릿의 `field_schema`를 OCR 추출 프롬프트/입력폼에 직접 주입하는 로직

따라서 지금은 **운영 기준을 정리하고 관리하는 기능**으로 이해하면 가장 정확합니다.

## 8) 저장되는 곳(DB)
이 기능은 아래 테이블/뷰를 사용합니다.

- `record_master_templates`
	- 템플릿 이름, 버전, 입력 항목(field_schema) 저장
- `record_master_assignments`
	- 그룹-템플릿 연결, 상태, 적용일 저장
- `record_master_groups`
	- 공종/팀 그룹 목록 조회용

관련 로딩/저장 로직은 [pages/OcrAnalysis.tsx](pages/OcrAnalysis.tsx#L308-L551)에 있습니다.

## 9) DB 적용 안내
현재 시스템은 중단 없는 전환을 위해 구 용어(company)와 신 용어(group)를 함께 지원합니다.

- 앱은 기본적으로 `group` 뷰/컬럼을 먼저 조회하고, 없으면 자동으로 기존 `company` 구조로 폴백합니다.

### 먼저 실행
- supabase_master_template_migration.sql

### 그 다음 실행
- supabase_master_group_compat_migration.sql
- supabase_master_group_postcheck.sql

위 파일들은 아래를 처리합니다.
- assignments 테이블에 `group_id` 병행 컬럼 추가
- `company_id`와 `group_id` 자동 동기화 트리거 추가
- `record_master_groups` 뷰 제공

운영자는 아래 문서를 따라 순서대로 실행하면 됩니다.
- SUPABASE_GROUP_MIGRATION_CHECKLIST.md
- SUPABASE_GROUP_CUTOVER_RUNBOOK.md
- supabase_master_group_cutover_rollback.sql

## 10) 자주 묻는 질문

### Q. 왜 company라는 이름이 아직 DB에 남아있나요?
- 기존 코드/데이터를 끊김 없이 유지하기 위한 호환층입니다.
- 화면과 신규 운영 문서는 group 기준으로 전환 중입니다.

### Q. 지금 당장 운영에 영향 있나요?
- 없습니다.
- 호환 방식이라 기존 데이터는 그대로 사용 가능합니다.

### Q. 배정을 만들면 OCR 결과 양식이 자동으로 바뀌나요?
- 현재 구현 기준으로는 아직 아닙니다.
- 지금은 운영 관리용 마스터 데이터 등록/관리 기능이 우선 제공됩니다.

### Q. 나중에 완전 전환도 가능한가요?
- 가능합니다.
- 충분히 안정화된 뒤 최종 rename 마이그레이션을 추가하면 됩니다.
