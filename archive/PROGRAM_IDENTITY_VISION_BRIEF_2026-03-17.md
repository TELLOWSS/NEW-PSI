## 0) 전체 구현 총망라 (누적 성과 기준)

아래 항목은 “최근 변경”이 아니라, 현재 PSI가 확보한 **누적 구현 범위 전체**를 경영진/심사/협업 관점에서 한 번에 볼 수 있도록 정리한 목록이다.

### 0-1. 위험성평가 AI 분석 엔진 고도화
- OCR 분석 결과의 **점수-등급-세부지표 정합성** 보정 로직 구축
- 심리 분석(필압/배치 이상) 신호를 구조화하여 위험 판단 입력값으로 반영
- 신뢰도 기반 신호등 배지(트래픽 라이트) 도입으로 관리자 의사결정 시간 단축
- 분석 실패/쿼터 초과 상황에서 재시도·백오프·우선순위 처리 로직 고도화

### 0-2. 무결성/감사 추적 체계 구축
- 점수 하향 시 사유코드 필수화 및 조정 이력 기록
- 무결성 점수 계산 유틸(이력 대비 불일치 탐지)과 경고 체계 도입
- 증빙 리포트/해시 중심 감사 대응 구조 확립
- 보고서 생성시 실패 항목 추적/피드백/타임아웃 방어로 운영 신뢰성 강화

### 0-3. 관리자/근로자 모드 분리 및 보안 강화
- 관리자 셸과 근로자 UI를 분리해 오조작 가능성 축소
- 관리자 수동 잠금 + 유휴 자동 잠금(10분) 적용
- 서명 링크 토큰(`sessionId + exp + sig`) 기반 위변조/만료 검증 적용
- 중복 서명(동일 세션+동일 이름) 서버 차단 로직 적용

### 0-4. 11개국 다국어 교육·확약 워크플로우 완성
- 관리자 화면에서 11개 언어 MP3 업로드/세션 생성/QR 배포/재발급/이력 관리
- 근로자 화면에서 국기 기반 언어 전환(스와이프/탭) 및 언어별 오디오·대본 동기화
- 오디오 청취 또는 대본 완독 전에는 숙지체크/서명 제출이 불가능한 게이트 적용
- 오디오 미업로드 언어는 텍스트 대체 경로로 안전하게 처리

### 0-5. API 안정성·장애 복원력 강화
- 관리자 API 응답 파싱 공통화(content-type 검사 + 안전 파서 + Raw 에러 노출)
- `create-training`에서 빈 세션 생성 허용(기본값/다단계 Insert fallback)
- Supabase 권한 경로 보강(service-role 우선, 실패 시 상세 원인 노출)
- Vercel 환경에서 함수 타임아웃/초기화 크래시 방지를 위한 방어(요청 파싱 방어, 번역 스킵 조건, 외부 호출 timeout, 함수 maxDuration)
- 비JSON/HTML 오류 응답에서도 프론트가 실제 실패 원인을 그대로 표시하도록 개선

### 0-6. 데이터/스토리지/정책 계층 정비
- `training_audio` 버킷 및 Storage 정책(RLS) 정비
- `training_sessions`의 `audio_urls`, `original_script` 등 운영 필드 보강
- 다국어 교육 서명 데이터(`training_acknowledgements`) 기반 통계 활용 구조 정립

### 0-7. 운영 문서·검증 자산 축적
- 배포 체크리스트, 관리자 운영가이드, QA/검증 문서, 릴리즈 노트 체계화
- 테스트 자산(무결성/배지/대량 시나리오) 누적 확보
- 최근 변경은 모두 빌드 검증 기준 통과 상태로 유지

### 0-8. 브리핑/PPT에서 반드시 강조할 핵심 메시지
1. PSI는 기능 모음이 아니라 **전달→이해→확약→증빙**을 닫는 운영 시스템이다.
2. 다국어/대규모/저예산 API 제약 조건에서도 실운영 가능한 복원력을 입증했다.
3. 최근 개선은 “신규 기능 추가”보다 “중단 없는 운영과 감사 대응력” 강화에 집중되어 있다.

## 1) 프로그램 정체성 (Identity)

### 1-1. 한 줄 정의
**PSI(Proactive Safety Intelligence)는 위험성평가 문서 기반의 현장 안전 이해도 검증·확약·증빙 자동화를 수행하는 운영형 AI 플랫폼**이다.

### 1-2. 존재 이유
- 외국인/다국적 현장에서 발생하는 **언어 장벽, 확인 누락, 형식적 서명** 문제를 줄인다.
- 안전교육이 “전달 완료”를 넘어 **실제 이해·행동 유도·감사 가능성**으로 이어지도록 만든다.
- 관리자는 대규모 인원(300+ 포함)도 **우선순위 기반으로 안정 운영**할 수 있어야 한다.

### 1-3. 현재 제품이 이미 확보한 핵심 성격
- OCR 분석 + 점수/등급/세부지표 정합성 유지
- 재검토 대기/최종확정 트러스트 상태 관리
- 점수 하향 사유코드 및 무결성 이력(해시/리포트 포함)
- API 할당량 제약 대응(스마트 재시도, 우선순위 처리)
- 11개국 다국어 교육(국기 선택, 언어별 MP3, 전자서명 게이트)

## 2) 비전 (Vision)

### 2-1. 단기 비전 (0~3개월)
- “**현장에서 바로 쓰는 안전 운영 콘솔**” 완성
- 관리자: 업로드·배포·모니터링·증빙 추출을 1화면에서 끝냄
- 근로자: 국기 선택 → 듣기/읽기 → 숙지 체크 → 서명까지 이탈 없이 완료

### 2-2. 중기 비전 (3~12개월)
- “**위험성평가 이해도 운영 표준**”으로 확장
- 교육·확약 데이터와 실제 사고/아차사고 지표를 연결해 **예측형 지표** 구축
- 현장/공종/국적별 패턴 분석으로 맞춤형 교육 루프 자동화

### 2-3. 장기 비전 (1~3년)
- “**감사·분쟁 대응 가능한 안전증빙 인프라**”로 고도화
- 단순 체크리스트 시스템이 아닌, **행동 전환율과 사고 저감률**을 입증하는 산업 안전 플랫폼으로 진화

## 3) 핵심 차별성 (Why PSI)

1. **다국어 전달 + 숙지 강제**
   - 언어별 음성/텍스트 제공, 국기 기반 직관 UX
   - 오디오 재생/스크롤 완료 전 서명 차단

2. **무결성 중심 설계**
   - 점수 하향 사유코드 필수화
   - 조정 이력, 감사 트레일, 증빙 해시 기반 추적

3. **현실 운영성**
   - 무료/제한 API 환경 대응 로직
   - 우선순위 큐 기반 재분석으로 자원 효율 극대화

4. **관리자 통제력**
   - 등급 컷오프/배치 크기/정책 설정 가능
   - 대규모 인원 운영시 병목 최소화

## 4) 추가 보강 과제 (우선순위 제안)

###[P0] 즉시 보강 (운영 안정성)
- **업로드/서명 이벤트 감사 로그 통합 뷰**
  - 누가, 언제, 어떤 언어, 어떤 파일 버전으로 배포했는지 추적
- **세션 만료/재발급 정책 명시화**
  - 링크 TTL, 재발급 이력, 무효화 정책 문서화
- **실패 복구 표준 운영절차(SOP)**
  - MP3 미업로드 언어 대응, 번역 fallback 기준, 현장 대응 문구 표준화

###[P1] 1차 고도화 (성과 가시화)
- **교육 완료율·숙지율·확약률 대시보드**
  - 현장/국적/작업반 단위 KPI 시각화
- **리스크 우선순위 추천 엔진**
  - 저숙지/반복위반/저신뢰 문항 자동 플래깅
- **정책 시뮬레이터**
  - 등급 컷오프·가중치 변경 시 분포 변화 예측

###[P2] 전략 고도화 (경영 보고/특허 확장)
- **사고·아차사고 연계 분석**
  - 교육/확약 데이터와 사고 데이터 상관 모델 구축
- **증빙 패키지 자동 생성**
  - 대외 감사용 PDF/CSV/해시 검증 세트 자동 출력
- **멀티 사이트 벤치마킹**
  - 현장 간 성과 비교 및 우수 패턴 전파

### 4-1. 개인안전역량세부지표 보강안 (최종 기준 제안)

현재 PSI의 강점은 **위험성평가 기록지 작성내용 자체의 품질**을 구조화하고 점수화하는 데 있다. 다만 실제 현장 운영에서는 이것만으로 충분하지 않다. 다음달 위험성평가를 앞두고 근로자가 **모국어 5분 교육 음성**을 듣고, **전파교육**을 이수하고, **위험성평가기록지**를 작성한 뒤에도, 교육 이전 현장 실천 수준과 불안전행동 발생 여부, 재교육·코칭 이행 여부까지 함께 검증되어야 진짜 무결성이 성립한다.

따라서 최종 기준은 **문서 품질 평가**와 **현장 실천 평가**를 분리하지 않고, 하나의 증빙 체계로 묶는 방식이 적합하다.

#### 4-1-1. 최종 판단 원칙
1. **기록지 내용이 좋아도 현장 실천 증빙이 없으면 확정 등급을 주지 않는다.**
2. **불안전행동이 발생했으면 반드시 재교육 또는 코칭 이력이 연결되어야 한다.**
3. **교육 이수 → 기록지 작성 → 현장 확인 → 재평가**의 시간 순서가 맞지 않으면 무결성 보류로 본다.
4. **동일 위험유형 재발**은 단순 감점이 아니라 별도 패널티와 관리자 확인 대상으로 분리한다.
5. **모국어 교육 이수 증빙**은 전달 완료가 아니라 이해·실천 전환의 출발점으로만 인정한다.

#### 4-1-2. 권장 평가 구조: 2축 6지표

| 구분 | 세부지표 | 핵심 질문 | 주요 증빙 |
|------|----------|-----------|-----------|
| 문서축 | 기록지 무결성 | 위험요인·예방대책·작업내용이 구체적인가 | OCR 원본, 수정이력, 최종 작성본 |
| 문서축 | 업무/위험 이해도 | 공종과 위험요인의 연결이 맞는가 | 공종 정보, 작성 답변, 관리자 검토 |
| 문서축 | 위험성평가 이해도 | 예방대책이 실제 작업 통제와 연결되는가 | 답변 근거, AI 분석, 관리자 코멘트 |
| 실천축 | 교육 이수도 | 모국어 음성 청취·전파교육·숙지확약이 완료되었는가 | 오디오 청취, 완독, 서명, 교육세션 로그 |
| 실천축 | 개선이행도 | 불안전행동 이후 재교육·코칭·시정조치가 완료되었는가 | 조치이력, 코칭 메모, 재교육 기록 |
| 실천축 | 반복위반도 | 동일/유사 위험행동이 재발했는가 | 불안전행동 로그, 월별 비교, 관리자 판정 |

#### 4-1-3. 권장 점수식

기존 가중치 체계는 유지하되, 해석 기준을 아래처럼 보강한다.

$$
P = w1\cdot 기록지무결성 + w2\cdot 업무이해도 + w3\cdot 위험성평가이해도 + w4\cdot 교육이수도 + w5\cdot 개선이행도 - w6\cdot 반복위반도
$$

- `w1`: 기록지 구체성, 상투어 여부, 문서-공종 일치성
- `w2`: 작업내용과 위험요인 연결 정확도
- `w3`: 예방대책의 적절성, 현장 적용 가능성
- `w4`: 모국어 음성 청취, 전파교육 참석, 숙지 체크, 서명 완료
- `w5`: 재교육, 코칭, 시정조치 완료 및 재확인 결과
- `w6`: 동일 위험행동 재발 빈도 및 중대성 패널티

즉, **기록지를 잘 썼는가**와 **실제로 행동이 개선되었는가**를 같은 점수식 안에서 보되, `w4~w6`이 실천축 무결성을 담당하도록 정의하는 것이 적절하다.

#### 4-1-4. 무결성 검증 게이트 (확정/보류 기준)

다음 5개 게이트를 모두 통과해야만 `확정` 처리한다.

1. **교육 게이트**
  - 모국어 음성 청취 또는 대본 완독 완료
  - 전파교육 참석 또는 대리확인 이력 존재
  - 숙지확약 서명 완료

2. **문서 게이트**
  - 위험요인, 위험성 감소대책, 작업내용이 비어있지 않음
  - 공종과 무관한 허위성 답변이 아님
  - 상투어 반복만으로 구성되지 않음

3. **행동 게이트**
  - 교육 전 최근 관찰기간 내 불안전행동 발생 여부 기록
  - 불안전행동 발생 시 유형, 날짜, 관찰자, 사진/메모 중 1개 이상 존재

4. **개선 게이트**
  - 불안전행동이 있었다면 재교육 또는 현장코칭 이력 필수
  - 조치 후 재확인 결과(`개선됨`, `재발`, `확인중`) 중 하나가 기록됨

5. **시간순서 게이트**
  - 불안전행동 발생 → 재교육/코칭 → 모국어 교육/전파교육 → 기록지 작성 → 관리자 확인 순서가 논리적으로 성립

위 5개 중 하나라도 누락되면 등급은 `확정`이 아니라 `검증보류` 또는 `관리자 검토필요`로 남겨야 한다.

#### 4-1-5. 현장 운영용 판정 규칙

| 상황 | 판정 | 운영 조치 |
|------|------|-----------|
| 기록지 우수 + 교육 이수 완료 + 불안전행동 없음 | 확정 가능 | 정상 등급 반영 |
| 기록지 우수 + 불안전행동 있음 + 코칭 없음 | 무결성 보류 | 코칭/재교육 등록 전 확정 금지 |
| 기록지 보통 + 교육 이수 + 코칭 후 개선 확인 | 조건부 확정 | 한 단계 보수적 등급 또는 추적관리 |
| 기록지 우수 + 동일 위험행동 반복 | 감점 + 관리자 검토 | 반복위반 패널티 적용 |
| 교육 미이수 또는 서명 없음 | 자동 보류 | 등급 산정 대상 제외 |

#### 4-1-7. 최종 운영 결론

최종적으로 PSI의 개인안전역량세부지표는 **기록지 작성 품질 평가 시스템**에서 멈추면 안 된다. 반드시 **교육 이수**, **불안전행동 관찰**, **재교육·코칭 이행**, **재발 여부**까지 연결된 폐쇄 루프 구조여야 한다. 이 구조를 기준으로 해야만 다음달 위험성평가 기록지가 단순 문서가 아니라, **실제 행동 변화를 입증하는 무결성 증빙**으로 기능할 수 있다.

## 8) 경영진 1페이지 요약본 (발표용)

### 8-1. 경영진 메시지 (30초)
PSI는 다국어 현장에서 발생하는 안전교육의 형식화 문제를 해결하기 위해, **전달(음성/텍스트) → 이해확인(행동 게이트) → 확약(전자서명) → 증빙(감사 추적)**을 하나의 운영 체계로 묶은 플랫폼입니다.

### 8-2. 현재 해결한 문제
- 다국어 전달 실패: 11개국 언어 기준 음성/텍스트 동시 제공
- 형식적 확인: 오디오 재생 또는 대본 완독 전 서명 차단
- 감사 대응 부족: 조정 이력·해시·승인 흐름 기반 추적 가능
- 대규모 운영 병목: 실패/고위험 우선 처리 + 배치 분할 운영

### 8-3. 핵심 성과지표 (KPI 제안)
- 교육 완료율 = 제출 인원 / 대상 인원
- 숙지 확약률 = 이해+확약 인원 / 제출 인원
- 다국어 커버리지 = 음성 업로드 언어 수 / 11
- 분석 운영 안정도 = (성공 처리 건 / 전체 처리 건), 재시도 성공률 포함
- 감사 대응 리드타임 = 요청~증빙패키지 제출까지 소요시간

### 8-4. 투자 우선순위 (90일)
1) 운영 안정화: 업로드/서명 이벤트 감사 로그 통합
2) 가시화: 현장·국적·작업반별 KPI 대시보드 구축
3) 확장성: 사고/아차사고 연계 분석으로 예방지표 고도화

### 8-5. 기대 효과
- 현장: 교육 누락·오해 감소, 근로자 행동 일치도 향상
- 관리자: 대규모 인원 운영 자동화 및 리스크 집중관리
- 경영진: 안전활동을 정량 KPI로 모니터링하고 투자효과 검증 가능

## 9-2. 화면 구성도 (텍스트 와이어프레임)

### A. 관리자 화면 (AdminTraining)
1) 헤더 영역
- 제목: 관리자 다국어 음성 안내 생성
- 설명: 관리자 화면 한국어 고정 안내

2) 원본 대본 입력 영역
- 대형 텍스트 입력
- placeholder: 한국어 위험성평가 핵심 문구

3) 11개국 MP3 업로드 그리드
- 각 카드: 국기 + 언어명 + 코드 + 파일입력
- 상태 라벨: 선택 파일명 / 업로드 URL 저장 여부
- 입력 제한: MP3만 허용

4) 생성/배포 영역
- 버튼: 세션 생성 및 업로드
- 성공 시: QR, 모바일 링크, 만료시각

5) 운영 모니터링 영역
- 제출 인원, 확약 인원, 확약률, 국적 수
- 최근 세션 목록, 재불러오기, 삭제

### B. 근로자 화면 (WorkerTraining)
1) 기본 정보
- 이름 입력
- 현재 선택 언어/국적 표기

2) 언어 선택 탭(가로 스와이프)
- 11개 국기 버튼
- 선택 시 즉시 오디오/대본 교체

3) 오디오 영역
- MP3가 있으면 audio player 노출
- 없으면 플레이어 숨김 + 대본 완독 안내

4) 대본 영역
- 번역 대본 스크롤 박스
- 스크롤 진척도(%) 표시
- 끝 도달 시 읽기 완료 상태 저장

5) 서명 게이트 영역
- 조건 충족 전: 체크박스/서명 캔버스 비활성
- 조건 충족 후: 체크박스 활성
- 체크 완료 후: 서명 캔버스 활성

6) 제출 영역
- 제출 버튼은 조건 미충족 시 비활성
- 성공/실패 메시지 표시

## 9-3. UI/UX 동작 규칙 (필수)

1) 언어 선택
- 탭 클릭 즉시 selectedLanguageCode 상태 변경
- selectedLanguageCode 기준으로 translated_texts, audio_urls 조회

2) 오디오 표시 규칙
- audio_urls[selectedLanguageCode] 존재 시만 player 렌더링
- 없으면 미업로드 안내 텍스트만 표시

3) 숙지 증빙 규칙
- hasEngagementProof = (audioPlayed OR scrolledToEnd)
- hasEngagementProof가 false면 체크박스/서명 비활성
- 체크박스 true일 때만 서명 캔버스 활성

4) 제출 방어 규칙(클라이언트+서버 동시)
- 이름 미입력 제출 차단
- hasEngagementProof=false 제출 차단

- 자동판정 결과(`확정`, `검증보류`, `재교육필요`, `관리자검토`) 배지 표시
- 관리자 승인/반려 코멘트 입력

## 9-7. 운영 시나리오 (E2E 확장)
1) 관리자가 다음달 위험성평가용 모국어 교육 세션을 생성
2) 교육 전 현장 순회 중 불안전행동을 관찰하면 근로자별 관찰 기록 등록
3) 불안전행동 발생 근로자에게 재교육 또는 현장코칭 조치 등록
4) 근로자가 QR 접속 후 국기 선택
5) 5분 모국어 음성 청취 또는 대본 완독
6) 숙지 체크 + 전자서명 제출
7) 근로자가 위험성평가기록지 작성 및 OCR 분석 반영
8) 서버가 기록지 품질, 교육 로그, 불안전행동, 코칭 이력을 합산해 월별 무결성 자동판정 수행
9) 관리자가 `검증보류` 또는 `관리자검토` 건만 후속 확인 후 확정
10) 최종적으로 증빙 패키지에 교육-행동-조치-판정 타임라인을 함께 저장

## 9-8. 무결성 자동판정 로직 (의사코드)

if (!educationCompleted || !signatureSubmitted) {

  integrityStatus = '검증보류';

if (repeatViolationCount >= 2) {

if (!timelineIsValid) {

  reasonCodes.push('TIMELINE_MISMATCH');
}

if (documentScore >= threshold && educationCompleted && (!unsafeBehaviorFlag || followupResult === '개선됨')) {
  integrityStatus = integrityStatus === '관리자검토' ? '관리자검토' : '확정';

## 10-1. 슬라이드 1: 문제정의 (Why Now)
- 제목: 다국어 현장의 안전교육, 전달은 했지만 숙지는 증명되지 않았다
- 핵심 문장: 현재 현장 안전교육은 전달 중심이며, 실제 이해·확약·증빙의 연결이 약하다.
- 시각 요소: 기존 프로세스(전달→종이서명 종료)와 누락 지점(이해확인/감사증빙) 비교 도식

- 제목: PSI = 전달·이해·확약·증빙을 연결하는 운영형 AI 플랫폼
- 핵심 문장: PSI는 문서 OCR부터 다국어 교육, 서명 게이트, 감사 추적까지 하나의 운영 체계로 통합한다.
- 시각 요소: 4단계 파이프라인(전달→이해확인→확약→증빙)
- 발표 멘트(20초): “PSI의 차별점은 기능 집합이 아니라 운영 루프를 완성했다는 점입니다.”

## 10-3. 슬라이드 3: 현재 구현 성과
- 제목: 이미 동작하는 핵심 기능

  - 11개국 언어별 MP3 + 국기 기반 선택 UX

“기대 효과는 세 가지입니다. 첫째, 현장 측면에서 교육 누락과 오해를 줄입니다. 둘째, 관리자 측면에서 대규모 인원 운영의 병목을 줄이고 고위험 대상에 자원을 집중할 수 있습니다. 셋째, 경영 측면에서 안전활동을 정량 데이터로 확인할 수 있어 투자 효과를 검증할 수 있습니다. 결국 PSI는 안전교육을 비용 항목이 아니라 성과 관리 항목으로 전환하는 기반입니다.”

## 11-5. Slide 5 스크립트 (KPI)
“경영진 관점의 핵심 KPI는 다섯 가지로 제안드립니다. 교육 완료율, 숙지 확약률, 다국어 커버리지, 운영 안정도, 감사 대응 리드타임입니다. 이 지표는 단순 사용량이 아니라 실제 운영 품질과 리스크 대응 수준을 보여줍니다. 월 단위로 트렌드를 보면 현장별 편차와 개선 효과가 명확해지고, 향후 투자 우선순위를 데이터 기반으로 결정할 수 있습니다.”

## 11-6. Slide 6 스크립트 (90일 계획)
“90일 계획은 단계적으로 구성했습니다. 1단계는 운영 안정화입니다. 로그, SOP, 링크 정책을 정비해 실패 비용을 줄입니다. 2단계는 KPI 대시보드 구축으로 가시성을 확보합니다. 3단계는 사고·아차사고 데이터 연계 파일럿을 시작해 예방지표를 고도화합니다. 이 순서를 따르면 단기 성과와 중장기 확장성을 동시에 확보할 수 있습니다.”

## 11-7. Slide 7 스크립트 (리스크)
“리스크는 세 축으로 관리합니다. 데이터 품질 리스크는 입력 검증과 게이트 강화로 통제합니다. 사용자 이탈 리스크는 UI 단순화와 현장 안내문 표준화로 대응합니다. 정책·권한 리스크는 RLS, 관리자 Secret, 감사로그로 통제합니다. 핵심은 리스크를 제거하는 것이 아니라, 위험이 발생해도 운영이 흔들리지 않는 제어 구조를 만드는 것입니다.”

## 11-8. Slide 8 스크립트 (의사결정 요청)
“오늘 요청드리는 의사결정은 세 가지입니다. 첫째, 90일 고도화 실행 승인. 둘째, 파일럿 현장 선정 승인. 셋째, KPI 월간 리뷰 체계 승인입니다. 이 세 가지가 확정되면 PSI는 기술 데모 단계에서 벗어나, 현장 운영 표준으로 확장될 수 있습니다. 승인 즉시 1주차 실행계획과 담당 역할표를 제출하겠습니다.”

## 12) 경영진 예상 질문 & 답변 템플릿

### Q1. “기존 안전교육 시스템과 무엇이 본질적으로 다른가?”
A. “기존은 전달 완료 중심이고, PSI는 숙지 행동과 확약을 게이트로 검증한 뒤 증빙까지 연결합니다. 즉, 결과 데이터의 품질이 다릅니다.”

### Q2. “법적/감사 대응에 실제로 도움이 되는가?”
A. “예. 언어 선택, 대본 확인, 제출 시점, 서명 등 핵심 이벤트가 이력으로 남아 감사자료 생성 시간을 단축합니다.”

### Q3. “현장 적용 시 가장 큰 장애요인은?”
A. “초기 운영 습관 전환입니다. 이를 줄이기 위해 UI 단순화와 표준 운영절차(SOP) 및 현장 가이드 문구를 함께 제공합니다.”

### Q4. “ROI는 어떻게 측정할 것인가?”
A. “교육 완료율·숙지 확약률·재작업 감소·감사 대응 리드타임 감소를 월 단위로 추적해 투자 대비 효과를 확인합니다.”

### Q5. “확장성은 충분한가?”
A. “언어·현장·정책을 설정 기반으로 분리해두어, 다국적 확장과 멀티 사이트 운영에 맞춰 수평 확장이 가능합니다.”

## 14) PPT 복붙용 본문 텍스트 (슬라이드별 5줄)

## 14-1. Slide 1 문제정의
1) 다국어 현장 안전교육은 전달 중심으로 운영되어 숙지 증명이 약합니다.
2) 서명 완료가 이해 완료를 의미하지 않는 구조적 한계가 존재합니다.
3) 언어 장벽과 운영 편차로 교육 품질이 현장마다 다르게 나타납니다.
4) 결과적으로 감사·분쟁 대응 시 증빙 완결성이 부족해집니다.
5) PSI는 이 단절을 전달→이해→확약→증빙의 루프로 연결합니다.

## 14-2. Slide 2 PSI 정의
1) PSI는 위험성평가 교육을 운영 체계로 표준화하는 플랫폼입니다.
2) 다국어 음성/텍스트를 제공해 전달 정확도를 높입니다.
3) 행동 게이트로 이해 여부를 확인한 뒤 확약을 받습니다.
4) 전 과정 이벤트를 저장해 감사 가능한 증빙으로 전환합니다.
5) 핵심은 AI 기능이 아니라 운영 품질의 일관성 확보입니다.

## 14-3. Slide 3 현재 구현 성과
1) 11개국 언어 기준 MP3 업로드·국기 선택 UX가 동작합니다.
2) 언어 선택 시 오디오와 번역 대본이 즉시 전환됩니다.
3) 오디오 재생/대본 완독 전에는 서명 제출이 차단됩니다.
4) 점수 조정 무결성 이력과 감사 추적 체계를 적용했습니다.
5) 대규모 운영을 위한 우선순위 재처리 로직도 확보했습니다.

## 14-4. Slide 4 비즈니스 효과
1) 현장: 교육 누락과 오해를 줄여 행동 일치도를 높입니다.
2) 관리자: 고위험 대상 중심으로 운영 자원을 집중할 수 있습니다.
3) 경영진: 안전활동 성과를 KPI 기반으로 모니터링할 수 있습니다.
4) 감사: 제출 이력과 증빙 패키지 생성 리드타임이 단축됩니다.
5) 결과적으로 안전활동이 비용에서 성과지표로 전환됩니다.

## 14-5. Slide 5 KPI
1) 교육 완료율: 제출 인원 / 대상 인원
2) 숙지 확약률: 이해·확약 인원 / 제출 인원
3) 다국어 커버리지: 음성 업로드 언어 수 / 11
4) 운영 안정도: 성공 처리율 + 재시도 성공률
5) 감사 대응 리드타임: 요청부터 증빙 제출까지 소요시간

## 14-6. Slide 6 90일 계획
1) 0~30일: 로그·SOP·링크 정책 중심 운영 안정화
2) 31~60일: 현장/국적/작업반 KPI 대시보드 구축
3) 61~90일: 사고·아차사고 연계 파일럿 실행
4) 단계별 산출물로 리스크를 관리하며 확장성을 확보합니다.
5) 90일 내 경영 보고 가능한 성과지표 체계를 완성합니다.

## 14-7. Slide 7 리스크와 통제
1) 데이터 품질 리스크는 입력 검증·게이트 강화로 통제합니다.
2) 사용자 이탈 리스크는 UX 단순화·현장 안내 표준화로 완화합니다.
3) 권한 리스크는 RLS·관리자 Secret·감사로그로 방어합니다.
4) 기술보다 중요한 것은 운영 통제 구조의 일관성입니다.
5) 리스크를 제거하기보다, 관리 가능한 상태로 유지합니다.

## 14-8. Slide 8 의사결정 요청
1) 90일 고도화 예산 승인 요청
2) 파일럿 대상 현장 및 책임조직 지정 요청
3) KPI 월간 리뷰 체계 승인 요청
4) 승인 즉시 주차별 실행계획과 담당 매트릭스를 제출합니다.
5) PSI를 데모가 아닌 현장 표준 운영 체계로 전환하겠습니다.
