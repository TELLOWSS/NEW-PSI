## 1. 결론

승인표에 표시된 삭제·통합·문구·역할·개인정보·호환성 결정을 프로그램에 반영했다. 별도지시 A1·A2는 프런트엔드 호출 경로와 Vercel 운영 로그를 함께 점검한 뒤, 완전 대체된 음성 전용 API는 삭제하고 외부 호환 가능성이 있는 교육대상 API는 통합 API로 연결되는 만료 예정 호환 경로로 유지했다.

## 2. 승인사항 반영 결과

### 삭제 및 정리

- 내용이 없거나 실제로 사용되지 않는 컴포넌트·페이지·테스트·유틸리티·구형 API 24개를 삭제했다.
- 구형 PPT/PDF 요약 화면은 현재 A4 교육자료 화면으로 호환 이동되도록 정리했다.
- 중복 모바일 바로가기, 중복 상태·조치 패널, 중복 교육완료 버튼, 작동하지 않는 월간지도 카드와 브라우저 알림을 제거했다.
- 루트에 흩어진 과거 문서 126개는 삭제하지 않고 `docs/archive/root-history`로 이동해 보존했다.
- 실행에 필요한 현재 문서와 특허·마이그레이션·감사·보안 자료는 유지했다.

### 공통화 및 중복 제거

- 패널, 인사이트 그리드, 배지, 보고서 실행버튼을 공통 UI 구성요소로 통합했다.
- Supabase 서버 연결과 구형 스키마 호환 경고를 공통 서버 모듈로 통합했다.
- 직종·교육언어 목록을 단일 기준 데이터로 통합해 화면과 API 간 불일치 가능성을 줄였다.
- 개인·그룹 서명 저장은 법적 의미와 서명 방식은 그대로 유지하면서 내부 저장 로직만 공통화했다.

### 역할·개인정보·상품 정책

- 관리자용 앱의 현장 역할을 `현장 실무 관리자`로 명확히 하고, 일반 근로자는 QR·키오스크 교육 흐름으로 분리했다.
- 개인 상위근로자 순위를 제거하고 3명 이상인 팀·직종만 익명 집계하는 `팀별 안전 실천 신호`로 교체했다.
- `안전조치 통합 허브`를 현장 실무자와 개발자에게 정식 메뉴로 노출하고 즉시 사용 화면으로 편입했다.
- 운영 진단·피드백은 개발자 진단 모드에만 노출되도록 유지했다.
- 개발·목업 중심 표현을 구매자·현장 사용자가 이해하는 상품 언어로 다듬었다.

## 3. 별도지시 A1·A2 API 확인 및 조치

| 항목 | 확인 결과 | 조치 |
|---|---|---|
| 교육 음성 갱신 전용 API | 현재 관리자 화면은 통합 `/api/admin/training`의 `upload-audio`를 사용하며 구형 전용 경로를 호출하는 화면이 없음 | 완전 대체된 `update-training-audio` 삭제 |
| 교육대상 갱신 API | 현재 관리자 화면은 통합 `/api/admin/training`의 `update-targets`를 사용함. 외부 연동 가능성은 운영 로그 보존기간 밖일 수 있어 완전 부재를 단정할 수 없음 | `update-training-targets`를 통합 API 연결용 호환 경로로 축소하고, 사용 중단·후속 경로·2026-10-30 만료 예정 헤더 추가 |

Vercel 최근 운영 로그에서 두 구형 경로와 통합 교육 API의 호출 기록은 확인되지 않았다. 로그 보존기간 한계가 있으므로, 외부 호출 가능성이 있는 교육대상 경로는 즉시 삭제하지 않는 보수적 조치를 적용했다.
