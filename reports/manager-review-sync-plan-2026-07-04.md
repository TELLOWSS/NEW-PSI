# 관리자 수정 동기화 상품화 계획

검토일: 2026-07-04

## 1. 핵심 결론

상품화 관점에서 관리자 확인검증은 단순한 수동 수정 화면이 아니라, 다음 3단계가 한 번에 닫혀야 한다.

1. 관리자가 한국어 보호 해석 또는 판정 내용을 수정한다.
2. 근로자에게 전달될 모국어 안내가 같이 갱신 또는 검증된다.
3. 개인 안전역량 세부지표와 종합점수가 같은 기준으로 다시 계산되고 감사기록에 남는다.

이번 개선은 이 흐름을 `관리자 수정 동기화 검증` 규칙으로 묶었다.

## 2. 사용자 관점 흐름

처음 보는 관리자는 다음 순서만 이해하면 된다.

1. OCR 결과를 연다.
2. 한국어 해석을 확인하고 필요한 부분만 고친다.
3. `수정 반영 재분석` 또는 일괄 2차 재분석을 실행한다.
4. 프로그램이 모국어 안내, 점수, 6대 지표를 같이 갱신한다.
5. 누락이나 품질 의심이 있으면 확인 필요 문구와 감사기록으로 남는다.

관리자는 번역·채점·추적관리 로직을 따로 이해할 필요가 없다. 수정하면 뒤쪽 계산과 안내가 따라오는 구조가 핵심이다.

## 3. 바이어 설득 포인트

| 질문 | 답변 |
|---|---|
| 관리자가 내용을 고치면 외국인 근로자 안내도 바뀌는가? | 2차 재분석에서 `aiInsights_native`, `score_reason_native`, `actionable_coaching_native`를 다시 생성하고, 저장 시 누락 여부를 검증한다. |
| 개인 안전역량 지표도 같이 바뀌는가? | 저장 직전 `deriveCompetencyProfile` 기준으로 `competencyProfile`을 재계산한다. |
| 나중에 왜 점수가 바뀌었는지 확인 가능한가? | `auditTrail`에 `관리자 수정 동기화 검증` 규칙 버전, 역량점수 변화, 누락·품질 이슈가 남는다. |
| 번역이 빠지거나 품질이 낮으면 어떻게 되는가? | 모국어 보호 안내 누락, 문항 번역 누락, 영어 혼입 등을 검증 이슈로 남기고 승인 전 확인 대상으로 보낸다. |
| 기존 데이터와 새 데이터가 같은 기준으로 관리되는가? | 상세 모달 저장 경로와 OCR 분석 화면의 일괄 2차 재분석 경로가 같은 공통 동기화 유틸을 사용한다. |

## 4. 무결성 메커니즘

적용 규칙: `psi-manager-review-sync-2026-07-04-v1`

- 점수·등급 정합성: 저장 전 `enforceSafetyLevel`로 점수 기준 등급을 다시 맞춘다.
- 개인 안전역량: `deriveCompetencyProfile`로 심리, 업무이해도, 위험성평가 이해도, 숙련도, 개선이행도, 반복지적 보완 지표를 재계산한다.
- 모국어 검증: `evaluateOcrVerificationCompleteness`, `evaluateOcrVerificationQuality`로 모국어 안내 누락, 문항 번역 누락, 영어 혼입, 점수 과대 의심을 확인한다.
- 감사기록: 변경 저장 시 `auditTrail`에 동기화 결과와 규칙 버전을 남긴다.

## 5. 다음 고도화 권장

1. 관리자 수정 화면에 `모국어 안내 갱신 필요` 배지를 더 명확히 표시한다.
2. 모국어 안내가 비어 있거나 품질 이슈가 있으면 승인 버튼을 보류하거나 재분석 버튼을 우선 노출한다.
3. 바이어 데모용으로 “관리자 문장 수정 전/후 → 모국어 안내 변경 → 6대 지표 변화” 1건을 자동 샘플 리포트로 출력한다.
4. 실제 현장 5~10건을 골라 수정 전/후 지표 변화와 모국어 안내 품질을 검수표로 남긴다.
