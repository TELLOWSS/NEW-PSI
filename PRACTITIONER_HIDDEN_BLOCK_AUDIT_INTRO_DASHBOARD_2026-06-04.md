## 2) Introduction 감사

### Block I-1
- 파일: `pages/Introduction.tsx`
- 근거 구간: QA 경보/런로그 영역 (`1070` 근처)
- 현재 노출 내용
  - `QA 경보 모드`
  - `QA 정상 모드`
  - `QA RUNLOG`
  - `12 SCREEN FEATURE CHECK`
- 판정
  - 실무자 비노출
- 이유

  - 실무자는 기능 상태보다 어디로 들어가 무엇을 해야 하는지가 먼저 보여야 함
- 권장 조치
  - 실무자 모드에서는 블록 전체 비노출
  - 관리자/개발자 모드에서만 노출
  - 실무자 대체 블록은 `오늘 바로 시작할 기능` 2~3개 카드로 교체

  - QA 대상 건수
  - 권한/환경 설정 문맥
  - 연결/데이터 준비도
- 판정
  - 실무자 비노출 또는 관리자 상세로 이동
- 이유
  - 실무자에게는 준비도 지표보다 작업 시작점이 중요함
- 권장 조치
  - 소개 카드에서는 `오늘 점검`, `경보 확인`, `문서 보기` 수준으로 단순화

## 5) 후속 구현 규칙
1. `isDevMode`만으로는 부족하다. `uiAudienceMode` 기준 가시성 제어를 함께 써야 한다.
2. 실무자 비노출 블록은 CSS 숨김이 아니라 렌더 자체 제거로 처리한다.
3. 역할별 대체 문구는 공통 사전에서 가져온다.
